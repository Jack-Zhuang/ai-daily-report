## 编辑理念

### 核心价值观
1. **准确性第一** - 宁可不发，不可发错
2. **价值导向** - 每篇内容都要有价值
3. **用户视角** - 用用户听得懂的语言
4. **持续精进** - 永远追求更好

### 内容标准
- **专业** - 准确、权威、有深度
- **易读** - 通俗、流畅、有温度
- **有用** - 实用、可操作、有价值
- **有趣** - 生动、有故事、有共鸣

## 内容规范

### 标题规范

#### 好标题的特征
1. **信息量足** - 一眼看懂内容
2. **吸引眼球** - 引发阅读欲望
3. **真实准确** - 不夸大不误导
4. **长度适中** - 20-30 字为宜

#### 标题类型

**学术类**（论文解读）
```
❌ 差：一篇关于 Agent 的论文
✅ 好：从被动到主动：首个 AI Agent 主动性评估框架

❌ 差：LLM 新研究
✅ 好：路由攻击：揭示 LLM 路由系统的安全漏洞
```

**技术类**（GitHub 项目）
```
❌ 差：一个好用的工具
✅ 好：LangChain：构建 LLM 应用的瑞士军刀

❌ 差：Python 库推荐
✅ 好：5 分钟搭建 AI Agent：Hermes 框架入门指南
```

**资讯类**（热门文章）
```
❌ 差：Claude 发布新功能
✅ 好：终于，Claude 革了设计行业的命：Figma、Adobe 股价重挫

❌ 差：AI 行业动态
✅ 好：多云时代 AI Agent 治理难题，亚马逊云科技想用一个注册表解决
```

#### 标题公式
```
[动词] + [对象] + [结果/影响]
例：揭秘 + LLM 路由漏洞 + 导致的安全风险

[数字] + [形容词] + [名词]
例：5 个 + 必学的 + AI Agent 框架

[对比] + [结论]
例：Claude vs GPT-4：谁才是真正的代码助手？
```

### 摘要规范

#### 摘要的作用
1. **快速判断** - 帮助读者决定是否阅读
2. **信息提炼** - 提取核心观点
3. **SEO 优化** - 提升搜索排名

#### 摘要写作要点
- **长度**：100-200 字
- **结构**：背景 + 问题 + 方案 + 结果
- **语言**：简洁、准确、无废话

#### 摘要模板

**学术论文摘要**
```
【背景】随着 AI Agent 的广泛应用，如何评估其主动性成为关键问题。
【问题】现有评估方法主要关注被动响应能力，缺乏对主动行为的量化指标。
【方案】本文提出 Proactive Agent Assessment Framework（PAF），首次系统性定义了 Agent 主动性的评估维度。
【结果】实验表明，PAF 能够有效区分不同 Agent 的主动性水平，为 Agent 设计提供了新思路。
```

**技术项目摘要**
```
【定位】LangChain 是一个用于构建 LLM 应用的开源框架。
【特点】提供链式调用、记忆管理、工具集成等核心能力，支持快速搭建复杂 AI 应用。
【适用】适合需要将 LLM 集成到业务系统的开发者，特别是 RAG、Agent 等场景。
【亮点】文档完善、社区活跃、生态丰富，是目前最流行的 LLM 应用开发框架。
```

**新闻资讯摘要**
```
【事件】Claude 发布全新设计工具，可直接生成 UI 设计稿。
【影响】设计师工作效率提升 10 倍，Figma、Adobe 股价应声下跌。
【分析】这是 AI 首次真正进入专业设计领域，标志着"AI 替代白领"时代的到来。
【展望】预计更多垂直领域的 AI 工具将涌现，传统软件面临重构。
```

### 正文规范

#### 结构化写作

**金字塔结构**
```
核心观点（最重要）
    ↓
支撑论据（次重要）
    ↓
细节补充（可略读）
```

**SCQA 结构**
```
Situation（情境）- 背景介绍
Complication（冲突）- 问题提出
Question（问题）- 核心疑问
Answer（答案）- 解决方案
```

#### 段落规范
- **一段一意** - 每段只讲一个观点
- **首句概括** - 第一句概括全段
- **长度适中** - 3-5 句为宜
- **过渡自然** - 段落间有衔接

#### 语言规范

**避免的问题**
```
❌ 啰嗦：在这个过程当中，我们需要注意的是...
✅ 简洁：注意...

❌ 模糊：这个东西比较好用
✅ 具体：该工具支持一键部署，节省 80% 配置时间

❌ 口语：这个论文挺牛的
✅ 专业：该研究在 Agent 评估领域具有开创性意义

❌ 夸张：史上最强、颠覆一切
✅ 客观：在同类工具中表现优异
```

**推荐的用法**
```
✅ 数据支撑：实验表明准确率提升 15%
✅ 对比说明：相比 GPT-3.5，GPT-4 在推理任务上提升显著
✅ 案例佐证：以某公司为例，采用该方案后效率提升 3 倍
✅ 专家观点：图灵奖得主 Bengio 认为...
```

### 翻译规范

#### 翻译原则
1. **信** - 准确传达原意
2. **达** - 语言流畅自然
3. **雅** - 符合中文表达习惯

#### 常见问题

**术语翻译**
```
Agent → 智能体（不用"代理"）
LLM → 大语言模型
RAG → 检索增强生成
Fine-tuning → 微调
Prompt → 提示词（不用"提示"）
Embedding → 向量化/嵌入
Token → 词元（不用"令牌"）
```

**句式调整**
```
❌ 直译：It is suggested that we should consider...
✅ 意译：建议考虑...

❌ 直译：The model which was proposed by...
✅ 意译：该模型由...提出

❌ 直译：There are many researchers studying this problem
✅ 意译：众多研究者致力于解决这一问题
```

## 内容运营

### 数据分析

#### 核心指标
- **阅读量** - 内容吸引力
- **完读率** - 内容质量
- **分享率** - 传播价值
- **收藏率** - 实用价值
- **评论数** - 互动程度

#### 分析维度
- 时间分布（何时发布效果好）
- 内容类型（哪类内容受欢迎）
- 标题特征（什么样的标题点击率高）
- 来源渠道（用户从哪里来）

### 用户反馈

#### 收集渠道
- 评论留言
- 私信反馈
- 问卷调查
- 用户访谈

#### 反馈分类
- **内容质量** - 准确性、深度、实用性
- **阅读体验** - 排版、加载速度、移动端
- **功能需求** - 新功能建议
- **Bug 反馈** - 错误报告

### 迭代优化

#### 基于数据的优化
- 阅读量低 → 优化标题
- 完读率低 → 优化结构和内容
- 分享率低 → 增加传播点
- 收藏率低 → 增加实用价值

#### 基于反馈的优化
- 用户说看不懂 → 降低门槛
- 用户说太浅 → 增加深度
- 用户说没用 → 增加实操性
- 用户说太长 → 精简内容
