## 📖 背景与动机

随着GPT-4、Claude等大语言模型在翻译任务上的表现超越传统NMT系统，越来越多的商业翻译场景开始采用LLM作为翻译引擎。然而，与传统神经机器翻译不同，LLM的生成式特性导致其可能输出超出原文范围的"过度生成"内容，这些内容既包括有益的上下文补充，也可能包含危险的虚假信息。与NMT中常见的"神经废话"（neurobabble）不同，LLM的过度生成呈现出更复杂的形态：从看似合理的自我解释，到可能误导读者的风险性虚构，再到模拟人类译者的恰当增译。如何准确识别这些不同类型的过度生成，成为商业部署中的核心挑战。


**关键要点：**

- LLM翻译正在商业场景快速普及，但生成式特性带来独特的质量风险
- 传统NMT的"神经废话"与LLM的"过度生成"有本质区别
- 过度生成并非全然有害，恰当的增译可提升读者理解
- 区分有益和有害的过度生成是工业落地的关键难题

## 💡 核心方法

### 方法概述

该研究采用多维度分类框架，将LLM翻译过度生成划分为三大类别，并设计了基于规则与模型相结合的混合检测策略，在商业环境中进行系统验证。


### 详细设计

研究团队首先建立了系统的过度生成分类体系：类型一为"自我解释型"，即LLM在翻译过程中加入原文并未明确表达的推理过程；类型二为"风险性虚构型"，指生成内容包含事实性错误或未经证实的敏感信息；类型三为"适当解释型"，是模拟专业人类译者的合理增译，如补充文化背景、解释术语等。

在检测策略方面，团队探索了多种方法：第一是基于规则的启发式方法，通过关键词匹配和模式识别捕捉明显的过度生成信号；第二是基于小型分类器的轻量级方法，使用BERT等模型对翻译片段进行二分类或多分类；第三是Prompt引导的元认知检测，通过设计专门的提示词让LLM自我评估输出是否超出原文范围；第四是混合集成方法，结合多种策略的优势进行综合判断。实验部分采用混淆矩阵对各类检测方法进行细粒度评估，重点关注精确率、召回率在不同过度生成类型上的差异。

class RuleBasedDetector:
    """关键词匹配 + 模式识别"""

        self.risky_keywords = ['未经证实', '据说', '可能']
        self.self_explain_patterns = ['我认为', '推理', '这说明']
    
    def detect(self, translation: str) -> Tuple[int, float]:
        """返回: (类型, 置信度)"""
        score = 0
        
        # 检测风险性虚构
        for kw in self.risky_keywords:
            if kw in translation:
                score += 0.4

            if pat in translation:
                score += 0.3
        
        if score > 0.5:

        return 0, score  # 无过度生成

class EnsembleDetector:
    """综合多种策略"""
    
    def __init__(self):
        self.rule_detector = RuleBasedDetector()
        self.cls_detector = ClassifierDetector()
        self.weights = [0.2, 0.5, 0.3]  # 各策略权重
    
    def detect(self, src: str, tgt: str) -> Tuple[int, float]:
        # 各检测器独立预测
        _, r_score = self.rule_detector.detect(tgt)
        cls_pred, cls_conf = self.cls_detector.detect(src, tgt)
        
        # 加权投票
        votes = np.zeros(4)
        votes[cls_pred] += self.weights[1] * cls_conf
        votes[0] += self.weights[0] * (1 - r_score)  # 规则否决
        
        final_pred = np.argmax(votes)
        final_conf = votes[final_pred]
        return final_pred, final_conf

# ============ 评估模块：混淆矩阵 + 细粒度指标 ============
class Evaluator:
    """使用混淆矩阵评估各检测方法"""
    
    def evaluate(self, y_true: List[int], y_pred: List[int]) -> Dict:
        n_classes = 4
        cm = np.zeros((n_classes, n_classes), dtype=int)
        
        for true, pred in zip(y_true, y_pred):
            cm[true][pred] += 1
        
        metrics = {}
        for i in range(1, n_classes):  # 跳过0(无越界)
            tp = cm[i][i]
            fp = cm[:, i].sum() - tp
            fn = cm[i, :].sum() - tp
            
            precision = tp / (tp + fp) if (tp + fp) > 0 else 0
            recall = tp / (tp + fn) if (tp + fn) > 0 else 0
            
            metrics[f'type_{i}'] = {
                'precision': precision,
                'recall': recall
            }
        
        return {'confusion_matrix': cm, 'metrics': metrics}

## ❓ 常见问题

**Q：为什么LLM翻译的过度生成与NMT的"神经废话"不同？**

A：NMT的神经废话通常是模型缺陷导致的乱码，而LLM的过度生成是模型遵循指令产生的有结构内容，往往看似合理且通顺，需要更精细的语义分析才能识别。


**Q：如果只想保留"适当解释"类型的增译，该怎么做？**

A：需要先通过分类器区分"适当解释"与"风险虚构"，然后对"适当解释"片段进行保留或增强，对其他类型进行回退到保守翻译或删除处理。


**Q：这篇论文对个人开发者有什么参考价值？**

A：提供了可复用的过度生成分类思路和混合检测架构设计，即使没有商业级标注数据，也可以基于少量样本训练轻量级分类器来改进翻译质量控制。
