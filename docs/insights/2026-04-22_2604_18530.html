## 💡 核心方法

### 方法概述

OGER框架通过统一的奖励建模视角，将离线教师指导与在线强化学习有机结合。核心思想是利用多教师协作训练构建辅助探索奖励，同时借助长度一致性过滤确保轨迹质量。


### 详细设计

OGER的完整流程包含三个核心阶段：首先是离线轨迹库构建阶段，通过多教师模型生成多样化的推理轨迹，构建高质量的离线数据库；其次是探索奖励建模阶段，训练专门的探索奖励模型（Exploration Reward Model），该模型综合评估轨迹的新颖性和潜在价值；最后是在线RL训练阶段，将验证奖励与探索奖励加权融合，通过PPO等算法更新主模型。在探索奖励的计算中，OGER创新性地结合了离线轨迹的分布信息和模型自身的采样熵，一方面通过对比学习识别低质量区域，另一方面通过熵正则化鼓励模型跳出舒适区。此外，框架引入了长度一致性过滤器（length-consistency filter），用于剔除长度与正确性不匹配的异常轨迹，确保训练信号的可靠性。实验表明，这种混合奖励策略能够有效平衡 exploitation 与 exploration，避免模型陷入局部最优或奖励hack。


### 🔧 关键组件

| 组件 | 说明 |
|------|------|
| 多教师协作轨迹生成器 | 集成多个不同能力的教师模型，通过差异化的采样策略生成多样化的推理轨迹，确保离线数据库覆盖广泛的探索空间。 |
| 探索奖励模型（ERM） | 专门训练的奖励模型，综合评估轨迹新颖度与潜在价值，输入为轨迹序列与模型当前状态的表示，输出为0-1之间的探索奖励分数。 |
| 混合奖励融合器 | 将可验证奖励与探索奖励按可学习权重动态加权，通过自适应调节机制在训练不同阶段自动调整探索-利用平衡。 |
| 长度一致性过滤器 | 基于统计的轨迹质量筛选模块，通过分析序列长度与最终正确性的相关性，剔除异常轨迹保证训练稳定性。 |

## ❓ 常见问题

**Q：OGER与传统RLHF/PPO方法的主要区别是什么？**

A：传统RLHF依赖人类反馈，PPO在RLVR中仅使用验证奖励。OGER的核心创新在于引入专门的探索奖励模型，该模型不仅考虑轨迹的最终正确性，还评估其新颖性和探索价值，从而避免模型陷入局部最优解。


**Q：多教师协作的具体协作机制是怎样的？**

A：多教师模型通过差异化的采样策略（如不同温度、不同的推理模式）生成互补轨迹。这些轨迹汇聚后，通过一致性过滤和质量评估，构建覆盖更广探索空间的离线数据库。每个教师可能擅长不同类型的推理问题，协作后能覆盖更多边界情况。


**Q：探索奖励是否会导致模型生成无意义但新颖的内容？**

A：OGER通过混合奖励机制解决了这一问题。探索奖励与验证奖励加权融合，且权重可自适应调节。在训练初期，探索奖励占比较高以鼓励探索；随着训练进行，系统自动提升验证奖励权重，确保生成内容保持正确性。


**Q：该方法对计算资源的需求如何？**

A：主要计算开销来自三个方面：(1) 多教师模型的轨迹生成；(2) 探索奖励模型的训练；(3) 在线PPO训练。相比标准PPO，额外开销约为30-50%，但通过分布式部署和教师模型共享可以有效控制成本。
