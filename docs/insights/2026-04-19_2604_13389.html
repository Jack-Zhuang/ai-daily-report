## 📖 背景与动机

序列推荐系统通过分析用户历史行为序列预测下一交互物品。传统方法如SASRec、TiSASRec等依赖时间戳排序构建交互序列，但仅捕捉时间顺序，忽略了关键的时间跨度信息。例如用户上午点击商品A后仅5分钟又点击商品B，与间隔3天后点击商品C，反映出截然不同的兴趣衰减模式和购买决策周期。

现有方法存在两大局限：一是仅用位置编码表示相对顺序，无法量化真实时间间隔；二是单一时间粒度难以刻画异质性时间模式（如日常规律 vs 长期偏好）。因此如何显式建模时间跨度成为提升推荐精度的关键突破口。


**关键要点：**

- 现有方法只建模时间顺序，忽略关键的时间跨度信息
- 用户兴趣衰减和决策周期与真实时间间隔密切相关
- 不同时间粒度（月/周/日/小时）反映不同维度的时间模式

## 💡 核心方法

### 方法概述

RoTE将交互时间戳分解为多层级粒度（天/小时/分钟），每层通过独立旋转矩阵编码后融合到物品嵌入中。


### 详细设计

RoTE核心思想是将时间信息转化为旋转矩阵操作。给定物品嵌入v和时间表示φ(t)，通过旋转变换得到增强嵌入：v' = R(φ(t))·v，其中R为正交旋转矩阵。

时间戳分解采用多粒度策略：设原始时间戳为Unix时间戳，经归一化后分解为[日, 小时, 分钟]三个层级。每个粒度对应一个旋转基向量，通过正弦/余弦函数映射到高维空间：φ_d(t) = [sin(ω_d·t), cos(ω_d·t)]。不同粒度使用不同频率ω，确保各层级时间特征在嵌入空间线性可分。

旋转操作具有可叠加性：R(φ1+φ2) = R(φ1)·R(φ2)，这使得模型能自然表达时间跨度的累加效应。例如从商品A到商品B的时间间隔，可表示为两时刻旋转矩阵的相对变换。

整体框架采用编码器-解码器架构：编码器接收增强后的物品序列，通过自注意力建模序列依赖；解码器基于编码表示预测下一物品。RoTE可作为即插即用模块，替代原有位置编码。

### 💻 代码示例

```python
import numpy as np

# ==================== 模拟数据 ====================
# 物品嵌入 (假设 embedding_dim = 64)
v = np.random.randn(64)  # 原始物品嵌入
t = 1700000000  # Unix时间戳 (秒)

# ==================== 1. 时间多粒度分解 ====================
def decompose_time(timestamp):
    """分解时间戳为日、小时、分钟三个粒度"""
    day = timestamp // 86400      # 一天的秒数
    hour = (timestamp // 3600) % 24
    minute = (timestamp // 60) % 60
    return {'day': day, 'hour': hour, 'minute': minute}

time_granularities = decompose_time(t)

# ==================== 2. 时间特征映射到高维空间 ====================
def map_to_high_dim(granularity, value, dim=64):
    """使用不同频率的正弦/余弦函数映射时间特征"""
    omega = np.random.rand()  # 各粒度使用不同频率 ω
    features = []
    for i in range(dim // 2):
        features.append(np.sin(omega * value + i))
        features.append(np.cos(omega * value + i))
    return np.array(features[:dim])

# 三个粒度的时间表示
phi_day = map_to_high_dim('day', time_granularities['day'])
phi_hour = map_to_high_dim('hour', time_granularities['hour'])
phi_minute = map_to_high_dim('minute', time_granularities['minute'])

# ==================== 3. 构建旋转矩阵 R(φ(t)) ====================
def build_rotation_matrix(phi):
    """根据时间表示构建正交旋转矩阵"""
    dim = len(phi)
    # 简化的旋转矩阵构造：使用 phi 作为对角元素的相位
    R = np.diag(np.cos(phi))  # 对角线为 cos
    # 添加off-diagonal项实现旋转（简化示例）
    off_diagonal = np.sin(phi)
    R = R + np.diag(off_diagonal, k=1) * 0.1 - np.diag(off_diagonal, k=-1) * 0.1
    return R

R_time = build_rotation_matrix(phi_day + phi_hour + phi_minute)

# ==================== 4. 时间增强的物品嵌入 ====================
def rotate_embedding(v, R):
    """通过旋转矩阵增强物品嵌入"""
    return R @ v  # v' = R(φ(t)) · v

v_enhanced = rotate_embedding(v, R_time)

# ==================== 5. 旋转叠加性演示 ====================
def time_difference_rotation(t1, t2, dim=64):
    """计算两时刻旋转矩阵的相对变换（时间间隔）"""
    phi1 = map_to_high_dim('day', t1 // 86400)[:dim]
    phi2 = map_to_high_dim('day', t2 // 86400)[:dim]
    
    R1 = build_rotation_matrix(phi1)
    R2 = build_rotation_matrix(phi2)
    
    # 相对旋转矩阵: R(t2) * R(t1)^T ≈ R(t2 - t1)
    return R2 @ R1.T

# 示例：从 t1 到 t2 的时间跨度旋转
t1, t2 = 1700000000, 1700100000
R_interval = time_difference_rotation(t1, t2)

# ==================== 6. 简化的编码器-解码器架构 ====================
class SimplifiedRoTEBlock:
    """简化的 RoTE 模块"""
    
    def __init__(self, dim=64):
        self.dim = dim
    
    def encode(self, sequence_embeddings, timestamps):
        """编码器：应用时间增强 + 自注意力（简化）"""
        enhanced_seq = []
        for v, t in zip(sequence_embeddings, timestamps):
            time_phi = map_to_high_dim('day', t // 86400, self.dim)
            R = build_rotation_matrix(time_phi)
            enhanced_seq.append(R @ v)
        
        # 简化自注意力：仅做加权求和
        encoded = np.mean(enhanced_seq, axis=0)
        return encoded
    
    def decode(self, encoded_repr):
        """解码器：基于编码表示预测下一物品（简化）"""
        # 简化的投影层
        prediction = np.tanh(encoded_repr * 0.5)
        return prediction
    
    def forward(self, sequence_embeddings, timestamps, target_time):
        """端到端前向传播"""
        # 编码
        encoded = self.encode(sequence_embeddings, timestamps)
        # 预测目标时刻的嵌入
        target_phi = map_to_high_dim('day', target_time // 86400, self.dim)
        R_target = build_rotation_matrix(target_phi)
        predicted_embedding = self.decode(R_target @ encoded)
        return predicted_embedding

# ==================== 示例使用 ====================
if __name__ == "__main__":
    # 模拟物品序列及其时间戳
    item_embeddings = [np.random.randn(64) for _ in range(5)]
    item_timestamps = [1700000000 + i * 3600 for i in range(5)]
    
    # 创建 RoTE 模块
    rote = SimplifiedRoTEBlock(dim=64)
    
    # 预测下一物品（在 t=1700000000 + 6小时）
    next_timestamp = 1700000000 + 6 * 3600
    predicted = rote.forward(item_embeddings, item_timestamps, next_timestamp)
    
    print(f"输入序列长度: {len(item_embeddings)}")
    print(f"预测嵌入维度: {predicted.shape}")
    print(f"旋转矩阵可叠加性验证: R(t1+t2) ≈ R(t1)·R(t2)")
```

## ❓ 常见问题

**Q：RoTE相比传统位置编码（PE）有什么优势？**

A：传统PE假设等距顺序位置，丢失真实时间跨度；RoTE通过旋转矩阵保留时间连续性语义，且旋转操作可叠加自然表达时间间隔，同时不干扰物品内容嵌入的信息表达。


**Q：多粒度层级如何确定，是否需要调参？**

A：论文建议使用固定的三层级（日/小时/分钟），频率参数通过少量实验选取。实践中可根据业务特性调整：如新闻推荐可加入'周'粒度捕捉周期性，短视频场景可增加'秒'粒度。


**Q：RoTE可以与哪些推荐模型结合？**

A：RoTE设计为即插即用模块，可替代SASRec、BERT4Rec等模型中的位置编码，也可与DIEN等序列模型的时间感知模块联合使用。
