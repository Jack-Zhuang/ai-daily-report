## 📖 背景与动机

大语言模型（LLM）在处理长文本时面临严峻的计算和内存挑战。自注意力机制的计算复杂度随输入长度呈二次方增长，导致长prompt处理成为实际部署的瓶颈。现有压缩方法主要在Token空间（如删减、合并Token）操作，但忽略了Token嵌入本身的信息冗余。本文的动机源于一个关键观察：连续Token的嵌入向量往往存在高度相似性，可在潜空间进行有损压缩而不显著损失语义信息。通过将压缩操作从Token空间迁移到潜嵌入空间，配合轻量级编码器和LoRA适配器，可在保持模型性能的同时大幅降低推理成本。


**关键要点：**

- 自注意力机制O(n²)复杂度是长文本处理的主要瓶颈
- 现有Token压缩方法局限在离散Token空间，未利用连续嵌入的语义冗余
- 潜空间压缩可捕获比离散操作更丰富的语义信息

## 💡 核心方法

### 方法概述

K-Token Merging在LLM的嵌入层与注意力层之间插入轻量级编码器，将连续K个Token嵌入向量压缩为单一向量，再由LoRA微调的LLM处理压缩后的序列。


### 详细设计

该方法包含三个核心组件：嵌入提取、轻量编码器和LoRA适配器。首先，原始Token序列通过预训练LLM的嵌入层获得向量序列。然后，设计专门的K-Token编码器（K-Encoder）处理每K个连续嵌入：编码器接收K个d维嵌入向量，输出单个d维压缩向量。编码器采用多层感知机（MLP）结构，包含降维层、非线性激活和重构层，实现从R^(K×d)到R^d的非线性映射。为保持原始模型能力，采用LoRA技术对LLM进行适配：冻结预训练权重，仅训练低秩分解矩阵。在预填充（prefill）阶段，输入文本经压缩后长度缩减为原来的1/K，大幅减少后续注意力计算。解码阶段保持原有Token空间输出，确保生成质量。训练时使用压缩重建损失和下游任务损失的联合优化。


### 🔧 关键组件

| 组件 | 说明 |
|------|------|
| K-Token编码器 | 轻量级MLP网络，将K个连续嵌入向量压缩为单一向量。包含输入投影层、降维层和输出层，配合LayerNorm和残差连接保证训练稳定。 |
| LoRA适配层 | 在LLM的注意力权重矩阵引入低秩分解Adapter，仅更新少量参数实现任务适配。α=16，dropout=0.05，保持预训练知识的同时学习压缩感知特征。 |

### 💻 代码示例

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

# ============ 1. K-Token 编码器 (K-Encoder) ============
class KEncoder(nn.Module):
    """
    将 K 个连续的 d 维嵌入压缩为单个 d 维向量
    输入: (batch, K*d) -> MLP -> (batch, d)
    """
    def __init__(self, d_model: int, k: int, hidden_dim: int = 256):
        super().__init__()
        self.k = k
        self.d_model = d_model
        
        # 降维层: R^(K*d) -> R^hidden_dim
        self.down_proj = nn.Linear(k * d_model, hidden_dim)
        
        # 激活层
        self.activation = nn.GELU()
        
        # 重构层: R^hidden_dim -> R^d
        self.up_proj = nn.Linear(hidden_dim, d_model)
    
    def forward(self, x: torch.Tensor) -> torch.Tensor:
        """
        x: (batch, seq_len, k, d) 或 展平后的 (batch, seq_len, k*d)
        """
        batch, seq_len = x.shape[0], x.shape[1]
        
        # 展平K个token的嵌入
        x = x.reshape(batch, seq_len, -1)  # (batch, seq_len, k*d)
        
        # MLP压缩
        x = self.down_proj(x)
        x = self.activation(x)
        x = self.up_proj(x)  # (batch, seq_len, d)
        
        return x


# ============ 2. LoRA 适配器 ============
class LoRALinear(nn.Module):
    """
    低秩适配: W + lora_A @ lora_B
    冻结原始权重，只训练 A 和 B
    """
    def __init__(self, d_in: int, d_out: int, rank: int = 4, alpha: float = 1.0):
        super().__init__()
        self.rank = rank
        self.alpha = alpha
        
        # 冻结原始权重
        self.weight = nn.Parameter(torch.randn(d_out, d_in), requires_grad=False)
        
        # 可训练的低秩矩阵
        self.lora_A = nn.Parameter(torch.randn(rank, d_in) * 0.01)
        self.lora_B = nn.Parameter(torch.zeros(d_out, rank))
    
    def forward(self, x: torch.Tensor) -> torch.Tensor:
        # 原权重输出 + LoRA调整
        base_out = F.linear(x, self.weight)
        lora_out = (self.lora_A.unsqueeze(0) @ self.lora_B.unsqueeze(-1)).squeeze(-1) * self.alpha
        return base_out + lora_out


# ============ 3. 压缩语言模型 (主框架) ============
class CompressedLLM(nn.Module):
    """
    压缩LLM核心结构
    1. 嵌入提取
    2. K-Encoder压缩
    3. 预训练LLM + LoRA适配
    """
    def __init__(self, vocab_size: int, d_model: int, k: int, lora_rank: int = 4):
        super().__init__()
        self.k = k
        self.d_model = d_model
        
        # 组件1: 嵌入层
        self.embedding = nn.Embedding(vocab_size, d_model)
        
        # 组件2: K-Encoder (压缩器)
        self.k_encoder = KEncoder(d_model, k)
        
        # 组件3: 预训练LLM (伪代码，用Linear代替)
        self.llm_backbone = nn.TransformerDecoder(
            nn.TransformerDecoderLayer(d_model, nhead=4), num_layers=2
        )
        
        # 组件4: LoRA适配
        self.lora_layers = nn.ModuleList([
            LoRALinear(d_model, d_model, rank=lora_rank)
            for _ in range(2)  # 假设2层LoRA
        ])
        
        # 输出层
        self.lm_head = nn.Linear(d_model, vocab_size)
    
    def compress_embeddings(self, embeddings: torch.Tensor) -> torch.Tensor:
        """
        嵌入压缩: 每K个token压缩为1个
        输入: (batch, seq_len, d) 
        输出: (batch, seq_len//k, d)
        """
        seq_len = embeddings.shape[1]
        
        # 重塑为 (batch, num_groups, k, d)
        num_groups = seq_len // self.k
        embeddings = embeddings[:, :num_groups * self.k, :]
        reshaped = embeddings.reshape(embeddings.shape[0], num_groups, self.k, self.d_model)
        
        # K-Encoder压缩
        compressed = self.k_encoder(reshaped)  # (batch, num_groups, d)
        
        return compressed
    
    def prefill(self, input_ids: torch.Tensor) -> torch.Tensor:
        """
        Prefill阶段: 嵌入 -> 压缩 -> LLM前向
        """
        # Step 1: 嵌入提取
        embeds = self.embedding(input_ids)  # (batch, seq_len, d)
        
        # Step 2: K-Encoder压缩
        compressed_embeds = self.compress_embeddings(embeds)  # (batch, seq_len/k, d)
        
        # Step 3: LLM前向 (带LoRA)
        memory = self.llm_backbone(compressed_embeds)
        
        return memory
    
    def decode(self, memory: torch.Tensor, lora_idx: int = 0) -> torch.Tensor:
        """
        解码阶段: 保持Token空间输出
        """
        # 应用LoRA适配
        adapted = self.lora_layers[lora_idx](memory)
        
        # 投影到词汇表
        logits = self.lm_head(adapted)
        
        return logits
    
    def forward(self, input_ids: torch.Tensor, labels: torch.Tensor = None):
        """
        完整前向: prefill + decode + loss计算
        """
        # Prefill压缩
        memory = self.prefill(input_ids)
        
        # 解码输出
        logits = self.decode(memory)
        
        if labels is not None:
            # 压缩重建损失 + 下游任务损失
            recon_loss = F.cross_entropy(logits.view(-1, logits.size(-1)), labels.view(-1))
            return recon_loss
        
        return logits


# ============ 4. 训练示例 ============
if __name__ == "__main__":
    # 超参数
    BATCH_SIZE = 2
    SEQ_LEN = 16  # 原始长度
    VOCAB_SIZE = 1000
    D_MODEL = 64
    K = 4  # 每K个token压缩为1个
    
    # 初始化模型
    model = CompressedLLM(vocab_size=VOCAB_SIZE, d_model=D_MODEL, k=K)
    
    # 模拟输入
    input_ids = torch.randint(0, VOCAB_SIZE, (BATCH_SIZE, SEQ_LEN))
    labels = torch.randint(0, VOCAB_SIZE, (BATCH_SIZE, SEQ_LEN // K))
    
    # 前向传播
    compressed_len = SEQ_LEN // K
    print(f"原始序列长度: {SEQ_LEN}")
    print(f"压缩后长度: {compressed_len}")
    print(f"压缩比: {K}x")
    
    logits = model(input_ids)  # Prefill + Decode
    print(f"输出logits形状: {logits.shape}")  # (batch, compressed_len, vocab)
    
    # 计算损失
    loss = model(input_ids, labels)
    print(f"训练损失: {loss.item():.4f}")
```

## ❓ 常见问题

**Q：K值如何选择？越大压缩率越高吗？**

A：K值确实决定压缩比（压缩率=1-1/K），但需权衡性能损失。建议从K=2开始测试，逐步增大至性能可接受的上限。不同任务的最优K值差异显著，需根据下游任务特点调优。


**Q：与现有Token删减方法相比有何优势？**

A：传统方法在离散Token空间操作（如删除或合并原文Token），损失不可恢复。K-Token Merging在连续嵌入空间学习最优压缩表示，可捕获Token间的非线性语义组合，在相同压缩比下性能更优。


**Q：能否用于已部署的LLM？**

A：可以。由于采用LoRA适配方案，只需加载冻结的预训练权重并新增轻量模块，训练和部署成本低。Encoder和LoRA参数规模远小于完整模型权重。
