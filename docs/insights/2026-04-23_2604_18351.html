利用用户-物品交互的协同信号进行聚类，让相似用户共享嵌入向量，实现高效压缩

📌 TL;DR

  • 一句话总结：通过平衡协同聚类让相似用户/物品共享嵌入，大幅压缩参数量同时保持推荐精度。
  • 核心贡献：提出BACO框架，创新性地将用户-物品协同聚类与嵌入压缩结合，在codebook中实现嵌入共享，兼顾压缩率与精度。
  • 实用价值：可直接应用于工业级推荐系统，显著降低GPU显存占用和推理延迟，助力边缘设备部署。

📖 背景与动机

现代推荐系统依赖深度学习模型将用户和物品映射为稠密嵌入向量。工业场景下，用户/物品规模动辄达数亿级，对应的嵌入表参数量极其庞大。例如，1亿用户×128维浮点嵌入需约51GB显存，这给训练和推理带来沉重的计算与存储负担。现有压缩方案存在两难：ID哈希虽快但精度损失严重；学习量化虽保精度但引入额外计算开销。因此，亟需一种高效且保真的嵌入压缩技术，让相似用户/物品共享嵌入，从根本上减少参数量。

关键要点：
  • 工业推荐系统嵌入表参数量可达数十GB，部署成本高昂
  • 传统ID哈希方法精度损失大，无法利用用户相似性
  • 现有量化/哈希方案存在精度-效率权衡，难以兼顾

💡 核心方法

方法概述

BACO核心思想是：先利用用户-物品交互行为构建协同关系图，再对用户和物品同时进行平衡聚类，使得聚类内用户共享同一组基础嵌入向量（codebook），从而大幅压缩参数量。

详细设计

BACO采用协同聚类框架，同时对用户和物品进行分组。具体流程如下：首先，基于用户-物品交互矩阵构建二部图，其中边权重反映交互强度（如点击、购买）。接着，设计平衡聚类目标函数，核心包含三个部分：① 聚类内连接性最大化——同一簇内用户-物品交互应更紧密；② 聚类间连接性最小化——跨簇交互应尽量稀疏；③ 平衡约束——保证各簇大小均匀，避免极端分配。在此目标下，采用交替优化策略：固定聚类分配学习最优codebook，再固定codebook更新聚类标签，直至收敛。最终，每个用户/物品被映射到所属簇，共享该簇对应的嵌入向量。推理时，只需查表+聚类嵌入复用，额外计算几乎为零。

def __init__(self, n_user_clusters, n_item_clusters, max_iter=50):

self.n_user_clusters = n_user_clusters

self.n_item_clusters = n_item_clusters

self.max_iter = max_iter

def fit(self, interaction_matrix):

"""

训练过程：交替优化

- interaction_matrix: 用户-物品交互矩阵 (n_users x n_items)

"""

n_users, n_items = interaction_matrix.shape

# Step 1: 初始化聚类分配（随机）

user_clusters = np.random.randint(0, self.n_user_clusters, n_users)

item_clusters = np.random.randint(0, self.n_item_clusters, n_items)

# Step 2: 构建二部图邻接矩阵（边权重 = 交互强度）

adjacency = interaction_matrix.copy()

for iteration in range(self.max_iter):

# ---- 交替优化 Phase 1: 固定聚类，更新codebook ----

codebook = self._update_codebook(adjacency, user_clusters, item_clusters)

# ---- 交替优化 Phase 2: 固定codebook，更新聚类标签 ----

user_clusters = self._update_user_clusters(adjacency, codebook, item_clusters)

item_clusters = self._update_item_clusters(adjacency, codebook, user_clusters)

# ---- 计算并监控目标函数 ----

loss = self._compute_objective(adjacency, user_clusters, item_clusters, codebook)

print(f"Iter {iteration}: Loss = {loss:.4f}")

# 保存最终结果

self.user_clusters = user_clusters

self.item_clusters = item_clusters

self.codebook = codebook

return self

def _update_codebook(self, adjacency, user_clusters, item_clusters):

"""

核心目标函数①: 聚类内连接性最大化

学习最优codebook使得簇内重建误差最小

"""

# 伪代码：计算每个(用户簇, 物品簇)对的平均交互强度

codebook = np.zeros((self.n_user_clusters, self.n_item_clusters))

for u_c in range(self.n_user_clusters):

for i_c in range(self.n_item_clusters):

# 获取该簇内的所有交互

mask = (user_clusters == u_c)[:, None] & (item_clusters == i_c)[None, :]

cluster_interactions = adjacency[mask]

# codebook值为该簇的平均交互强度

codebook[u_c, i_c] = np.mean(cluster_interactions) if len(cluster_interactions) > 0 else 0

return codebook

def _update_user_clusters(self, adjacency, codebook, item_clusters):

"""

核心目标函数②: 聚类间连接性最小化

核心目标函数③: 平衡约束（簇大小均匀）

"""

n_users = adjacency.shape[0]

user_clusters = np.zeros(n_users, dtype=int)

for u in range(n_users):

# 计算分配到每个簇的评分（含平衡惩罚）

scores = []

for u_c in range(self.n_user_clusters):

# 重构误差（交互强度与codebook的差异）

reconstruction_error = 0

for i in range(adjacency.shape[1]):

pred = codebook[u_c, item_clusters[i]]

reconstruction_error += (adjacency[u, i] - pred) ** 2

# 平衡惩罚：使簇大小尽量均匀

balance_penalty = abs(np.sum(self.user_clusters == u_c) - n_users / self.n_user_clusters)

scores.append(reconstruction_error + 0.1 * balance_penalty)

# 选择评分最低的簇

user_clusters[u] = np.argmin(scores)

return user_clusters

def _update_item_clusters(self, adjacency, codebook, user_clusters):

"""类似用户聚类更新"""

n_items = adjacency.shape[1]

item_clusters = np.zeros(n_items, dtype=int)

for i in range(n_items):

scores = []

for i_c in range(self.n_item_clusters):

reconstruction_error = sum(

(adjacency[u, i] - codebook[user_clusters[u], i_c]) ** 2

for u in range(adjacency.shape[0])

)

balance_penalty = abs(np.sum(self.item_clusters == i_c) - n_items / self.n_item_clusters)

scores.append(reconstruction_error + 0.1 * balance_penalty)

item_clusters[i] = np.argmin(scores)

return item_clusters

def _compute_objective(self, adjacency, user_clusters, item_clusters, codebook):

"""计算总目标函数值"""

# ① 聚类内连接性（重建误差）

reconstruction = np.sum(

(adjacency[u, i] - codebook[user_clusters[u], item_clusters[i]]) ** 2

for u in range(adjacency.shape[0])

for i in range(adjacency.shape[1])

)

return reconstruction

def predict(self, user_ids, item_ids):

"""推理：查表获取嵌入"""

return np.array([

self.codebook[self.user_clusters[u], self.item_clusters[i]]

for u, i in zip(user_ids, item_ids)

])

❓ 常见问题

Q：BACO与传统ID哈希的本质区别是什么？

A：ID哈希随机映射，不利用任何用户相似性信息，冲突严重时精度暴跌。BACO则通过协同聚类让行为相似的用户共享嵌入，天然保留协同过滤信号，压缩更智能。

Q：聚类数量W如何选择？

A：W过小会导致过度聚合丢失个性化，过大则压缩率低。论文建议通过验证集调优，通常取用户数的1%~5%作为起点，观察Recall@20-W曲线的拐点。

Q：BACO适用于冷启动场景吗？

A：相比基线方法，BACO对冷启动用户更友好。新用户可基于人口统计或上下文快速匹配到相似聚类，获得较合理的初始嵌入，而非从零学起。

Q：线上如何部署BACO压缩后的模型？

A：只需额外存储聚类标签映射表（用户ID→簇ID），推理时先查映射表获取簇ID，再读取对应簇的codebook嵌入，替换原嵌入表查找。延迟增加<1ms。
