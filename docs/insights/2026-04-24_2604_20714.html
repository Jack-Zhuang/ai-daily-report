📖 背景与动机

多智能体系统（MAS）正在成为复杂AI应用的主流架构。然而，当前多智能体系统的设计与优化面临严峻挑战：开发者在「Agent Engineering」过程中需要手动编排大量智能体、工具和工作流，这不仅是劳动密集型工作，更需要对系统交互有深入理解。现有自动优化方法主要局限于「flat prompt tuning」——即简单调整单个智能体的提示词，这种方式缺乏对系统整体结构的感知能力。更关键的是，这些优化器是「静态的」：它们无法从历史经验中学习并改进自身的优化策略，就像一个不会成长的学生，每次考试都是从头开始。本文提出的TPGO框架正是为了解决这两个核心问题：结构化感知缺失和优化器无法自我进化。

关键要点：
  • 多智能体系统设计高度复杂，需要协调多个智能体、工具和数据流
  • 现有flat prompt tuning方法缺乏对系统结构的感知能力
  • 传统优化器是静态的，无法从历史经验中学习和改进
  • 急需一种能同时理解系统结构并具备自我进化能力的优化框架

💡 核心方法

方法概述

TPGO框架通过三阶段循环实现多智能体系统的自我进化：图结构构建、梯度驱动演化、群组相对优化。核心创新在于将系统建模为可优化的文本参数图，并通过文本梯度指导结构化修改。

详细设计

TPGO框架首先将多智能体系统建模为Textual Parameter Graph（TPG）。在TPG中，每个智能体、工具和工作流都被表示为可独立优化的模块化节点，节点之间通过语义边连接，形成完整的系统拓扑结构。这种建模方式保留了系统的结构化信息，为后续的结构化优化奠定基础。

第二阶段是梯度驱动演化。传统深度学习通过数值梯度更新参数，TPGO则推导出「文本梯度」——一种结构化的自然语言反馈。这种文本梯度从执行轨迹中提取，不仅能定位失败点，还能以自然语言形式给出具体的修改建议。例如，文本梯度可能输出：「智能体A与智能体B之间的协作效率低下，建议增加中间协调节点」。这种可解释的反馈使得优化过程完全透明。

第三阶段是Group Relative Agent Optimization（GRAO），这是框架的核心元学习策略。GRAO不追求单一最优解，而是维护一个智能体群体的多样化策略池。每次优化时，系统会比较不同策略组的表现差异，学习「哪些调整策略更有效」。通过相对比较学习，框架逐渐掌握优化的「方法论」——如何根据任务特点选择合适的调整策略，而非仅记忆特定任务的解决方案。这种设计确保了优化的泛化能力和稳定性。

| Textual Parameter Graph (TPG) | 将多智能体系统表示为模块化图结构，节点包括Agent、Tool、Workflow，支持结构级和参数级的细粒度优化。 |

| Group Relative Agent Optimization (GRAO) | 元学习策略，通过维护策略群体并学习组间相对表现，掌握「如何选择优化策略」的方法论，而非仅记忆特定解。 |

| Evolution Controller | 控制进化过程，包括收敛判定、探索-利用平衡、异常处理等，确保优化过程稳定可靠。 |

❓ 常见问题

Q：TPGO与传统的Prompt Tuning有什么本质区别？

A：传统Prompt Tuning仅调整单个智能体的提示词，忽视了系统整体结构。TPGO将多智能体系统建模为完整的参数图，不仅能优化单个节点，还能调整节点间的连接关系和工作流。更重要的是，TPGO的优化器本身会通过GRAO持续改进，而传统方法是静态的。

Q：文本梯度是否总是可靠的？

A：文本梯度由LLM生成，存在一定的幻觉风险。但TPGO通过结构化约束（强制输出特定格式的修改建议）和多轮验证（先小范围测试修改效果）来降低风险。此外，框架提供了回滚机制，确保错误的优化尝试不会破坏系统稳定性。

Q：GRAO的「学习如何学习」具体是怎么实现的？

A：GRAO维护一个策略群体（每个策略定义一组优化规则）。在每轮优化中，系统不仅更新TPG配置，还记录「哪种策略组合产生了更好的效果」。通过对比学习，群体逐渐学会识别不同任务特征对应的有效策略，最终形成一套可泛化的优化方法论。

Q：这套框架在实际部署中最大的挑战是什么？

A：最大的挑战是可观测性。生产环境中的多智能体系统往往涉及异步调用、海量日志和分布式执行，获取清晰的结构化执行轨迹本身就很难。建议从相对简单的同步场景入手，积累经验后再扩展到复杂环境。
