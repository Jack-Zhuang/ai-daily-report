## 📖 背景与动机

闭环仿真（Closed-loop Simulation）是自动驾驶开发的核心环节，能在实车部署前实现规模化测试与安全验证。传统方法依赖人工建模或实验室采集，成本高昂且难以覆盖长尾场景。神经场景重建虽能将驾驶日志转化为交互式3D环境，但输出结果通常是整体场景表示，缺乏独立的3D物体资产，无法满足agent manipulation（智能体操控）和大视角新视角合成的需求。自动驾驶场景的特殊性在于：物体观测稀疏（单次经过）、视角受限（主要观察车辆前后）、传感器异构（Camera/LiDAR/Radar）、遮挡严重。如何从这种in-the-wild数据中提取出完整的、仿真就绪的3D资产，成为打通数据闭环的关键瓶颈。


**关键要点：**

- 闭环仿真是AV安全部署的必要条件，但依赖大量3D场景资产
- 现有神经重建方法产出整体场景，无法提供独立可操控的3D物体资产
- 真实驾驶日志中物体观测稀疏且视角受限，给资产提取带来独特挑战

## 💡 核心方法

### 方法概述

Asset Harvester是端到端图像转3D系统，处理真实AV日志（NCore格式），通过几何感知预处理、稀疏视图条件多视图生成、3D Gaussian提升三个阶段，将稀疏观测转换为完整仿真资产。


### 详细设计

系统首先从AV日志中裁剪并校正物体区域，通过异构传感器的几何对齐实现感知预处理，确保多视角图像的精确配准。核心模块SparseViewDiT采用Diffusion Transformer架构，接收稀疏输入视图（1-3张），条件生成多视角一致的新视图。区别于传统单目方法，SparseViewDiT显式建模视角间几何关系，通过cross-attention机制在视图间传递信息。在条件信息处理上，系统将binary indicator mask与latent feature maps通道级拼接，引导生成模型关注前景物体区域。生成的稀疏视图随后通过3D Gaussian lifting模块提升至3D空间：该模块将2D特征反投影并聚类为3D Gaussian primitives，每个primitive由中心μ∈R³、协方差Σ∈R³ˣ³（实践中用scale+rotation参数化）定义。训练时要求从各相机渲染预测的Gaussians与对应图像匹配，使用ℓ1/SSIM/perceptual等损失函数联合优化几何与外观。最终输出的3D Gaussians可直接用于实时渲染，支持大视角新视图合成和物理级交互。


### 🔧 关键组件

| 组件 | 说明 |
|------|------|
| SparseViewDiT | 稀疏视图条件的多视图扩散生成器，使用Diffusion Transformer架构，通过条件视图+mask引导生成目标视角图像，引入cross-attention建模视角间几何一致性 |
| 3D Gaussian Lifting | 将2D多视图特征提升至3D空间的核心模块，通过可微渲染将Gaussians投影回2D并与原图匹配优化，输出带语义属性的3D Gaussian primitives |
| 几何感知预处理 | 处理异构传感器（Camera/LiDAR）的几何对齐，实现跨模态配准，确保输入视图精确对应同一时刻的物体状态 |
| NCore数据接口 | 与Industry-level AV日志格式NCore对接，支持大规模真实场景数据的自动化接入 |

import torch
import torch.nn as nn

# ========== 1. 感知预处理模块 ==========
class PerceptionPreprocessor(nn.Module):
    """异构传感器几何对齐，实现多视角图像精确配准"""
    def __init__(self):
        super().__init__()
    
    def forward(self, av_logs):
        # 裁剪并校正物体区域
        cropped_regions = self.crop_and_rectify(av_logs)
        # 几何对齐配准
        aligned_views = self.geometric_alignment(cropped_regions)
        return aligned_views

# ========== 2. SparseViewDiT 核心模块 ==========
class SparseViewDiT(nn.Module):
    """Diffusion Transformer架构，稀疏视图条件生成"""
    def __init__(self, num_input_views=3):
        super().__init__()
        self.num_input_views = num_input_views
        # Diffusion Transformer blocks
        self.dit_blocks = nn.ModuleList([
            DiTBlock() for _ in range(12)
        ])
        # Cross-attention for view-to-view information transfer
        self.cross_attention = CrossAttentionLayer()
    
    def forward(self, sparse_views, binary_masks, latent_features, timestep):
        # 通道级拼接: mask引导前景区域
        conditioned_features = torch.cat([latent_features, binary_masks], dim=1)
        
        # Diffusion过程
        noise = self.add_noise(conditioned_features, timestep)
        
        # 逐层Transformer处理
        for block in self.dit_blocks:
            # Self-attention within view
            x = block.self_attention(noise)
            # Cross-attention between views (显式建模视角间几何关系)
            x = self.cross_attention(x, sparse_views)
            # Feed-forward
            x = block.ffn(x)
        
        return x
    
    def add_noise(self, x, t):
        # 简化的噪声添加
        noise = torch.randn_like(x) * t.unsqueeze(-1)
        return x + noise

# ========== 3. 3D Gaussian Lifting 模块 ==========
class GaussianLifting(nn.Module):
    """将2D特征反投影为3D Gaussian primitives"""
    def __init__(self):
        super().__init__()
        self.feature_projection = nn.Linear(512, 256)
        self.gaussian_predictor = GaussianPredictor()
    
    def forward(self, lifted_features, camera_params):
        """
        返回: 3D Gaussians (μ, scale, rotation, opacity, sh)
        """
        # 2D特征反投影
        projected_3d = self.project_to_3d(lifted_features, camera_params)
        
        # 聚类为Gaussian primitives
        gaussians = self.gaussian_predictor(projected_3d)
        
        # 参数化: 中心μ, 协方差Σ(用scale+rotation表示)
        gaussian_struct = {
            'means': gaussians[:, :, :3],           # μ ∈ R³
            'scales': gaussians[:, :, 3:6],         # scale
            'rotations': gaussians[:, :, 6:10],     # quaternion
            'opacity': gaussians[:, :, 10],          # 不透明度
            'features': gaussians[:, :, 11:]         # 外观特征
        }
        return gaussian_struct

# ========== 4. 可微渲染器 ==========
class DifferentiableRenderer(nn.Module):
    """3D Gaussians到2D图像的可微渲染"""
    def __init__(self):
        super().__init__()
    
    def render(self, gaussians, cameras):
        rendered_images = []
        for cam_idx, cam in enumerate(cameras):
            # 将Gaussians投影到相机平面
            projected = self.project_gaussians(gaussians, cam)
            # Alpha混合渲染
            image = self.alpha_compositing(projected, gaussians['opacity'])
            rendered_images.append(image)
        return torch.stack(rendered_images)

# ========== 5. 完整pipeline ==========
class MultiViewSynthesisSystem(nn.Module):
    """端到端多视角新视图合成系统"""
    def __init__(self):
        super().__init__()
        self.preprocessor = PerceptionPreprocessor()
        self.sparse_view_dit = SparseViewDiT(num_input_views=3)
        self.gaussian_lifting = GaussianLifting()
        self.renderer = DifferentiableRenderer()
    
    def forward(self, av_logs, input_views, camera_params, timestep=None):
        # Step 1: 感知预处理与配准
        aligned_views = self.preprocessor(av_logs)
        
        # Step 2: SparseViewDiT生成新视图
        generated_views = self.sparse_view_dit(
            sparse_views=input_views,
            binary_masks=aligned_views['masks'],
            latent_features=aligned_views['features'],
            timestep=timestep
        )
        
        # Step 3: 3D Gaussian lifting
        gaussians = self.gaussian_lifting(generated_views, camera_params)
        
        # Step 4: 渲染
        rendered_images = self.renderer(gaussians, camera_params)
        
        return rendered_images, gaussians

# ========== 6. 训练损失函数 ==========
class MultiViewLoss(nn.Module):
    """联合优化几何与外观的损失函数"""
    def __init__(self):
        super().__init__()
        self.l1_loss = nn.L1Loss()
        self.ssim_loss = SSIMLoss()
        self.perceptual_loss = PerceptualLoss()
    
    def forward(self, rendered, target):
        # ℓ1 loss
        loss_l1 = self.l1_loss(rendered, target)
        # SSIM loss
        loss_ssim = 1 - self.ssim_loss(rendered, target)
        # Perceptual loss
        loss_percep = self.perceptual_loss(rendered, target)
        
        total_loss = loss_l1 + 0.5 * loss_ssim + 0.1 * loss_percep
        return total_loss

## ❓ 常见问题

**Q：为什么传统单目3D重建方法不适用于自动驾驶场景？**

A：单目方法依赖单张图像推断3D结构，缺乏多视角几何约束，在稀疏观测下容易产生几何歧义。自动驾驶场景中物体只有1-3张有效视图，且视角受限（正面/侧面为主），单目方法难以保证多视角一致性，而Asset Harvester的SparseViewDiT通过多视图生成扩展观测视角，再用Gaussians进行3D提升，实现了更好的几何完整性。


**Q：SparseViewDiT与传统NeRF/3DGS方法有何区别？**

A：NeRF/3DGS需要多视角图像直接作为输入进行体渲染优化，而SparseViewDiT处理的是极端稀疏视角（1-3张）的in-the-wild数据，通过扩散模型先生成多视角一致的新视图作为'隐式3D表示'，再提升到显式Gaussians。这是两阶段方法，更适合观测受限的真实场景。


**Q：该方法对传感器配置有要求吗？**

A：系统设计支持异构传感器融合，但核心依赖多视角相机图像。LiDAR提供了额外的深度先验，有助于预处理阶段的物体区域裁剪和深度校正。对于纯相机系统，只要有多视角覆盖即可工作，但重建质量可能下降。


**Q：生成的3D资产在仿真器中如何使用？**

A：输出的3D Gaussians资产可直接用于支持Gaussian渲染的仿真器（如NVIDIA Omniverse平台），或转换为标准3D格式（mesh/point cloud）导入传统仿真器。由于保留了语义属性，可用于场景编辑、物体替换、物理交互等操作。
