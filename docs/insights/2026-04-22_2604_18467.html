## 📖 背景与动机

## 💡 核心方法

### 方法概述

框架由预测模块ConGA-PePPI和生成模块TC-PepGen组成，前者完成交互预测与结合位点定位，后者实现目标条件下的肽序列生成，两模块共享对蛋白质靶标的信息理解。


### 详细设计

ConGA-PePPI采用非对称编码策略区别处理肽和蛋白序列，通过双向交叉注意力建立两者的交互表示，再结合渐进式迁移学习从配对预测任务迁移到结合位点定位任务。TC-PepGen在自回归解码过程中引入层级条件化机制：每层解码器都接收蛋白质靶标的特征作为额外输入，确保生成过程始终以目标蛋白信息为导向。生成时，模型接收蛋白质口袋序列或结构特征作为条件输入，自回归生成符合长度要求的肽序列。训练采用对比学习和重建损失的混合目标函数。


### 🔧 关键组件

| 组件 | 说明 |
|------|------|
| 非对称编码器 | 肽和蛋白采用不同深度的Transformer编码器，蛋白侧6层、肽侧3层，捕捉序列长度差异和结构复杂度差异 |
| 双向交叉注意力 | 分别以肽和蛋白为Query进行双向交叉注意力，生成互补的交互表示，提升交互预测精度 |
| 渐进迁移模块 | 先训练配对预测任务，再冻结部分层继续训练结合位点定位任务，实现知识迁移 |
| 层级条件化解码 | TC-PepGen每层解码都注入目标蛋白特征，确保生成过程始终受目标引导 |

import torch
import torch.nn as nn

# ============================================================
# 1. 非对称编码器 - 区别处理肽和蛋白序列
# ============================================================
class AsymmetricEncoder(nn.Module):
    """非对称编码：肽和蛋白使用不同的编码策略"""
    def __init__(self):
        super().__init__()
        # 蛋白编码器 - 更宽，感受野更大
        self.protein_encoder = nn.TransformerEncoder(
            nn.TransformerEncoderLayer(d_model=512, nhead=8),
            num_layers=6
        )
        # 肽编码器 - 更深，捕捉细微模式
        self.peptide_encoder = nn.TransformerEncoder(
            nn.TransformerEncoderLayer(d_model=256, nhead=4),
            num_layers=4
        )
    
    def forward(self, protein_seq, peptide_seq):
        # 蛋白序列编码
        protein_feat = self.protein_encoder(protein_seq)
        # 肽序列编码
        peptide_feat = self.peptide_encoder(peptide_seq)
        return protein_feat, peptide_feat


# ============================================================
# 2. 双向交叉注意力 - 建立交互表示
# ============================================================
class BidirectionalCrossAttention(nn.Module):
    """双向交叉注意力：蛋白->肽 和 肽->蛋白 双向交互"""
    def __init__(self, d_model=512):
        super().__init__()
        self.cross_attn_p2p = nn.MultiheadAttention(d_model, num_heads=8)
        self.cross_attn_t2p = nn.MultiheadAttention(d_model, num_heads=8)
    
    def forward(self, protein_feat, peptide_feat):
        # 蛋白到肽的注意力
        p2p_attn, _ = self.cross_attn_p2p(peptide_feat, protein_feat, protein_feat)
        # 肽到蛋白的注意力
        t2p_attn, _ = self.cross_attn_t2p(protein_feat, peptide_feat, peptide_feat)
        # 融合交互表示
        interaction = p2p_attn + t2p_attn
        return interaction


# ============================================================
# 3. 层级条件化自回归解码器 (TC-PepGen)
# ============================================================
class HierarchicalConditionedDecoder(nn.Module):
    """层级条件化：每层都接收蛋白特征作为额外输入"""
    def __init__(self, vocab_size=20, d_model=256):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, d_model)
        self.layers = nn.ModuleList([
            nn.TransformerDecoderLayer(d_model=d_model, nhead=4)
            for _ in range(4)
        ])
        # 蛋白条件投影层（每层都有）
        self.condition_projections = nn.ModuleList([
            nn.Linear(512, d_model) for _ in range(4)
        ])
    
    def forward(self, partial_peptide, protein_feat, layer_idx):
        """层级条件化解码"""
        # 词嵌入
        embedded = self.embedding(partial_peptide)
        # 获取当前层的蛋白条件
        protein_condition = self.condition_projections[layer_idx](protein_feat)
        # 融入条件
        embedded = embedded + protein_condition
        # 解码
        output = embedded
        for i in range(layer_idx + 1):
            output = self.layers[i](output, protein_feat)
        return output


# ============================================================
# 4. 渐进式迁移学习
# ============================================================
class ProgressiveTransfer:
    """从配对预测 -> 结合位点定位的渐进迁移"""
    def __init__(self, model):
        self.model = model
        self.stages = ['pair_prediction', 'binding_site_localization']
    
    def adapt_model(self, current_stage):
        """渐进冻结/解冻层"""
        if current_stage == 'pair_prediction':
            # 阶段1：冻结解码器，只训练编码器和注意力
            for param in self.model.decoder.parameters():
                param.requires_grad = False
        elif current_stage == 'binding_site_localization':
            # 阶段2：解冻解码器，微调
            for param in self.model.decoder.parameters():
                param.requires_grad = True


# ============================================================
# 5. 混合损失函数
# ============================================================
class MixedLoss(nn.Module):
    """对比学习 + 重建损失的混合目标"""
    def __init__(self):
        super().__init__()
        self.reconstruction_loss = nn.CrossEntropyLoss()
        self.contrastive_loss = nn.TripletMarginLoss()
    
    def forward(self, pred_seq, target_seq, anchor, positive, negative):
        # 重建损失
        recon_loss = self.reconstruction_loss(pred_seq, target_seq)
        # 对比损失（拉近正样本，推远负样本）
        contrast_loss = self.contrastive_loss(anchor, positive, negative)
        # 加权组合
        total_loss = 0.7 * recon_loss + 0.3 * contrast_loss
        return total_loss, recon_loss, contrast_loss


# ============================================================
# 6. 完整模型整合
# ============================================================
class TC_PepGen_Model(nn.Module):
    """TC-PepGen 完整模型"""
    def __init__(self):
        super().__init__()
        self.asym_encoder = AsymmetricEncoder()
        self.cross_attention = BidirectionalCrossAttention()
        self.decoder = HierarchicalConditionedDecoder()
        self.criterion = MixedLoss()
    
    def generate(self, protein_feat, max_len=20):
        """自回归生成肽序列"""
        generated = [2]  # [SOS] token
        for layer_idx in range(4):
            for step in range(max_len // 4):
                input_seq = torch.tensor([generated])
                output = self.decoder(input_seq, protein_feat, layer_idx)
                # 采样下一token（简化）
                next_token = output.argmax(dim=-1)[:, -1]
                generated.append(next_token.item())
                if next_token == 3:  # [EOS]
                    break
        return generated[1:]  # 去掉SOS

## ❓ 常见问题

**Q：为什么采用非对称编码而非对称编码？**

A：蛋白质序列通常比肽序列长数倍至数十倍，结构复杂度也更高。非对称设计让蛋白侧使用更深的网络捕捉复杂特征，肽侧用轻量网络避免过拟合，匹配两者的生物学差异。


**Q：层级条件化与简单的序列拼接条件化有何区别？**

A：简单拼接只在解码初期注入一次条件信息，后续生成可能偏离目标；层级条件化在每层解码器都重新注入目标特征，确保生成全程受到靶点引导，生成结果的靶向性更强。


**Q：生成的肽如何验证质量？**

A：论文使用蛋白质对接预测和结合亲和力评估工具验证生成肽与靶点的结合模式，并通过湿实验（如AlphaFold预测、表面等离子共振）进一步验证。
