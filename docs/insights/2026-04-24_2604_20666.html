📖 背景与动机

检索增强生成（RAG）在跨语言场景面临独特挑战：传统多语言模型（如mBERT、XLM-R）将参数分散至数十种语言，导致在特定语言对上性能受限。希腊语因其复杂的词形变化系统（一个词根可产生数十种形态变体）和独特的领域术语（如古希腊哲学、医学术语），对现有模型构成特殊挑战。本研究源于实际业务需求——为希腊-英语双语应用构建高效RAG系统，需要模型同时具备语义理解和跨语言对齐能力。现有通用多语言模型难以胜任这一专业化任务。

关键要点：
  • RAG系统在跨语言场景需要处理语义匹配和语言转换的双重挑战
  • 希腊语词形变化复杂，形态学特征显著区别于英语
  • 通用多语言模型容量分散导致特定语言对优化不足
  • 双语文档检索在法律、医疗、旅游等领域存在强烈需求

💡 核心方法

方法概述

ORPHEAS采用两阶段训练范式：首先利用知识图谱构建高质量双语训练数据，再通过多域语料微调实现跨语言语义对齐。

详细设计

方法包含三个核心模块：知识图谱采样器、形态增强编码器、对比学习对齐器。知识图谱采样器从多领域知识图谱（如Wikidata希腊-英语实体对）中提取高质量双语信号，生成包含词级对齐和句级语义的训练样本。形态增强编码器针对希腊语形态复杂性设计了专门的子词切分策略，将形态丰富的希腊词分解为语义一致的子单元。对比学习对齐器通过InfoNCE损失函数，拉近相同语义在不同语言表示空间中的距离，同时推开语义无关的表示。训练过程中采用课程学习策略，先在通用语料建立基础对齐，再在专业领域语料（如法律、医学）精调，确保模型同时具备通用语义理解和领域适配能力。

========== 3. 对比学习对齐器 (Contrastive Alignment) ==========

class ContrastiveAligner(nn.Module):

"""基于InfoNCE的跨语言对比学习对齐"""

def __init__(self, embed_dim, temperature=0.1):

super().__init__()

self.el_encoder = MorphologyEncoder(vocab_size=50000, embed_dim=embed_dim)

self.en_encoder = MorphologyEncoder(vocab_size=50000, embed_dim=embed_dim)

self.temperature = temperature

def forward(self, el_tokens, en_tokens, apply_morph=True):

"""

计算对比学习损失，拉近语义相近表示

"""

# 编码两种语言

el_emb = self.el_encoder(el_tokens, apply_morpheme_seg=apply_morph)

en_emb = self.en_encoder(en_tokens, apply_morpheme_seg=False)

# 归一化

el_emb = nn.functional.normalize(el_emb, dim=1)

en_emb = nn.functional.normalize(en_emb, dim=1)

# 计算InfoNCE损失

loss = self._info_nce_loss(el_emb, en_emb)

return loss, el_emb, en_emb

def _info_nce_loss(self, z_i, z_j):

"""

InfoNCE损失: 拉近正例，推开负例

"""

batch_size = z_i.shape[0]

# 计算相似度矩阵

sim_matrix = torch.matmul(z_i, z_j.T) / self.temperature

# 对角线为正例，其余为负例

labels = torch.arange(batch_size, device=z_i.device)

loss = nn.CrossEntropyLoss()(sim_matrix, labels)

return loss

❓ 常见问题

Q：为什么专用双语模型比通用多语言模型效果好？

A：多语言模型需将有限参数分配给所有支持语言，导致每个语言对的表示容量受限。专用模型将全部参数用于优化单一语言对，可针对目标语言特征（如希腊语形态复杂性）进行定制优化。

Q：ORPHEAS如何处理希腊语的形态复杂性？

A：通过形态增强编码器实现，核心是将复杂形态词分解为语义一致的子词单元，并在编码过程中注入形态特征，使模型能理解door、doors、dooring等形态变体的语义关联。

Q：知识图谱在训练中扮演什么角色？

A：知识图谱提供高质量的双语对齐信号（如Wikidata中的希腊-英语实体对），解决了高质量平行语料稀缺的问题，同时确保跨语言对齐的准确性。

Q：该模型如何用于实际的RAG系统？

A：在RAG系统中，ORPHEAS用于将文档库和用户查询编码为向量表示。通过向量相似度检索找到最相关的文档片段，再输入大语言模型生成答案，支持跨语言检索场景。
