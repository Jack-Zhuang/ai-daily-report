📖 背景与动机

大语言模型（LLMs）在推荐系统领域展现出强大潜力，但直接将LLM部署到工业推荐系统面临严峻挑战：LLM输出的高维表示（如768-4096维）会大幅增加存储和计算负担。以拥有百万用户的平台为例，仅存储用户表示就需要TB级内存，加上实时推理的计算开销，成本难以承受。

当前主流做法是离线生成并缓存LLM增强表示，但这一策略存在根本缺陷：首先，压缩通常针对最终层进行，忽视了不同层级表示的语义差异；其次，最终层在预训练任务（如next-token prediction）上专门化，可能丢失对推荐任务有益的信息。这些问题促使研究者重新审视表示压缩的层级选择问题。

关键要点：
  • LLM高维表示导致存储和计算成本呈指数增长
  • 现有压缩方法默认压缩最终层，忽视层级差异
  • 实验发现中间层表示在推荐任务上反而更优

💡 核心方法

方法概述

提出模块化表示压缩（MRC）框架，通过识别LLM中的功能模块，针对中间层进行选择性压缩，避免最终层的有害压缩。

详细设计

方法包含三个核心步骤：

  • 层级分析阶段：对LLM各层表示进行任务相关性分析，计算每层表示与推荐目标的相关性分数。通过探针网络（probe network）评估各层表示的下游性能，确定最优压缩层。
  • 模块化分解：基于模块化理论，将LLM表示空间分解为任务相关模块和任务无关模块。中间层包含更多可迁移的通用语义模块，而最终层专门化于预训练代理任务（如语言建模）。
  • 自适应压缩网络：设计轻量级压缩网络，包含降维映射层和重建正则化项。压缩网络学习将中间层高维表示映射到低维紧凑表示，同时保留对推荐任务关键的信息。采用对比学习对齐压缩表示与原始表示的语义分布。

    • 关键设计：压缩比可调（论文展示4x-10x压缩），支持不同场景的精度-效率权衡。

    🔧 关键组件

    | 组件 | 说明 |

    |------|------|

    | 层级分析器 (Layer Analyzer) | 通过探针网络评估每层表示的任务相关性，自动识别最优压缩层（通常位于中间偏后位置）。输出各层的任务相关性分数。 |

    | 模块化分解器 (Modular Decomposer) | 基于信息论分解表示空间为任务相关模块和任务无关模块。采用互信息最大化准则，确保压缩后的表示保留任务关键信息。 |

    | 自适应压缩网络 (Adaptive Compression Network) | 轻量级MLP网络，包含降维映射、分布对齐和重建正则化三部分。支持1x-10x压缩比的可配置输出维度。 |

    import torch.nn as nn

    简化版 LLM 表示压缩框架

    class LayerAnalysisProbe(nn.Module):

    """步骤1: 层级分析 - 探针网络评估各层表示"""

    def __init__(self, layer_count, hidden_dim):

    super().__init__()

    # 为每层创建独立的探针

    self.probes = nn.ModuleList([

    nn.Sequential(

    nn.Linear(hidden_dim, 128),

    nn.ReLU(),

    nn.Linear(128, 1) # 相关性分数

    ) for _ in range(layer_count)

    ])

    def compute_relevance_scores(self, layer_representations):

    """

    计算每层与推荐目标的相关性分数

    返回: 每层的相关性分数 [num_layers]

    """

    scores = []

    for i, probe in enumerate(self.probes):

    # 使用探针评估该层表示的下游性能

    score = torch.sigmoid(probe(layer_representations[i]))

    scores.append(score)

    return torch.cat(scores)

    class ModularDecomposition(nn.Module):

    """步骤2: 模块化分解 - 分离任务相关/无关模块"""

    def __init__(self, hidden_dim, module_dim):

    super().__init__()

    # 任务相关模块 (可迁移的通用语义)

    self.task_related = nn.Linear(hidden_dim, module_dim)

    # 任务无关模块 (预训练特定，如语言建模)

    self.task_agnostic = nn.Linear(hidden_dim, module_dim)

    # 门控机制

    self.gate = nn.Linear(hidden_dim, 2)

    def forward(self, representation):

    """

    分解表示为任务相关和无关部分

    """

    # 门控权重决定分解比例

    gate_weights = torch.softmax(self.gate(representation), dim=-1)

    related_weight = gate_weights[:, 0:1]

    agnostic_weight = gate_weights[:, 1:2]

    # 任务相关: 中间层通用语义

    related_feat = self.task_related(representation) * related_weight

    # 任务无关: 最终层预训练特性

    agnostic_feat = self.task_agnostic(representation) * agnostic_weight

    return related_feat, agnostic_feat

    class AdaptiveCompressionNetwork(nn.Module):

    """步骤3: 自适应压缩网络 - 降维映射 + 对比学习对齐"""

    def __init__(self, input_dim, compressed_dim, compression_ratio=4):

    super().__init__()

    self.compression_ratio = compression_ratio

    self.compressed_dim = compressed_dim

    # 降维映射层

    self.encoder = nn.Sequential(

    nn.Linear(input_dim, input_dim // compression_ratio),

    nn.LayerNorm(input_dim // compression_ratio),

    nn.GELU(),

    nn.Linear(input_dim // compression_ratio, compressed_dim)

    )

    # 重建层

    self.decoder = nn.Sequential(

    nn.Linear(compressed_dim, input_dim // compression_ratio),

    nn.GELU(),

    nn.Linear(input_dim // compression_ratio, input_dim)

    )

    def contrastive_loss(self, original, compressed):

    """对比学习对齐语义分布"""

    # 归一化

    original_norm = torch.nn.functional.normalize(original, dim=-1)

    compressed_norm = torch.nn.functional.normalize(compressed, dim=-1)

    # InfoNCE 风格对比损失

    similarity = torch.matmul(original_norm, compressed_norm.T)

    labels = torch.arange(len(similarity), device=similarity.device)

    loss = nn.CrossEntropyLoss()(similarity, labels)

    return loss

    def forward(self, x, lambda_reg=0.1):

    """

    压缩 + 重建 + 对比学习

    """

    # 编码压缩

    compressed = self.encoder(x)

    # 重建正则化

    reconstructed = self.decoder(compressed)

    recon_loss = nn.MSELoss()(reconstructed, x)

    # 对比损失

    contrast_loss = self.contrastive_loss(x, compressed)

    # 总损失

    total_loss = recon_loss + lambda_reg * contrast_loss

    return compressed, total_loss

    class LLMCompressor(nn.Module):

    """完整的LLM压缩框架"""

    def __init__(self, num_layers, hidden_dim, compressed_dim=128, compression_ratio=4):

    super().__init__()

    self.layer_analysis = LayerAnalysisProbe(num_layers, hidden_dim)

    self.modular_decomp = ModularDecomposition(hidden_dim, hidden_dim // 2)

    self.compression_net = AdaptiveCompressionNetwork(

    hidden_dim, compressed_dim, compression_ratio

    )

    def forward(self, layer_representations):

    """

    输入: 各层表示列表 [num_layers, batch, hidden_dim]

    输出: 压缩后的紧凑表示

    """

    # 1. 层级分析 - 确定最优压缩层

    relevance_scores = self.layer_analysis.compute_relevance_scores(layer_representations)

    target_layer_idx = torch.argmax(relevance_scores)

    target_repr = layer_representations[target_layer_idx]

    # 2. 模块化分解 - 提取任务相关特征

    task_related, _ = self.modular_decomp(target_repr)

    # 3. 自适应压缩

    compressed, loss = self.compression_net(task_related)

    return compressed, loss, relevance_scores

    ❓ 常见问题

    Q：为什么LLM最终层表示反而对推荐任务有害？

    A：LLM最终层在预训练阶段被专门优化用于next-token prediction等代理任务，导致表示空间向语言建模目标偏移。推荐任务需要的语义（如用户偏好、商品属性）与语言建模存在差异，因此最终层表示包含的信息反而成为干扰源。

    Q：压缩后的表示还能用于其他任务吗？

    A：可以，但有一定限制。压缩后的表示针对推荐任务优化，保留的是任务相关模块。对于其他下游任务（如分类、排序），可能需要重新压缩或采用多任务压缩策略。论文建议每个任务单独训练压缩网络。

    Q：如何确定最优的压缩层和压缩比？

    A：论文建议通过探针网络遍历各层表示，选择下游性能最优的层级作为压缩点。压缩比需在精度和效率间权衡，实验表明4x-6x压缩比在大多数场景下是较优选择，可保持90%+性能的同时大幅降低存储和计算开销。
