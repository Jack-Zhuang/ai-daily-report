## 📖 背景与动机

大语言模型（LLM）在复杂推理任务中表现优异，但自回归生成的特性导致推理延迟较高。投机解码（Speculative Decoding）通过轻量级draft模型快速生成候选token，再由target模型验证，成为加速推理的主流方案。然而，现有的token级投机解码在多步推理场景存在根本缺陷：早期推理步骤的错误会像「滚雪球」一样传播放大，导致后续所有验证结果都基于错误前提。此外，使用外部奖励模型进行验证虽然有效，但引入额外延迟、计算开销，且泛化性受限。SpecGuard的核心动机是：在不使用外部工具的前提下，如何仅依靠模型自身的内部信号来准确识别并纠正推理错误？


**关键要点：**

- 投机解码通过draft-target双模型协作实现推理加速
- 多步推理中错误具有级联效应，单token错误可导致整条推理链失效
- 外部奖励模型方案存在延迟、算力开销和泛化性三重问题
- 模型内部包含丰富的自我验证信号，关键在于如何有效提取

## 💡 核心方法

### 方法概述

SpecGuard将投机解码的粒度从token提升到step（推理步骤），通过自一致性采样生成多个候选步骤，并利用两个轻量级内部信号进行验证，动态决定接受或重算。


### 详细设计

SpecGuard框架包含三个核心模块：

1. **自一致性采样（Self-Consistency Sampling）**：在每个推理步骤，对同一个输入执行K次采样（通常K=4或8），生成K条不同的推理路径。通过比较这些路径的输出，选择出现频率最高的「共识答案」作为该步骤的候选输出。这种设计基于关键假设：正确的推理路径在多次采样中具有更高的再现性。

2. **注意力Grounding Score（AGS）**：基于transformer的注意力机制特性，计算当前token对输入序列和已接受步骤的注意力权重分布。该分数衡量输出的「可归因性」——高分数意味着输出与输入/前序推理存在明确的语义关联，而非凭空杜撰。具体实现时，取多层注意力的加权平均，并对跨序列的注意力头进行特殊处理。

3. **Log-probability Score（LPS）**：直接利用语言模型的softmax前输出，衡量模型对当前token的置信度。该信号捕获token级的确定性：模型高度自信时LPS高，犹豫不决时LPS低。这与人类的「知道自己知道什么」的直觉相符。

验证阶段，SpecGuard计算AGS和LPS的加权组合，通过可学习的阈值函数判断当前step是否可信。低于阈值时，使用target模型重新计算该步骤；高于阈值则直接接受。整个过程无需外部模型介入，实现了真正的「自验证」。

### 🔧 关键组件

| 组件 | 说明 |
|------|------|
| Self-Consistency Sampler | 执行K次独立采样，通过投票选择共识输出。K值通常设为4-8，可在推理质量和计算开销间取得平衡。 |
| Attention Grounding Score (AGS) | 基于多层注意力权重计算，衡量输出对输入和历史的归因程度。无需额外训练，直接从模型权重提取。 |
| Log-probability Score (LPS) | 利用softmax前的logits，评估模型对每个token的置信度。计算效率极高，几乎无额外开销。 |
| Adaptive Verifier | 融合AGS和LPS的决策模块，包含可学习阈值。通过历史验证数据微调，可适配不同任务类型。 |

import random
from typing import List, Dict, Any

# ============================================================
# SpecGuard: 自验证推理框架
# ============================================================

class SpecGuard:
    def __init__(self, model, k_samples=4, lambda_ags=0.6, lambda_lps=0.4):
        self.model = model
        self.k_samples = k_samples  # 自一致性采样次数
        self.lambda_ags = lambda_ags  # AGS权重
        self.lambda_lps = lambda_lps  # LPS权重
        self.threshold = 0.5  # 可学习阈值
    
    # --------------------------------------------------------
    # 模块1: 自一致性采样
    # --------------------------------------------------------
    def self_consistency_sampling(self, input_text: str, prompt: str) -> List[str]:
        """
        对同一输入执行K次采样，生成多条推理路径
        返回K条不同的推理路径
        """
        paths = []
        for _ in range(self.k_samples):
            # 每次采样使用随机温度产生不同路径
            response = self.model.generate(
                input_text, 
                prompt, 
                temperature=random.uniform(0.7, 1.2)  # 随机温度
            )
            paths.append(response)
        return paths
    
    def get_consensus_answer(self, paths: List[str]) -> str:
        """
        选择出现频率最高的答案作为共识答案
        """
        from collections import Counter
        answers = [self.extract_final_answer(p) for p in paths]
        counter = Counter(answers)
        # 返回出现最频繁的答案
        return counter.most_common(1)[0][0]
    
    def extract_final_answer(self, reasoning_path: str) -> str:
        """从推理路径中提取最终答案"""
        # 伪代码：简化处理
        return reasoning_path.split("####")[-1].strip()
    
    # --------------------------------------------------------
    # 模块2: 注意力Grounding Score (AGS)
    # --------------------------------------------------------
    def compute_ags(self, input_ids, output_ids, attention_weights) -> float:
        """
        计算注意力Grounding Score
        衡量输出与输入的语义关联程度
        """
        # 多层注意力加权平均
        multi_layer_weights = self.model.get_multi_layer_attention()
        
        # 计算跨序列注意力（输入->输出的注意力）
        cross_attention = self.compute_cross_attention(
            multi_layer_weights, 
            input_ids, 
            output_ids
        )
        
        # AGS = 加权平均注意力分数
        ags_score = sum(
            layer_weight * cross_attention[layer] 
            for layer, layer_weight in enumerate(self.layer_weights)
        )
        
        return ags_score
    
    def compute_cross_attention(self, weights, input_ids, output_ids) -> Dict:
        """计算跨序列的注意力头权重"""
        # 伪代码：实际需遍历所有注意力头
        cross_scores = {}
        for layer_idx, layer_attn in enumerate(weights):
            # 关注输入tokens对输出tokens的注意力
            cross_scores[layer_idx] = layer_attn[:, :, :len(input_ids), :].mean()
        return cross_scores
    
    # --------------------------------------------------------
    # 模块3: Log-probability Score (LPS)
    # --------------------------------------------------------
    def compute_lps(self, logits, target_token_id) -> float:
        """
        计算token级别的置信度
        使用softmax前的logit值
        """
        # 获取目标token的logit
        target_logit = logits[target_token_id]
        
        # 转换为log概率（未归一化的概率对数）
        lps_score = target_logit - logits.max()  # 相对logit
        
        return lps_score
    
    def batch_compute_lps(self, logits_sequence, token_ids) -> float:
        """计算整个序列的平均LPS"""
        token_lps = [
            self.compute_lps(logits, token_id) 
            for logits, token_id in zip(logits_sequence, token_ids)
        ]
        return sum(token_lps) / len(token_lps)
    
    # --------------------------------------------------------
    # 验证阶段: 综合评分与决策
    # --------------------------------------------------------
    def compute_trust_score(self, ags: float, lps: float) -> float:
        """
        计算综合可信度分数
        trust_score = λ_ags * AGS + λ_lps * LPS
        """
        return self.lambda_ags * ags + self.lambda_lps * lps
    
    def verify_step(self, step_output: str, input_text: str, 
                    current_reasoning: str) -> bool:
        """
        验证当前推理步骤是否可信
        True: 接受该步骤
        False: 需要使用target模型重新计算
        """
        # 获取注意力权重
        attention_weights = self.model.get_attention()
        
        # 计算AGS
        ags = self.compute_ags(
            self.model.input_ids, 
            self.model.output_ids,
            attention_weights
        )
        
        # 计算LPS
        logits = self.model.get_logits()
        lps = self.batch_compute_lps(logits, self.model.output_ids)
        
        # 综合评分
        trust_score = self.compute_trust_score(ags, lps)
        
        # 决策：是否接受
        return trust_score >= self.threshold
    
    # --------------------------------------------------------
    # 主推理流程
    # --------------------------------------------------------
    def verify_inference(self, problem: str) -> str:
        """
        SpecGuard自验证推理主流程
        """
        # Step 1: 自一致性采样
        sampling_paths = self.self_consistency_sampling(
            problem, 
            prompt="Let's solve this step by step."
        )
        consensus = self.get_consensus_answer(sampling_paths)
        
        # Step 2: 逐步验证
        reasoning_steps = self.split_reasoning(consensus)
        verified_reasoning = []
        
        for step in reasoning_steps:
            # 验证当前步骤
            is_trusted = self.verify_step(
                step, 
                problem, 
                "\n".join(verified_reasoning)
            )
            
            if is_trusted:
                # 接受步骤
                verified_reasoning.append(step)
            else:
                # 使用target模型重新计算（伪代码）
                new_step = self.target_model_refine(step, problem)
                verified_reasoning.append(new_step)
        
        return " #### ".join(verified_reasoning) + f" #### {self.extract_final_answer(consensus)}"
    
    def split_reasoning(self, text: str) -> List[str]:
        """将推理文本分割为步骤"""
        # 伪代码：按换行或步骤标记分割
        return [s.strip() for s in text.split("\n") if s.strip()]
    
    def target_model_refine(self, step: str, problem: str) -> str:
        """使用target模型重新计算不可信步骤"""
        # 伪代码：实际调用更强大的模型
        return f"[Refined by target model] {step}"

## ❓ 常见问题

**Q：SpecGuard相比使用外部奖励模型的方案有何优势？**

A：主要优势体现在三个方面：①零额外算力开销，无需加载和运行第二个模型；②无部署复杂度，适配任意预训练模型；③泛化性强，不依赖特定奖励模型的训练数据分布。


**Q：自一致性采样K值如何选择？**

A：论文实验表明K=4-8是较优区间。K过小（如K=2）自一致性投票效果不明显；K过大（如K=16）边际收益递减且增加延迟。实际应用中可根据推理任务的复杂度动态调整。


**Q：如果AGS和LPS判断不一致怎么办？**

A：论文采用加权求和方案：Final_Score = α×AGS + β×LPS。权重参数可通过验证集学习。在实验中，当两者差异超过阈值时，会触发保守策略（强制重算），因为不一致本身就可能预示推理的模糊性。


**Q：SpecGuard适用于所有推理任务吗？**

A：更适用于需要显式多步推理的任务（如数学解题、逻辑推导），对短回复任务（如问答、摘要）加速效果有限。核心原因是短任务错误传播空间小，step级验证的收益不明显。
