## 📖 背景与动机

随着OpenClaw、Claude Code等通用智能体系统的兴起，对基础大模型的能力要求日益严苛。Agentic RL作为关键的后训练范式，旨在赋予LLM决策制定、工具使用等核心能力。与传统的单轮token级对齐（如RLHF、RLVR）不同，Agentic RL面向多轮交互场景，面临延迟奖励、稀疏奖励、长可变上下文等新挑战。传统的token级MDP建模已无法充分捕捉真实LLM智能体行为，导致优化效率低下。StepPO正是为解决这一根本性建模局限而提出的。


**关键要点：**

- 通用智能体系统对LLM能力要求不断提升
- Agentic RL面临token级建模与多轮交互的本质矛盾
- 延迟/稀疏奖励和长上下文是关键挑战

## 💡 核心方法

### 方法概述

StepPO提出step级MDP框架，将优化粒度从token提升到step层面，实现与人类协作模式的对齐


### 详细设计

StepPO的核心创新在于三个层面：

1. **Step级MDP建模**：传统方法将MDP建立在单个token上，而StepPO将MDP建立在完整的交互step上。每个step包含观察st（智能体可获取的信息）、完整动作at（可能包含多个token的工具调用序列）和环境奖励rt。这种建模方式更贴近实际智能体执行流程，能够自然处理多轮交互中的因果依赖关系。

2. **Trajectory表示层**：支持长且可变的上下文，将历史信息聚合为step级别的状态表示，便于后续优化计算。

3. **Step级信用分配**：从token/轨迹级信用分配升级到step级，通过δt = rt + γV(st+1) − V(st)计算step级TD误差，实现更精细、更准确的梯度估计。

在策略优化层面，StepPO将πθ(y|x)重新定义为πθ(at|st)，与step级MDP对齐，并基于旧策略πθold(at|st)进行策略更新，确保训练稳定性。

### 🔧 关键组件

| 组件 | 说明 |
|------|------|
| Step级MDP建模 | 将MDP建立在step层面而非token层面，用(st, at, rt)三元组建模完整交互动作 |
| Step级信用分配 | 基于step级TD误差δt进行梯度估计，实现延迟奖励场景下的精准信用分配 |
| 轨迹表示层 | 聚合历史信息为step级状态表示，支持长上下文场景 |

### 💻 代码示例

```python
import numpy as np
from typing import List, Dict, Tuple

# ============================================
# StepPO 简化实现
# ============================================

class Step:
    """Step级MDP基本单元"""
    def __init__(self, obs, action, reward, done=False):
        self.obs = obs          # st: 观察
        self.action = action    # at: 完整动作(工具调用序列)
        self.reward = reward    # rt: 环境奖励
        self.done = done        # 是否终止

class StepTrajectory:
    """Trajectory表示层 - 聚合历史为step级状态"""
    def __init__(self):
        self.steps: List[Step] = []
        self.step_repr: List[np.ndarray] = []  # step级状态表示
    
    def add_step(self, step: Step):
        self.steps.append(step)
        # 聚合历史信息生成step级状态表示
        history_context = self._aggregate_history()
        self.step_repr.append(history_context)
    
    def _aggregate_history(self) -> np.ndarray:
        """将历史信息聚合为step级表示"""
        if not self.steps:
            return np.zeros(128)  # 简化表示维度
        
        # 伪代码: 聚合历史step的信息
        all_rewards = [s.reward for s in self.steps]
        last_obs = self.steps[-1].obs
        return np.mean([last_obs, np.mean(all_rewards)], axis=0)

class ValueFunction:
    """Step级价值函数 V(st)"""
    def __init__(self, dim=128):
        self.weights = np.random.randn(dim) * 0.1
    
    def forward(self, state_repr: np.ndarray) -> float:
        """V(st)"""
        return np.dot(self.weights, state_repr)

class StepPO:
    """StepPO策略优化器"""
    
    def __init__(self, gamma=0.99, lr=0.001):
        self.gamma = gamma
        self.lr = lr
        self.value_net = ValueFunction()
        self.policy_params = np.random.randn(128) * 0.1
    
    def compute_step_td_error(self, trajectory: StepTrajectory, t: int) -> float:
        """
        Step级信用分配: δt = rt + γV(st+1) − V(st)
        """
        steps = trajectory.steps
        reprs = trajectory.step_repr
        
        if t >= len(steps) - 1:
            return 0.0  # 终止状态
        
        rt = steps[t].reward           # 当前奖励
        V_st = self.value_net.forward(reprs[t])      # V(st)
        V_st1 = self.value_net.forward(reprs[t + 1])  # V(st+1)
        
        # TD误差
        delta = rt + self.gamma * V_st1 - V_st
        return delta
    
    def policy_loss(self, trajectory: StepTrajectory, t: int, 
                    old_policy_logits: np.ndarray) -> float:
        """
        策略损失: 基于step级策略 πθ(at|st)
        伪代码: 使用重要性采样和策略比
        """
        step = trajectory.steps[t]
        state_repr = trajectory.step_repr[t]
        
        # 旧策略概率 (伪代码)
        pi_old = self._softmax(old_policy_logits, state_repr)
        
        # 新策略概率 (伪代码)
        pi_new = self._softmax(self.policy_params, state_repr)
        
        # 策略比
        ratio = pi_new / (pi_old + 1e-8)
        
        # 使用step级TD误差加权
        td_error = self.compute_step_td_error(trajectory, t)
        
        # PPO-style 裁剪损失 (伪代码)
        return -min(ratio * td_error, np.clip(ratio, 0.8, 1.2) * td_error)
    
    def _softmax(self, params: np.ndarray, state: np.ndarray) -> float:
        """简化的策略概率计算"""
        logits = np.dot(params, state)
        return 1.0 / (1.0 + np.exp(-logits))
    
    def update(self, trajectories: List[StepTrajectory]):
        """更新策略 - 基于旧策略的稳定性更新"""
        total_loss = 0.0
        
        for traj in trajectories:
            for t in range(len(traj.steps) - 1):
                # 保存旧策略 (伪代码)
                old_policy = self.policy_params.copy()
                
                # 计算step级TD误差
                delta = self.compute_step_td_error(traj, t)
                
                # 策略梯度更新 (伪代码)
                policy_grad = delta * traj.step_repr[t]
                self.policy_params += self.lr * policy_grad
                
                # 价值网络更新 (伪代码)
                value_grad = delta * traj.step_repr[t]
                self.value_net.weights += self.lr * value_grad


# ============================================
# 使用示例
# ============================================

if __name__ == "__main__":
    # 初始化StepPO
    steppo = StepPO(gamma=0.99)
    
    # 收集trajectories (伪代码)
    trajectories = []
    
    for _ in range(10):
        traj = StepTrajectory()
        
        # 模拟交互step序列
        for step_idx in range(5):
            # 模拟环境交互
            obs = np.random.randn(128)
            action = f"call_tool_{step_idx}"
            reward = np.random.randn()
            done = (step_idx == 4)
            
            step = Step(obs, action, reward, done)
            traj.add_step(step)
        
        trajectories.append(traj)
    
    # 执行StepPO更新
    steppo.update(trajectories)
    
    print("StepPO更新完成!")
```
