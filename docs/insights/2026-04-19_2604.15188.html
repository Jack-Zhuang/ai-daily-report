📖 背景与动机

视觉语言模型（VLM）在处理高分辨率图像或长视频帧时面临计算量二次增长的严峻挑战。视觉token剪枝技术通过移除冗余的视觉token来降低计算成本，已成为缓解这一问题的关键技术。然而，现有方法普遍采用预定义的固定剪枝配置（如每层剪枝固定比例的token），存在两个核心问题：一是缺乏对配置是否达到「计算-性能帕累托最优」的判断依据；二是固定配置难以适应不同模型架构和任务需求，需要大量人工调参。学术界和工业界迫切需要一种自动化、理论支撑的配置优化方法，能够在给定计算预算下自动找到性能最优的剪枝策略。

关键要点：
  • VLM处理高分辨率图像时计算量呈二次增长，视觉token剪枝是有效的缓解手段
  • 现有剪枝方法依赖预定义配置，缺乏对配置最优性的理论分析和自动优化能力
  • 手动调参成本高，且固定配置难以迁移到不同模型和任务
  • 需要一种方法能在计算预算约束下自动找到Pareto最优的剪枝配置

💡 核心方法

方法概述

VisPCO将视觉token剪枝配置优化问题形式化为带计算预算约束的Pareto多目标优化，通过连续松弛使离散的剪枝决策可微分，结合直通估计器实现梯度反向传播，最终使用增广拉格朗日方法求解约束优化问题。

详细设计

VisPCO的核心创新在于将原本离散的剪枝配置选择问题转化为连续可优化的数学规划问题。首先，论文定义每层的剪枝比例为连续变量，通过sigmoid函数将实数域映射到[0,1]区间，实现配置的连续松弛。其次，采用直通估计器（Straight-Through Estimator, STE）处理不可微分的剪枝操作，使梯度能够绕过离散决策直接传递到配置参数。在优化层面，论文使用增广拉格朗日方法处理「计算预算约束」这一优化目标，将原问题转化为增广拉格朗日函数的无约束优化，通过交替更新原始变量、拉格朗日乘子和惩罚参数实现收敛。值得注意的是，论文还引入了可学习的核函数来建模层间token依赖关系，实现更精细的层级别剪枝策略。整体框架可端到端训练，自动搜索满足计算约束的最优配置组合。

🔧 关键组件

| 组件 | 说明 |

|------|------|

| 连续松弛模块 | 使用sigmoid函数将每层剪枝比例从离散的{0%, 25%, 50%, 75%}等选项松弛为连续变量，允许任意比例的token被保留，实现配置空间的平滑化 |

| 直通估计器（STE） | 在前向传播时执行真实的二值化剪枝决策（保留或丢弃token），反向传播时绕过不可导的离散操作，直接传递梯度到配置参数，实现端到端训练 |

| 增广拉格朗日优化器 | 构建增广拉格朗日函数，通过交替方向乘子法（ADMM）交替更新：①剪枝配置参数 ②拉格朗日乘子 ③惩罚参数，逐步逼近约束满足的Pareto最优解 |

| 可学习核函数 | 建模相邻层token之间的依赖关系，学习跨层的token重要性评分，使剪枝决策能够考虑全局token分布，而非仅依赖单层统计信息 |

import torch

import torch.nn as nn

import torch.nn.functional as F

============== 核心组件 ==============

class LearnablePruningRatio(nn.Module):

"""可学习的剪枝比例参数 (连续变量)"""

def __init__(self, num_layers):

super().__init__()

# 实数域参数，通过sigmoid映射到[0,1]

self.ratio_logits = nn.Parameter(torch.zeros(num_layers))

def forward(self):

# Sigmoid松弛：实数域 -> [0,1]区间

return torch.sigmoid(self.ratio_logits)

class StraightThroughEstimator(torch.autograd.Function):

"""直通估计器(STE)：绕过不可微分操作传递梯度"""

@staticmethod

def forward(ctx, ratios, thresholds):

# 前向：离散决策（不可微分）

masks = (ratios > thresholds).float()

ctx.save_for_backward(masks)

return masks

@staticmethod

def backward(ctx, grad_output):

# 反向：直接将梯度传递给输入（STE核心）

masks, = ctx.saved_tensors

return grad_output * masks, None

class LayerDependencyKernel(nn.Module):

"""可学习的核函数：建模层间token依赖关系"""

def __init__(self, num_layers, kernel_size=3):

super().__init__()

# 可学习的卷积核

self.kernel = nn.Parameter(torch.ones(kernel_size))

def forward(self, features):

# 使用可学习核卷积建模层间依赖

return F.conv1d(features.unsqueeze(0), self.kernel.view(1, 1, -1)).squeeze(0)

============== 增广拉格朗日优化器 ==============

class AugmentedLagrangianOptimizer:

"""增广拉格朗日方法：处理计算预算约束"""

def __init__(self, budget_constraint_fn, rho_init=1.0, mu=10.0):

self.lambda_mult = 0.0 # 拉格朗日乘子

self.rho = rho_init # 惩罚参数

self.mu = mu # 放大系数

self.budget_fn = budget_constraint_fn

def compute_aug_lagrangian(self, model, masks):

# 约束项：计算预算偏离

budget_violation = self.budget_fn(masks)

# 增广拉格朗日函数：原目标 + 乘子*约束 + 惩罚项

aug_lagrangian = (

model.task_loss()

+ self.lambda_mult * budget_violation

+ 0.5 self.rho budget_violation ** 2

)

return aug_lagrangian

def update_multipliers(self, budget_violation):

# 更新拉格朗日乘子

self.lambda_mult += self.rho * budget_violation

# 自适应调整惩罚参数

self.rho = min(self.rho * self.mu, 1e6)

============== VisPCO 完整框架 ==============

class VisPCO(nn.Module):

"""VisPCO: 连续松弛 + STE + 增广拉格朗日 + 层间依赖"""

def __init__(self, model, num_layers, compute_budget):

super().__init__()

self.model = model

self.num_layers = num_layers

# 1. 连续剪枝比例参数

self.pruning_ratios = LearnablePruningRatio(num_layers)

# 2. 层间依赖核函数

self.layer_kernel = LayerDependencyKernel(num_layers)

# 3. 增广拉格朗日优化器

self.aug_lagrangian = AugmentedLagrangianOptimizer(

budget_constraint_fn=lambda m: (m.sum() - compute_budget)

)

# 4. 阈值（用于STE离散化）

self.threshold = 0.5

def forward(self, x):

# 获取连续剪枝比例 [0,1]

ratios = self.pruning_ratios()

# 融合层间依赖信息

refined_ratios = ratios + 0.1 * self.layer_kernel(ratios.unsqueeze(0))

# STE前向：离散掩码生成

masks = StraightThroughEstimator.apply(refined_ratios, self.threshold)

# 应用掩码到模型

return self.model(x, masks)

def training_step(self, x, target):

# 前向传播

output = self.forward(x)

# 计算任务损失

task_loss = F.cross_entropy(output, target)

# 获取当前掩码

masks = self.pruning_ratios() > self.threshold

# 计算增广拉格朗日损失

total_loss = self.aug_lagrangian.compute_aug_lagrangian(

type('obj', (object,), {'task_loss': lambda: task_loss})(),

masks.float()

)

# 梯度反向传播（STE保证梯度流）

total_loss.backward()

return total_loss, task_loss

def update_constraints(self, masks):

"""交替更新：调整乘子和惩罚参数"""

budget_violation = masks.sum() - self.compute_budget

self.aug_lagrangian.update_multipliers(budget_violation)

❓ 常见问题

Q：VisPCO与传统网格搜索剪枝配置相比有什么优势？

A：网格搜索需要遍历所有可能的配置组合，时间复杂度随层数和配置选项数指数增长（如4层×4种配置需256次评估）；VisPCO通过梯度优化在单次训练中学习连续配置空间，效率提升数十倍，且能找到网格搜索难以覆盖的中间配置，实现更精细的Pareto前沿逼近。

Q：连续松弛后如何保证最终剪枝比例的离散性？

A：论文采用直通估计器（STE）解决这一问题：前向传播时使用sigmoid输出的概率进行伯努利采样得到二值化决策（保留/丢弃），反向传播时梯度直接绕过采样节点传到sigmoid输入，实现训练时的连续优化和推理时的离散执行。

Q：VisPCO如何处理不同计算预算下的配置需求？

A：通过增广拉格朗日方法中的预算约束项实现：约束项惩罚超出预算的配置，惩罚系数动态调整以平衡原目标（性能）和约束（预算）。只需设置不同的预算上限B，即可生成整条Pareto前沿上的配置，无需重新训练。

Q：VisPCO是否适用于所有VLM架构？
