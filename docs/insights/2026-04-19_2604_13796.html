## 📖 背景与动机

每日梦幻体育（DFS）是一种用户参与虚拟球队组建并竞争奖金的娱乐形式。与传统电商推荐不同，DFS的赛事推荐面临独特挑战：每场比赛都有一个"round-lock"截止时间，用户必须在比赛开始前完成组队。一旦错过窗口期，用户将失去参与资格，导致参与度和收入的双重损失。

现有的推荐系统（如协同过滤、 DIN等）主要针对静态商品目录设计，假设用户随时可以购买。这些系统忽略了时间约束的紧迫性——即将开始的比赛应该比远期比赛获得更高的推荐权重。同时，用户的历史行为模式也随时间衰减，新近的交互行为往往更具预测价值。

因此，如何构建一个能感知时间紧迫性、动态调整推荐策略的系统，成为DFS平台提升用户参与度的关键技术难题。


**关键要点：**

- DFS赛事推荐具有硬性时间约束，截止后用户无法参与
- 传统推荐系统假设静态物品目录，无法处理动态截止时间
- 用户历史行为的预测价值随时间衰减，需要时序敏感性
- 推荐时机与内容同样重要，早一分钟触达可能带来完全不同的参与结果

## 💡 核心方法

### 方法概述

该方法基于深度兴趣网络（DIN）进行改造，在模型中注入双层时间感知能力：通过实时紧迫性特征编码赛事截止倒计时，通过时间间隔位置编码建模用户行为的时间衰减效应。


### 详细设计

整体架构包含输入层、紧迫性特征注入模块、时间间隔编码模块和DIN核心模块。

【紧迫性特征注入】
对于每个候选赛事，模型实时计算"距截止时间"（Time-to-Round-Lock），并将其离散化为紧迫性等级（如：0-30分钟、30-60分钟、1-2小时、>2小时）。此外，还引入赛事重要性特征（是否是重要比赛）、用户已选球员数量等上下文紧迫性信号。这些特征与候选赛事的Embedding向量拼接后输入模型，使其能够学习到"越紧迫的赛事越应提前推荐"的规律。

【时间间隔位置编码】
DIN通过注意力机制聚合用户历史行为，但原始实现未考虑行为发生的时间间隔。论文提出Temporal Positional Encoding：对于用户历史交互序列中的第i个行为与当前推荐请求之间的时间间隔Δt，将其映射为位置编码向量P(Δt)。该编码与行为Embedding相加，使模型能够区分"昨天点击"和"一个月前点击"的相同行为。编码函数采用对数变换以适应时间跨度的不均匀分布：P(Δt) = f(log(1 + Δt))。

【损失函数】
采用Listwise Neural NDCG损失，直接优化推荐列表的整体排序质量，而非逐个物品的点击预测。该损失函数与评价指标NDCG一致，能够更好地指导模型学习最优排序。

【在线服务架构】
系统采用实时特征工程管道：赛事数据流进入特征服务，计算每个候选赛事的紧迫性得分；用户画像服务提供历史行为序列；推荐引擎在100ms内完成特征组装、模型推理和结果排序，最终返回Top-K推荐列表。

### 🔧 关键组件

| 组件 | 说明 |
|------|------|
| Urgency Feature Injection | 实时计算每个候选赛事的距截止时间、赛事重要性、用户参与进度等紧迫性信号，与候选Embedding拼接后输入模型，使模型感知时间约束。 |
| Temporal Positional Encoding | 对用户历史行为与当前请求的时间间隔进行对数变换编码，与行为Embedding融合，让模型学习行为的时效性权重。 |
| Listwise Neural NDCG Loss | 以整个推荐列表为单位优化NDCG指标，而非逐点优化，确保模型输出的排序列表整体质量最优。 |

### 💻 代码示例

```python
import numpy as np

# ==================== 1. 紧迫性特征注入模块 ====================

class UrgencyFeatureInjector:
    """将紧迫性特征注入到候选赛事表示中"""
    
    # 紧迫性等级离散化
    URGENCY_LEVELS = {
        'extreme': (0, 30),       # 0-30分钟
        'high': (30, 60),         # 30-60分钟
        'medium': (60, 120),      # 1-2小时
        'low': (120, float('inf')) # >2小时
    }
    
    def compute_time_to_lock(self, current_time, round_lock_time):
        """计算距截止时间（分钟）"""
        return (round_lock_time - current_time).total_seconds() / 60
    
    def discretize_urgency(self, time_to_lock):
        """将时间离散化为紧迫性等级"""
        for level, (low, high) in self.URGENCY_LEVELS.items():
            if low <= time_to_lock < high:
                return level
        return 'low'
    
    def inject_features(self, candidate_embedding, current_time, round_lock_time,
                       is_important, selected_players_count):
        """
        拼接紧迫性特征与候选赛事Embedding
        """
        # 计算紧迫性等级
        time_to_lock = self.compute_time_to_lock(current_time, round_lock_time)
        urgency_level = self.discretize_urgency(time_to_lock)
        
        # 构建紧迫性特征向量
        urgency_feature = np.array([
            time_to_lock / 1440,  # 归一化到天
            1 if urgency_level == 'extreme' else 0,
            1 if urgency_level == 'high' else 0,
            1 if is_important else 0,
            selected_players_count / 10  # 归一化
        ])
        
        # 与Embedding拼接
        enhanced_embedding = np.concatenate([candidate_embedding, urgency_feature])
        return enhanced_embedding


# ==================== 2. 时间间隔位置编码 ====================

class TemporalPositionalEncoding:
    """时间间隔编码：区分近期与远期行为"""
    
    def __init__(self, embed_dim):
        self.embed_dim = embed_dim
    
    def compute_time_interval(self, behavior_time, request_time):
        """计算行为与请求之间的时间间隔（秒）"""
        return (request_time - behavior_time).total_seconds()
    
    def encode(self, time_interval):
        """
        对数变换编码
        P(Δt) = f(log(1 + Δt))
        """
        # 对数变换
        log_interval = np.log(1 + time_interval)
        # 映射到编码维度（伪代码：实际用可学习的MLP或傅里叶变换）
        encoding = np.sin(log_interval * np.pi / 2)  # 缩放到[-1, 1]
        return np.tile(encoding, self.embed_dim)[:self.embed_dim]
    
    def add_to_behavior_sequence(self, behavior_embeddings, behavior_times, request_time):
        """为行为序列添加时间间隔编码"""
        enhanced_sequence = []
        for emb, bt in zip(behavior_embeddings, behavior_times):
            delta_t = self.compute_time_interval(bt, request_time)
            pos_encoding = self.encode(delta_t)
            # 编码与行为Embedding相加
            enhanced_emb = emb + pos_encoding
            enhanced_sequence.append(enhanced_emb)
        return np.array(enhanced_sequence)


# ==================== 3. Listwise NDCG损失 ====================

class ListwiseNDCGLoss:
    """Listwise Neural NDCG损失，直接优化排序质量"""
    
    def compute_dcg(self, relevance_scores):
        """计算DCG"""
        dcg = 0
        for i, rel in enumerate(relevance_scores):
            dcg += rel / np.log2(i + 2)  # i+2因为从位置1开始
        return dcg
    
    def compute_ndcg(self, predicted_scores, true_labels, k=None):
        """计算NDCG@k"""
        # 按预测分数排序
        ranked_indices = np.argsort(-predicted_scores)
        ranked_labels = true_labels[ranked_indices]
        
        if k:
            ranked_labels = ranked_labels[:k]
        
        dcg = self.compute_dcg(ranked_labels)
        ideal_labels = np.sort(true_labels)[::-1][:k] if k else np.sort(true_labels)[::-1]
        idcg = self.compute_dcg(ideal_labels)
        
        return dcg / idcg if idcg > 0 else 0
    
    def forward(self, predictions, labels):
        """
        计算Listwise NDCG损失
        损失 = 1 - NDCG（最小化损失即最大化NDCG）
        """
        ndcg = self.compute_ndcg(predictions, labels)
        loss = 1 - ndcg
        return loss


# ==================== 4. 在线服务架构 ====================

class OnlineRecommendationSystem:
    """简化版在线推荐服务"""
    
    def __init__(self):
        self.urgency_injector = UrgencyFeatureInjector()
        self.temporal_encoder = TemporalPositionalEncoding(embed_dim=64)
        self.ndcg_loss = ListwiseNDCGLoss()
    
    def online_serving_pipeline(self, request, candidate_events, user_profile, model):
        """
        在线服务管道：100ms内完成推荐
        """
        # Step 1: 实时特征工程 - 计算紧迫性得分
        current_time = request.timestamp
        for event in candidate_events:
            event['enhanced_embedding'] = self.urgency_injector.inject_features(
                event['embedding'],
                current_time,
                event['round_lock_time'],
                event['is_important'],
                user_profile['selected_players_count']
            )
        
        # Step 2: 获取用户历史行为序列并添加时间编码
        behavior_sequence = user_profile['behavior_history']
        enhanced_behaviors = self.temporal_encoder.add_to_behavior_sequence(
            behavior_sequence['embeddings'],
            behavior_sequence['timestamps'],
            current_time
        )
        
        # Step 3: 模型推理 - DIN注意力聚合
        candidate_embeddings = [e['enhanced_embedding'] for e in candidate_events]
        attention_output = model.din_attention(
            query=candidate_embeddings,
            keys=enhanced_behaviors,
            keys_mask=user_profile['behavior_mask']
        )
        
        # Step 4: 排序输出
        scores = model.predict(attention_output)
        ranked_indices = np.argsort(-scores)
        
        # Step 5: 返回Top-K推荐
        top_k = request.top_k
        recommendations = [candidate_events[i] for i in ranked_indices[:top_k]]
        
        return recommendations
```

**适用场景：**

- 🎯 票务平台，对即将售罄或演出临近的票务进行紧迫性加权推荐


**实现难度**：中等

**工程挑战：**

- ⚠️ 实时特征计算：需要稳定可靠的实时特征服务，保证紧迫性特征的时效性
- ⚠️ 特征工程复杂度：紧迫性特征的离散化阈值需要根据业务数据调优
- ⚠️ 系统延迟要求：推荐引擎需在100ms内完成响应，对工程架构提出高要求
- ⚠️ 冷启动问题：新用户缺乏历史行为序列，时间间隔编码无法生效


**代码实现思路**：

核心实现思路：
1. 特征层定义紧迫性特征类，继承nn.Module，输入候选赛事特征和当前时间戳
2. 时间间隔编码使用nn.Embedding，将时间差值对数化后映射为向量
3. DIN的Attention层接收行为序列、时间间隔编码、候选向量，输出聚合表示
4. 损失函数使用LambdaRank风格的listwise损失，直接优化NDCG
5. 在线服务使用特征存储（Tair/Redis）加速特征读取，配合模型服务框架（TensorFlow Serving）实现毫秒级推理

## ❓ 常见问题

**Q：为什么传统DIN无法直接用于DFS推荐？**

A：传统DIN假设用户可以随时对推荐物品采取行动，忽略了物品的时间有效性约束。在DFS场景中，赛事存在截止时间，如果推荐已过期的赛事，用户将无法参与。DIN缺乏对这种'机会窗口'的建模能力。


**Q：紧迫性特征的阈值如何确定？**

A：论文通过离线数据分析用户参与行为的时间分布，确定紧迫性等级划分。例如，若大多数用户在比赛开始前2小时内完成组队，则以此为分界线将赛事划分为不同紧迫等级。实际阈值需结合业务数据调优。


**Q：时间间隔编码为什么采用对数变换？**

A：用户行为的时间间隔跨度极大（从几分钟到数月不等）。线性编码会导致短间隔区分度不足，长间隔又被压缩。对数变换能在短间隔区域保持高分辨率，同时有效压缩长间隔，符合用户兴趣衰减的非线性特征。


**Q：Listwise损失相比Pointwise损失有何优势？**

A：Pointwise损失优化每个物品的独立预测概率，不考虑物品间的相对顺序；Listwise损失直接以排序指标（如NDCG）作为优化目标，能学习到'排序靠前的物品更重要'的规律，输出结果更符合实际推荐需求。
