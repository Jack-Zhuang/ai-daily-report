LLM-as-Judge已成为NLG领域的主流自动评估范式，通过让大语言模型扮演评判者，对生成文本的质量打分。这种方法在Chatbot Arena、MT-Bench等评测基准中被广泛采用。然而，这种范式存在一个根本性问题：我们知道LLM在聚合层面的表现尚可，但对于具体到"每一个"样本的评估结果，它的可靠性如何？是否有隐藏的系统性错误？

传统评估方法只关注整体准确率或相关性等宏观指标，忽视了实例级别的可靠性诊断。就像医疗诊断不仅需要知道整体治愈率，还需要知道对每个病人的诊断可信度。LLM评估同样需要这种细粒度的可靠性度量。

## 💡 核心方法

### 方法概述

论文提出双管齐下的诊断框架：传递性分析检验评判偏好的逻辑一致性，保角预测提供理论保证的置信区间。两者互补，从不同维度诊断LLM评判者的可靠性。


### 详细设计

**传递性分析**：在数学上，传递性指如果A>B且B>C，则必有A>C。对于LLM评判，传递性违规表现为"有向3-循环"——如模型A认为B比C好，B认为C比D好，C又认为D比A好，形成闭环。论文在SummEval数据集上发现，尽管整体违规率很低（0.8-4.1%），但33-67%的文档至少存在一个3-循环，说明LLM在细微偏好上存在系统性不一致。

**分裂保角预测（Split Conformal Prediction）**：这是核心创新点。传统置信区间只告诉我们"答案对不对"，保角预测告诉我们"正确答案在哪些选项中"。具体步骤：
1. 将数据分为校准集和测试集
2. 在校准集上计算非一致性分数（LCrit = |score - ground_truth|）
3. 取(1-α)分位数q̂作为阈值
4. 对测试样本，预测集C(x) = {y: |score(x,y) - median| ≤ q̂}
5. 理论保证：P(y* ∈ C(x)) ≥ 1-α

论文发现预测集宽度与人类可靠性标注高度相关（rs=+0.576），且跨评判者一致（r̄=0.32-0.38），说明宽度确实捕获了"文档难度"而非"评判者噪声"。

| 跨评判者一致性分析器 | 计算不同LLM评判者在预测集宽度上的Spearman相关性，验证宽度捕获的是文档难度而非评判者特异性 |

### 💻 代码示例

```python
import numpy as np
from typing import List, Set, Dict
from collections import defaultdict

# ============================================================
# 1. 传递性分析：检测有向3-循环
# ============================================================

def detect_3cycles(judgments: Dict[str, Dict[str, float]]) -> List[tuple]:
    """
    检测LLM评判中的传递性违规（3-循环）
    
    参数:
        judgments: 模型A认为B优于C的得分 {model_A: {model_B: score, model_C: score}}
    返回:
        3-循环列表，如 [(A,B,C), ...]
    """
    models = list(judgments.keys())
    cycles = []
    
    for A in models:
        for B in models:
            if A == B:
                continue
            for C in models:
                if C in [A, B]:
                    continue
                # A > B > C > A 形成闭环
                if (judgments[A].get(B, 0) > judgments[B].get(A, 0) and  # A > B
                    judgments[B].get(C, 0) > judgments[C].get(B, 0) and  # B > C
                    judgments[C].get(A, 0) > judgments[A].get(C, 0)):    # C > A
                    cycles.append((A, B, C))
    
    return cycles


def compute_transitivity_violation_rate(judgments: Dict) -> float:
    """计算传递性违规率"""
    cycles = detect_3cycles(judgments)
    n_models = len(judgments)
    n_pairs = n_models * (n_models - 1)
    return len(cycles) / max(n_pairs, 1)


# ============================================================
# 2. 分裂保角预测 (Split Conformal Prediction)
# ============================================================

class SplitConformalPredictor:
    """
    简化版分裂保角预测器
    核心思想：不预测单一答案，而是预测"正确答案在哪些选项中"
    """
    
    def __init__(self, alpha: float = 0.1):
        """
        参数:
            alpha: 期望的错误率（如0.1表示10%的概率漏掉正确答案）
        """
        self.alpha = alpha
        self.threshold = None
        self.median = None
    
    def fit(self, calibration_scores: np.ndarray, 
            ground_truth: np.ndarray):
        """
        步骤1-3: 在校准集上计算阈值
        
        参数:
            calibration_scores: 校准集上的模型得分
            ground_truth: 校准集的真实得分
        """
        # 计算非一致性分数 (LCrit = |score - ground_truth|)
        nonconformity_scores = np.abs(calibration_scores - ground_truth)
        
        # 计算中位数
        self.median = np.median(ground_truth)
        
        # 取 (1 - alpha) 分位数作为阈值
        n = len(nonconformity_scores)
        self.threshold = np.quantile(
            nonconformity_scores, 
            np.ceil((n + 1) * (1 - self.alpha)) / n
        )
        return self
    
    def predict(self, test_scores: np.ndarray, 
                candidates: List) -> List[Set]:
        """
        步骤4-5: 对测试样本生成预测集
        
        返回:
            每个测试样本的候选答案集合
        """
        prediction_sets = []
        
        for score in test_scores:
            # C(x) = {y: |score(x,y) - median| ≤ threshold}
            valid_candidates = {
                cand for cand in candidates
                if abs(score - self.median) <= self.threshold
            }
            prediction_sets.append(valid_candidates)
        
        return prediction_sets
    
    def coverage(self, prediction_sets: List[Set], 
                 true_labels: List) -> float:
        """
        计算覆盖率：正确答案在预测集中的比例
        理论保证: ≥ 1 - alpha
        """
        hits = sum(
            true in pred_set 
            for true, pred_set in zip(true_labels, prediction_sets)
        )
        return hits / len(true_labels)


# ============================================================
# 3. 演示示例（伪代码风格）
# ============================================================

def demo_split_conformal():
    """演示分裂保角预测的完整流程"""
    
    # 模拟数据
    np.random.seed(42)
    
    # 假设有10个文档，每个文档有5个候选摘要评分
    n_calibration = 100
    n_test = 20
    n_candidates = 5
    
    # 校准集
    cal_scores = np.random.randn(n_calibration) * 2 + 3  # 均值3，std 2
    cal_truth = np.random.randn(n_calibration) * 2 + 3
    
    # 测试集
    test_scores = np.random.randn(n_test) * 2 + 3
    test_truth = np.random.randn(n_test) * 2 + 3
    
    candidates = [f"candidate_{i}" for i in range(n_candidates)]
    
    # 训练预测器
    predictor = SplitConformalPredictor(alpha=0.1)
    predictor.fit(cal_scores, cal_truth)
    
    # 预测
    pred_sets = predictor.predict(test_scores, candidates)
    
    # 评估
    coverage = predictor.coverage(pred_sets, test_truth)
    
    # 计算平均预测集宽度（与人类可靠性相关）
    avg_width = np.mean([len(s) for s in pred_sets])
    
    print(f"阈值 (q_hat): {predictor.threshold:.3f}")
    print(f"覆盖率: {coverage:.2%}")  # 理论上 ≥ 90%
    print(f"平均预测集宽度: {avg_width:.1f}")
    print(f"宽度越大 → 文档越难 → 人类评判越不可靠")


if __name__ == "__main__":
    # 1. 演示传递性分析
    judgments = {
        "A": {"B": 0.6, "C": 0.4},
        "B": {"C": 0.7, "A": 0.3},
        "C": {"A": 0.5, "B": 0.2}
    }
    
    cycles = detect_3cycles(judgments)
    print(f"检测到的3-循环: {cycles}")
    
    # 2. 演示分裂保角预测
    demo_split_conformal()
```

## ❓ 常见问题

**Q：为什么传递性违规率很低（0.8-4.1%）却说明LLM不可靠？**

A：因为宏观违规率是聚合指标，掩盖了微观不一致性。33-67%的文档存在至少一个3-循环，说明在具体样本上LLM经常自相矛盾——今天说A比B好，明天说B比A好。0.8%是全局平均值，但分布极不均匀，某些文档可能100%违规。类比：95%的癌症检测准确率听起来很高，但如果漏检了所有早期癌症患者，仍是灾难性的。


**Q：保角预测和传统置信区间有什么区别？**

A：传统置信区间告诉你单个预测值的可信范围（如"分数在3.2-3.8之间"）。保角预测告诉你正确答案在哪些选项中（如"正确答案在{3,4}里"）。更关键的是，保角预测有理论保证：无论数据分布、模型复杂度如何，P(y*∈C(x))≥1-α，而传统方法依赖假设检验的前提条件。


**Q：这个研究对实际应用有什么直接帮助？**

A：开发者可以：1)用预测集宽度过滤低可靠性评估，自动转人工复核；2)识别LLM评判的"困难样本"，针对性改进；3)在合规场景（如自动审批）用覆盖率保证做风险控制；4)对比不同评判模型，选择更可靠的那个。
