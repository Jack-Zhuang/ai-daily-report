📖 背景与动机

随着自动驾驶从L2向L4级别演进，感知系统的安全性成为核心挑战。传统的目标检测模型在训练时假设测试集与训练集分布一致（闭集识别），但真实道路会遭遇各种未知障碍物（行人突然冲出、掉落货物、异形车辆等），这些「分布外」物体对基于闭集假设的感知系统构成严重威胁。异常检测（Anomaly Detection）应运而生，旨在让感知系统在保证正常物体检测性能的同时，识别出「不该出现的异常物体」。当前学术界已提出多种视觉异常检测（VAD）方法，但缺乏统一的评测基准，且现有数据集与真实驾驶场景存在显著差异。AD4AD正是为解决这一痛点而诞生的。

关键要点：
  • 自动驾驶需应对开放世界，感知系统必须识别训练集外的未知物体
  • 现有VAD方法缺乏统一评测基准，各研究使用不同数据集和指标，难以公平比较
  • 真实驾驶场景的异常检测面临光照变化、遮挡、快速运动等实际问题

💡 核心方法

方法概述

AD4AD是一个标准化基准平台，包含多样化驾驶场景数据集、统一评测指标及标准化测试流程。论文系统评估了包括Dinomaly在内的多种VAD方法，揭示了当前方法在自动驾驶场景下的实际性能边界。

详细设计

AD4AD基准平台的核心设计理念是「贴近量产需求」。数据集方面，收集了多种天气（晴天、雨天、雾天）、时段（白天、黄昏、夜间）及场景（高速公路、城市道路、停车场）的真实驾驶数据，标注了正常物体与各类异常物体（如散落物、动物、奇怪载具等）。评测指标采用P-AP（Performance-Average Precision）这一复合指标，兼顾异常检测的精确率和召回率。

在方法层面，论文对比了基于重构的方法（如Autoencoder、GAN-based）、基于嵌入的方法（如OpenMax、Energy-based）、以及基于Transformer的方法。Dinomaly作为亮点方法，使用DeiT（Data-efficient Image Transformer）作为骨干网络，通过学习正常样本的特征分布来识别异常：当输入图像的特征与学习到的正常分布偏离过大时，判定为异常。实验部分详细报告了各方法在AD4AD上的P-AP、ROC-AUC、FPR95等指标，并提供了定性结果（异常热力图可视化），揭示了当前方法的优劣势。

class DeiTBackbone(nn.Module):

"""Data-efficient Image Transformer 特征提取器"""

def __init__(self):

super().__init__()

# 模拟Transformer编码器

self.encoder = nn.Sequential(

def forward(self, x):

# 提取特征向量 (简化版)

return features

==================== 3. Dinomaly 异常检测模型 ====================

class DinomalyDetector:

"""基于正常分布学习的异常检测"""

def __init__(self):

self.backbone = DeiTBackbone()

self.normal_mean = None # 正常特征分布的均值

self.normal_cov = None # 正常特征分布的协方差

def train_normal_distribution(self, normal_features):

"""学习正常样本的特征分布"""

self.normal_mean = normal_features.mean(dim=0)

# 计算协方差矩阵（简化：使用单位矩阵）

self.normal_cov = torch.eye(normal_features.shape[1])

def compute_anomaly_score(self, features):

"""计算马氏距离作为异常分数"""

diff = features - self.normal_mean

# 简化：使用欧氏距离代替马氏距离

scores = torch.norm(diff, dim=1)

return scores

def predict(self, features, threshold=None):

"""判断是否为异常"""

scores = self.compute_anomaly_score(features)

if threshold is None:

# 自适应阈值

threshold = scores.mean() + scores.std()

return scores > threshold, scores

==================== 4. 评估指标 ====================

def compute_metrics(scores, labels):

"""计算 P-AP, ROC-AUC, FPR95"""

# 简化实现

from sklearn.metrics import roc_auc_score, average_precision_score

# 转换

labels = labels.cpu().numpy()

scores = scores.cpu().numpy()

# ROC-AUC

auc = roc_auc_score(labels, scores)

# P-AP (Precision-Recall AUC)

ap = average_precision_score(labels, scores)

# FPR@95 (False Positive Rate at 95% TPR)

# 简化：计算假阳性率

threshold = sorted(scores, reverse=True)[int(len(scores) * 0.05)]

fpr95 = (scores[labels == 0] > threshold).mean()

return {'P-AP': ap, 'ROC-AUC': auc, 'FPR95': fpr95}

❓ 常见问题

Q：为什么自动驾驶需要专门做异常检测？用目标检测不行吗？

A：目标检测只能识别训练集中定义的类别（如车、人、交通标志），对于训练时从未见过的异常物体（如散落轮胎、建筑垃圾）会完全「看不见」导致漏检。异常检测旨在识别「未知/异常」，与目标检测形成安全冗余。

Q：AD4AD相比其他异常检测数据集有什么优势？

A：现有数据集（如FS、StreetAnomaly）场景单一、规模小，且评测指标不统一。AD4AD提供了多样化场景覆盖（天气/时段/道路类型）和标准化的P-AP指标，更贴近真实自动驾驶的评测需求。

Q：Dinomaly中的DeiT为什么比ResNet等CNN骨干效果好？

A：DeiT等Transformer预训练模型在大规模图像数据上学习了更丰富的视觉表示，特征具有更强的泛化性和判别性，能够更好地区分「见过的正常样本」与「未见过的新样本」。

Q：异常检测能直接用于量产吗？

A：目前还不行。主要瓶颈在于：1）异常物体千变万化，检测召回率有限；2）恶劣天气下性能急剧下降；3）实时推理对算力要求高。建议作为安全冗余模块，辅助而非替代现有感知系统。
