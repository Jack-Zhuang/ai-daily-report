📖 背景与动机

联邦学习（FL）作为隐私保护下协同训练的主流范式，允许多个客户端在不共享原始数据的情况下联合优化模型。然而，现实场景中分布式客户端的数据质量参差不齐，标签噪声广泛存在且难以检测，严重制约模型性能。传统方法主要依赖两类策略：一是设计抗噪损失函数（如对称损失），二是利用训练过程中的损失动态（如自步学习）。这些方法通常假设全局噪声分布均匀，且需要大量本地迭代才能见效，对异构噪声场景的适应性有限。更重要的是，现有方法难以区分「噪声样本」与「噪声客户端」，无法针对不同数据质量的客户端采取差异化策略。FedSIR正是针对这一痛点，提出利用深度特征天然具备的类内聚合性与类间分离性，通过谱分析技术从特征几何结构层面识别噪声源。

关键要点：
  • 联邦学习中噪声标签问题普遍存在且严重制约模型性能
  • 现有方法多聚焦于损失函数设计或训练动态分析，忽视特征空间的几何结构
  • 不同客户端的数据质量差异显著，需要区分处理

💡 核心方法

方法概述

FedSIR是一个三阶段的多层次框架，通过谱分析技术依次完成客户端识别、标签重标记与噪声感知训练。核心思想是利用深度网络学习到的特征表示在类维度上呈现的谱结构差异来刻画数据质量。

详细设计

第一阶段为谱一致性分析。每个客户端在本地训练后，计算其特征表示的类相似矩阵S_k。具体而言，对每个类别c，提取属于该类的特征构成矩阵，然后计算其右奇异向量v_{k,c}作为该类的「主导方向」。两个类别c1和c2的相似度定义为这两个主导方向的内积绝对值。干净客户端由于数据标注准确，其类内特征分布紧凑、类间分离清晰，因此类相似矩阵的对角线元素大、非对角线元素小。论文进一步定义了「off-diagonal mean」和「off-diagonal energy」两个度量，前者反映平均类间混淆程度，后者衡量这种混淆的离散程度。服务器聚合所有客户端的这两个统计量后，通过简单的阈值即可将客户端分为干净与噪声两类。

第二阶段为谱引导重标记。对于被识别为噪声的客户端，FedSIR利用干净客户端提供的谱参考进行标签纠正。每个类k维护一个主导方向v_k（由干净客户端的平均得到），噪声客户端的每个样本特征x被投影到这些方向上，通过softmax获得软标签。具体而言，计算x与各v_k的内积，经过温度归一化后得到重标记概率。这一过程同时利用了主导类方向（捕获主要变异方向）和残差子空间（捕获类内细节），实现了对噪声标签的精准修正。

第三阶段为噪声感知训练。修正后的标签并非直接替换原始标签，而是作为软目标参与训练。论文采用蒸馏风格的损失函数，将重标记分布作为监督信号的一部分，结合传统的交叉熵损失，实现对残留噪声的鲁棒训练。

🔧 关键组件

| 组件 | 说明 |

|------|------|

| 类相似矩阵 (Class Similarity Matrix) | 基于主导奇异向量构建的C×C矩阵，对角线元素接近1（类内紧凑），非对角线元素反映类间混淆程度，是区分干净/噪声客户端的核心指标 |

| 谱引导重标记 (Spectral-Guided Relabeling) | 利用干净客户端的特征谱结构（主导方向+残差子空间）作为参考，为噪声客户端的样本预测更可靠的标签分布 |

| 噪声感知训练 (Noise-Aware Training) | 将重标记分布作为软目标，结合硬标签进行混合监督训练，容忍一定程度的标签不确定性 |

def compute_class_dominant_direction(features, labels, k):

"""计算第k个类别的主导方向（右奇异向量）"""

# 提取属于类别k的特征

class_features = features[labels == k]

# 简化：使用特征均值作为主导方向

# 完整实现应使用SVD分解获取右奇异向量

dominant_dir = class_features.mean(axis=0)

return dominant_dir / (np.linalg.norm(dominant_dir) + 1e-8)

def compute_similarity_matrix(client_features, client_labels, num_classes):

"""计算客户端的类相似矩阵S_k"""

similarity_matrix = np.zeros((num_classes, num_classes))

for c in range(num_classes):

v_c = compute_class_dominant_direction(client_features, client_labels, c)

for c2 in range(num_classes):

v_c2 = compute_class_dominant_direction(client_features, client_labels, c2)

# 相似度 = 主导方向内积的绝对值

similarity_matrix[c, c2] = np.abs(np.dot(v_c, v_c2))

return similarity_matrix

def compute_off_diagonal_metrics(S):

"""计算off-diagonal mean和off-diagonal energy"""

mask = ~np.eye(S.shape[0], dtype=bool) # 非对角线掩码

off_diagonal = S[mask]

off_diag_mean = np.mean(off_diagonal) # 平均类间混淆程度

off_diag_energy = np.mean(off_diagonal**2) # 混淆离散程度

return off_diag_mean, off_diag_energy

def classify_clients(all_clients_stats, threshold_mean=0.1, threshold_energy=0.01):

"""服务器聚合统计量，判断客户端干净/噪声"""

clean_clients = []

noisy_clients = []

for client_id, (mean, energy) in enumerate(all_clients_stats):

if mean < threshold_mean and energy < threshold_energy:

clean_clients.append(client_id)

else:

noisy_clients.append(client_id)

return clean_clients, noisy_clients

def compute_clean_reference_directions(clean_clients, all_features, all_labels, num_classes):

"""从干净客户端聚合得到各类别的主导方向"""

reference_dirs = np.zeros((num_classes, feature_dim))

for k in range(num_classes):

# 收集所有干净客户端的类别k主导方向

dirs = []

for client_id in clean_clients:

d = compute_class_dominant_direction(

all_features[client_id], all_labels[client_id], k)

dirs.append(d)

# 平均得到参考方向

reference_dirs[k] = np.mean(dirs, axis=0)

reference_dirs[k] /= (np.linalg.norm(reference_dirs[k]) + 1e-8)

return reference_dirs

def relabel_samples(noisy_features, noisy_labels, reference_dirs, temperature=0.1):

"""基于谱参考方向对噪声客户端样本进行软重标记"""

num_samples = noisy_features.shape[0]

num_classes = reference_dirs.shape[0]

soft_labels = np.zeros((num_samples, num_classes))

for i, x in enumerate(noisy_features):

x = x / (np.linalg.norm(x) + 1e-8)

# 计算样本与各类别主导方向的相似度

similarities = np.array([np.dot(x, v) for v in reference_dirs])

# 温度归一化的softmax

exp_sim = np.exp(similarities / temperature)

soft_labels[i] = exp_sim / np.sum(exp_sim)

# 选择概率最大的类作为伪标签

pseudo_labels = np.argmax(soft_labels, axis=1)

return pseudo_labels, soft_labels

def distillation_loss(student_logits, soft_labels, original_labels, alpha=0.5, T=2):

"""蒸馏风格损失：软目标 + 硬目标"""

# 软目标损失（KL散度）

soft_student = student_logits / T

soft_loss = np.sum(soft_labels * np.log(soft_labels / (soft_student + 1e-8)))

# 硬目标损失（交叉熵）

hard_loss = -np.sum(np.eye(num_classes)[original_labels] *

np.log(student_logits + 1e-8))

# 加权组合

total_loss = alpha soft_loss + (1 - alpha) hard_loss

return total_loss

def noisy_aware_training(model, data, original_labels, soft_targets, epochs=10):

"""使用软标签进行噪声感知训练"""

for epoch in range(epochs):

# 前向传播

logits = model(data)

# 计算蒸馏损失

loss = distillation_loss(logits, soft_targets, original_labels)

# 反向传播更新（伪代码）

❓ 常见问题

Q：为什么谱结构能够反映标签质量？

A：干净标注的数据在特征空间中类内紧凑、类间分离，对应的类相似矩阵对角线元素大、非对角线小；噪声标注会破坏这种几何结构，导致类间混淆增加。

Q：FedSIR的通信开销如何？

A：极低。每个客户端只需向服务器上传两个标量（off-diagonal mean和energy），不涉及原始特征或梯度传输，通信量可忽略不计。

Q：阈值如何确定？

A：论文未详述，但通常可基于历史数据或验证集确定，也可采用自适应阈值如均值加减k倍标准差。阈值对最终性能有一定影响，但框架整体较为鲁棒。
