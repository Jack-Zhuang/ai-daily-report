📖 背景与动机

随着ChatGPT、Claude、Gemini等大语言模型能力持续提升，如何科学评估其「好与坏」成为关键挑战。现有的主流基准测试（如MMLU、HellaSwag）依赖固定数据集，采用聚合评分排名，实质上假设「所有用户有相同偏好」。然而，真实场景中用户背景迥异：有人偏爱逻辑严谨的解释，有人欣赏幽默风趣的表达；有人需要专业代码，有人想要温暖对话。Chatbot Arena通过众包对战收集了大量用户偏好数据，但最终仍汇总为单一排名。本研究首次系统揭示：聚合排名究竟在多大程度上代表了真实用户的个性化需求？

关键要点：
  • 现有LLM基准测试依赖聚合评分，隐含假设所有用户偏好相同
  • Chatbot Arena虽收集了用户级对战数据，但未深入挖掘个体差异
  • 用户查询的主题多样性和写作风格差异可能导致模型偏好显著分化
  • 亟需建立个性化评估范式，支撑精准的模型选择与推荐

💡 核心方法

方法概述

本研究从Chatbot Arena抽取115名活跃用户的历史对战数据，构建个性化排名系统。采用两种经典评分机制——ELO和Bradley-Terry——分别建模用户对模型的偏好，并通过话题建模和风格分析挖掘用户特征与排名变化的关联。

详细设计

研究首先从Chatbot Arena获取115名用户的完整对战记录，每条记录包含用户ID、两个模型A/B的响应、用户选择偏好及查询文本。随后，对每个用户独立运行ELO评分系统：初始化所有模型基准分为1500，根据用户对战结果按公式更新评分——若用户选择模型A且A获胜，则A获得K(1-Ea)分，B获得K(0-Eb)分，其中Ea=1/(1+10^((ELOb-ELOa)/400))为A的期望胜率。

同时，对每对模型(m_a, m_b)拟合Bradley-Terry模型，估计用户u对两模型的偏好强度参数β_u,m。模型i被选中的概率为P(i wins) = 1/(1+exp(-(β_i - β_j)))，通过极大似然估计参数。

为分析用户特征与排名差异的关联，研究团队使用LDA主题模型（20-50个话题）对用户查询进行向量化，并提取LISA风格维度（包括正式度、创意性、结构化程度等），最终计算用户特征向量与排名变化Spearman相关系数。

🔧 关键组件

| 组件 | 说明 |

|------|------|

| 用户级ELO评分系统 | 为每个用户独立维护一套模型评分，评分更新公式为ELOu(m_a) ← ELOu(m_a) + K(1 - E_a)，其中E_a = 1/(1+10^((ELOb-ELOa)/400))。K值为学习率，控制单次对战的影响权重。 |

| Bradley-Terry偏好模型 | 将用户u对模型m的选择建模为概率问题：P(选m_a|对战m_a,m_b) = 1/(1+exp(-(β_u,m_a - β_u,m_b)))。通过最大化似然函数估计每个用户的偏好参数β_u,m，可捕捉非线性偏好结构。 |

| LISA风格分析框架 | 从用户查询中提取多维风格特征，包括正式程度、创意性、结构化程度、比喻/类比使用频率等。通过PCA降维后保留主要成分，用于后续相关性分析。 |

| 话题混合分布模型 | 使用LDA（潜在狄利克雷分配）将用户查询映射到K个话题的概率分布，K通过困惑度（perplexity）优化确定。话题分布反映用户的兴趣多样性。 |

class SimpleEloSystem:

def __init__(self, models, k=32, base_score=1500):

self.models = models

self.k = k

self.elo = {m: base_score for m in models}

def expected_win_rate(self, elo_a, elo_b):

"""计算A对B的期望胜率"""

return 1 / (1 + 10 ** ((elo_b - elo_a) / 400))

def update_ratings(self, model_a, model_b, winner):

"""根据对战结果更新评分"""

elo_a = self.elo[model_a]

elo_b = self.elo[model_b]

# 计算期望胜率

ea = self.expected_win_rate(elo_a, elo_b)

eb = 1 - ea # B的期望胜率

# 根据胜负更新评分

if winner == 'A':

self.elo[model_a] += self.k * (1 - ea)

self.elo[model_b] += self.k * (0 - eb)

else:

self.elo[model_a] += self.k * (0 - ea)

self.elo[model_b] += self.k * (1 - eb)

def run_user_matches(self, user_matches):

"""运行单个用户的所有对战记录"""

for match in user_matches:

model_a, model_b, winner = match

self.update_ratings(model_a, model_b, winner)

return self.elo.copy()

def bradley_terry_likelihood(params, pairs, outcomes):

"""Bradley-Terry极大似然函数（简化版）"""

beta = params # 模型偏好参数

# P(i wins) = 1 / (1 + exp(-(β_i - β_j)))

log_lik = 0

for (i, j), y in zip(pairs, outcomes):

prob = 1 / (1 + np.exp(-(beta[i] - beta[j])))

log_lik += y np.log(prob + 1e-10) + (1 - y) np.log(1 - prob + 1e-10)

return -log_lik # 负对数似然

def fit_bradley_terry(pairs, outcomes, n_models):

"""拟合Bradley-Terry模型"""

# pairs: [(model_a, model_b), ...]

# outcomes: [1 if a wins else 0, ...]

init_params = np.zeros(n_models)

result = minimize(

bradley_terry_likelihood,

init_params,

args=(pairs, outcomes),

method='BFGS'

)

return result.x # 返回每个模型的β参数

❓ 常见问题

Q：为什么Bradley-Terry相关性（ρ=0.04）远低于ELO相关性（ρ=0.43）？

A：两者建模层次不同：ELO基于胜负结果的数值评分，适合捕捉「谁更强」的Ordinal关系；Bradley-Terry建模选择概率，能捕捉微妙偏好差异。当用户偏好与模型真实实力不一致时，BT的零相关恰好揭示了「用户可能故意选择弱者」的悖论，而ELO的平滑机制会稀释这一信号。

Q：普通开发者如何应用这一研究？

A：有三条路径：(1) 直接使用论文开源代码为你的用户群构建个性化排名；(2) 在产品中加入A/B测试框架，记录用户选择偏好，用ELO实时评估不同模型表现；(3) 使用论文的LISA框架分析你的目标用户写作风格，针对性微调或选择模型。

Q：个性化评估会增加多少计算开销？

A：核心开销在参数估计：Bradley-Terry需要对每个用户拟合独立模型，若有N个用户、M个模型，计算复杂度为O(N×M²×迭代次数)。但稀疏对战矩阵下可用贝叶斯压缩感知加速。ELO更新为O(1)，适合实时系统。实践中建议采用混合策略：全局排名用ELO，top-K个性化用BT。

Q：研究有哪些局限性？

A：主要有三点：(1) 样本偏向Chatbot Arena的科技爱好者用户，结论未必推广至普通大众；(2) 仅分析显式选择偏好，未捕捉用户满意度、使用时长等隐式反馈；(3) 未考虑模型组合场景——用户可能同时使用多个模型，单一排名过于简化。
