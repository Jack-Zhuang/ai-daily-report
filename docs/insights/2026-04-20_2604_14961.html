## 📖 背景与动机

上下文老虎机（Contextual Bandits）是推荐系统、广告投放等场景的核心算法，其核心挑战在于平衡探索与利用。然而，在冷启动阶段，由于缺乏足够的观测数据，学习器难以准确区分优质与劣质选项（arm），导致 regret（后悔值）居高不下。传统方法如LinUCB通过探索bonus缓解此问题，但探索本身也会带来收益损失。近年来，大语言模型（LLM）展现出强大的世界知识和推理能力，能否借助LLM的先验知识来弥补冷启动阶段的数据不足？这一思路面临一个关键问题：LLM的预测并非总是可靠，如何自适应控制LLM信息的影响力？


**关键要点：**

- 上下文老虎机在冷启动阶段因数据稀疏导致高regret
- 传统LinUCB依赖随机探索，数据效率受限
- LLM具备领域知识，可预测未探索arm的反事实奖励
- LLM预测存在误差，需自适应机制控制其影响权重

## 💡 核心方法

### 方法概述

本文在Disjoint LinUCB基础上，引入LLM伪观测机制。通过校准门控的指数移动平均衰减计划，动态调整伪观测权重：预测准确时权重上升，预测失误时权重衰减。


### 详细设计

论文的核心方法包含三个关键组件：

1. **LinUCB基础架构**：采用Disjoint LinUCB作为基线算法，对每个arm维护独立的线性模型。在标准UCB公式基础上，对每个arm的奖励上界进行估计。

2. **LLM伪观测生成**：每轮交互后，将未选中的arm及其上下文输入LLM，生成反事实奖励预测。这些预测作为伪观测注入学习器，形式化为带权重w的观测样本。真实观测使用w=1，伪观测使用自适应权重wt<1。

3. **校准门控机制**（Calibration-Gated Decay）：这是本文的核心创新。定义校准误差为预测值与真实值之差的绝对值，使用指数移动平均（EMA）追踪LLM的预测准确性。权重衰减公式为：wt = λ * exp(-decay_t * calibration_error)，其中λ为基准权重，decay_t控制衰减速率。高校准误差时wt趋近于0，抑制不准确预测的影响；低误差时wt保持较高值，使LLM知识充分贡献。

4. **平滑机制**：引入平滑参数β∈[0,1)，对校准误差进行时间平滑，避免单次预测失误导致权重剧烈波动。


### 🔧 关键组件

| 组件 | 说明 |
|------|------|
| 伪观测注入器 | 接收LLM对未选arm的奖励预测，将预测值与当前权重wt打包为伪观测样本，注入LinUCB的观测池 |
| 校准误差追踪器 | 使用指数移动平均实时追踪LLM预测与真实奖励的偏差，平滑参数β控制历史信息权重 |
| 动态权重计算器 | 根据校准误差和衰减率τ计算当前伪观测权重wt，实现自适应门控 |

### 💻 代码示例

```python
import numpy as np

class CalibrationGatedLinUCB:
    """基于校准门控机制的LinUCB算法"""
    
    def __init__(self, n_arms, dim_context, lambda_param=0.5, 
                 decay_rate=0.1, beta=0.3, llm=None):
        self.n_arms = n_arms
        self.dim_context = dim_context
        self.lambda_param = lambda_param  # 基准权重
        self.decay_rate = decay_rate       # 衰减速率
        self.beta = beta                    # 平滑参数
        self.llm = llm                      # LLM接口
        
        # ==================== LinUCB 基础架构 ====================
        # 每个arm维护独立的特征矩阵A和向量b
        self.A = [np.eye(dim_context) for _ in range(n_arms)]
        self.b = [np.zeros(dim_context) for _ in range(n_arms)]
        
        # 校准误差的EMA追踪
        self.calibration_ema = 0.0
        self.t = 0
    
    def _compute_ucb(self, arm, context):
        """计算指定arm的UCB值"""
        # 线性估计: theta = A^-1 @ b
        theta = np.linalg.solve(self.A[arm], self.b[arm])
        pred = np.dot(context, theta)
        
        # 置信上界
        A_inv = np.linalg.inv(self.A[arm])
        uncertainty = np.sqrt(np.dot(context, A_inv @ context))
        ucb = pred + uncertainty
        
        return ucb, pred
    
    def select_arm(self, context):
        """选择arm: 基于UCB最大原则"""
        ucb_values = []
        for arm in range(self.n_arms):
            ucb, _ = self._compute_ucb(arm, context)
            ucb_values.append(ucb)
        
        return np.argmax(ucb_values)
    
    def _compute_calibration_error(self, arm, context, real_reward):
        """计算校准误差: |预测值 - 真实值|"""
        _, pred_reward = self._compute_ucb(arm, context)
        return abs(pred_reward - real_reward)
    
    def _update_calibration_ema(self, calibration_error):
        """更新校准误差的指数移动平均"""
        self.calibration_ema = (self.beta * calibration_error + 
                                 (1 - self.beta) * self.calibration_ema)
    
    def _compute_weight(self):
        """计算伪观测权重: wt = λ * exp(-decay_t * calibration_error)"""
        weight = self.lambda_param * np.exp(-self.decay_rate * self.calibration_ema)
        return np.clip(weight, 0, 1)  # 限制在[0,1]范围
    
    def update(self, arm, context, reward):
        """更新真实观测 (权重w=1)"""
        # LinUCB标准更新
        self.A[arm] += np.outer(context, context)
        self.b[arm] += reward * context
        
        # 更新校准误差追踪
        cal_error = self._compute_calibration_error(arm, context, reward)
        self._update_calibration_ema(cal_error)
        
        self.t += 1
    
    def generate_counterfactual(self, selected_arm, context):
        """LLM伪观测生成"""
        if self.llm is None:
            return None
        
        counterfactuals = []
        for arm in range(self.n_arms):
            if arm == selected_arm:
                continue
            
            # 构建反事实查询
            query = f"Context: {context}, Arm: {arm}, 预测奖励"
            cf_reward = self.llm.predict(query)  # 调用LLM
            
            # 计算该arm的伪观测权重
            weight = self._compute_weight()
            
            counterfactuals.append({
                'arm': arm,
                'context': context,
                'reward': cf_reward,
                'weight': weight  # wt < 1
            })
        
        return counterfactuals
    
    def update_with_counterfactual(self, cf_samples):
        """注入伪观测样本 (带权重wt)"""
        for cf in cf_samples:
            arm = cf['arm']
            ctx = cf['context']
            reward = cf['reward']
            weight = cf['weight']
            
            # 加权更新: A += w * x*x^T, b += w * r * x
            self.A[arm] += weight * np.outer(ctx, ctx)
            self.b[arm] += weight * reward * ctx


# ==================== 使用示例（伪代码） ====================
if __name__ == "__main__":
    # 初始化
    n_arms = 5
    dim_context = 10
    
    # 模拟LLM接口
    class DummyLLM:
        def predict(self, query):
            return np.random.randn()  # 随机生成伪奖励
    
    agent = CalibrationGatedLinUCB(
        n_arms=n_arms,
        dim_context=dim_context,
        lambda_param=0.5,   # λ
        decay_rate=0.1,    # decay_t
        beta=0.3,          # β
        llm=DummyLLM()
    )
    
    # 模拟交互过程
    for t in range(100):
        context = np.random.randn(dim_context)
        
        # 1. 选择arm
        chosen_arm = agent.select_arm(context)
        
        # 2. 获取真实奖励
        real_reward = np.random.randn()
        
        # 3. 更新真实观测
        agent.update(chosen_arm, context, real_reward)
        
        # 4. 生成并注入伪观测
        cf_samples = agent.generate_counterfactual(chosen_arm, context)
        if cf_samples:
            agent.update_with_counterfactual(cf_samples)
```

## ❓ 常见问题

**Q：为什么在UCI Mushroom上LLM增强反而增加了regret？**

A：可能原因包括：(1) Mushroom是简单2臂任务，基线已足够有效；(2) LLM对蘑菇毒性分类的直觉可能与数据分布不一致；(3) 非对称奖励结构下，错误预测的代价不对称放大。


**Q：如何确定校准门控的参数（λ、τ、β）？**

A：论文建议通过小规模线上实验或历史数据回测进行网格搜索。实践中可从保守参数开始（低λ、高τ），观察伪观测的实际贡献后逐步调整。核心原则是确保早期阶段LLM不主导模型更新。


**Q：该方法与传统的探索策略（如ε-greedy、Boltzmann）有何本质区别？**

A：传统方法通过随机探索获取真实反馈，而本文利用LLM知识直接生成伪反馈，本质上是将探索成本转移至LLM推理。校准门控机制则相当于对LLM知识的质量进行实时审计，决定何时信任这一转移。
