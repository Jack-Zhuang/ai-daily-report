## 💡 核心方法

### 方法概述

CPGRec+框架通过费舍尔分布指导评分符号决策，结合信息容量体积评估机制，利用图神经网络聚合多阶邻居信息，实现平衡导向的个性化推荐。


### 详细设计

框架核心包含四大组件：首先，费舍尔分布建模模块通过统计检验确定评分的置信度方向，将高风险低置信度评分降权；其次，信息容量体积评估模块计算用户-游戏交互的信息增益，优先推荐能提供最大新信息的游戏；第三，时空感知图卷积网络(SAGCN)层叠设计，深层节点聚合远距离游戏特征，缓解冷启动问题；最后，游戏提示(Game Prompt)模块融合游戏元数据与统计特征，强化语义理解。框架采用对比学习策略，通过设计正负样本对，区分高价值长尾推荐与噪声交互。训练过程中，模型同时优化预测准确性与公平性指标，确保热门与冷门游戏的推荐比例趋于均衡。


### 🔧 关键组件

| 组件 | 说明 |
|------|------|
| 费舍尔分布决策模块 | 基于费舍尔统计量判断评分可信度，对极端或异常评分进行符号修正，降低噪声影响 |
| 信息容量体积评估 | 借鉴费舍尔信息理论，量化用户交互的信息增量，优先推荐能消除不确定性的游戏 |
| 时空感知图卷积网络 | 多层堆叠结构，深层关注远距离游戏节点，浅层捕获局部邻居特征 |
| 游戏提示工程模块 | 融合平均评分、游戏标签、玩家数等统计特征作为辅助输入，增强语义表征 |

class ContrastiveLearning:
    """对比学习策略 - 区分长尾与噪声"""
    
    def __init__(self, temperature=0.1):

    def create_pairs(self, high_value_samples, noise_samples):

    def contrastive_loss(self, z_i, z_j, is_positive):
        """对比损失"""
        similarity = np.dot(z_i, z_j) / self.temp
        
        if is_positive:
            return -np.log(np.exp(similarity) / np.sum(np.exp(similarity)))
        else:
            return np.log(1 - np.exp(similarity))


class FairMultiTaskOptimizer:
    """公平性多目标优化器"""
    
    def __init__(self, alpha=0.7):
        self.alpha = alpha  # 准确性与公平性权重平衡
    
    def compute_accuracy_loss(self, predictions, targets):
        """预测准确性损失"""
        return np.mean((predictions - targets) ** 2)
    
    def compute_fairness_loss(self, hot_recs, cold_recs):
        """公平性损失 - 热门/冷门游戏推荐比例均衡"""

        return ratio_diff

        """联合优化目标"""

        total_loss = self.alpha * acc_loss + (1 - self.alpha) * fair_loss

## ❓ 常见问题

**Q：费舍尔分布与普通正态分布在推荐系统中的核心区别是什么？**

A：费舍尔分布描述的是样本方差/总体方差的分布特性，更适合处理有限样本下的参数估计不确定性，能为评分置信度提供统计检验依据。


**Q：框架如何避免过度推荐冷门游戏导致用户体验下降？**

A：通过多目标优化同时约束准确性与公平性指标，设置曝光比例上下界，确保推荐结果在用户兴趣相关性与内容多样性间保持合理平衡。


**Q：SAGCN与标准GCN相比有何改进？**
