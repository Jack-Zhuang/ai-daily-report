## 📖 背景与动机

序列推荐系统旨在根据用户历史行为序列预测其下一步可能感兴趣的内容。传统方法如马尔可夫链和因子分解机难以捕捉复杂的序列依赖关系。近年来，基于Transformer的序列编码器在推荐领域取得显著进展，但面临两个核心挑战：一是仅依赖ID特征的表示学习容易过拟合于训练数据分布；二是忽视用户行为序列中潜在的图结构关系（如物品共现模式）。此外，标注数据有限导致监督信号不足，对比学习虽能缓解这一问题，但现有方案大多采用单一视图，难以全面刻画用户兴趣。因此，如何有效融合多种视图的互补信息，成为提升序列推荐性能的关键突破口。


**关键要点：**

- 序列推荐需捕捉用户行为的时序依赖和复杂交互模式
- 单一ID视图容易过拟合，需要引入结构化先验知识
- 多视图学习可从不同角度挖掘用户兴趣，实现信息互补

## 💡 核心方法

### 方法概述

MVCrec提出一种新颖的多视图对比学习框架，核心思想是将序列推荐任务分解为ID视图和图视图两个互补的视角，通过对比学习提取视图内和视图间的协同信号，并利用多视图注意力融合模块动态整合不同视图的预测结果。


### 详细设计

该框架首先构建用户行为序列的两种视图表示：ID视图直接使用物品ID嵌入，通过Transformer编码器捕捉序列中的位置信息和上下文依赖；图视图则将序列中的物品视为节点，基于共现关系构建图结构，使用图神经网络（GNN）聚合邻域信息，提取高阶协同过滤信号。在对比学习阶段，框架设计了视图内对比损失（最大化同一视图不同增强版本的一致性）和视图间对比损失（对齐不同视图的表示空间）。关键创新在于多视图注意力融合模块，该模块根据当前输入序列的注意力分布，动态计算两个视图的融合权重，通过加权求和产生最终预测。形式化地，设ID视图输出为h_id，图视图输出为h_graph，融合过程为：h_fusion = α·h_id + (1-α)·h_graph，其中权重α由注意力网络学习得到。这种设计使模型能够自适应地为不同类型的序列分配置信度：简单序列可能更依赖ID视图的细粒度匹配，而复杂序列则受益于图视图的结构化推理。

### 🔧 关键组件

| 组件 | 说明 |
|------|------|
| ID视图编码器 | 基于Transformer的序列编码器，将物品ID序列映射为稠密向量，捕捉时序依赖和上下文信息 |
| 图视图编码器 | 基于GNN的图结构编码器，将物品共现关系建模为图，通过消息传递聚合高阶邻居信息 |
| 多视图注意力融合模块 | 通过注意力网络学习动态融合权重，根据序列特性自适应调整ID视图和图视图的贡献比例 |
| 对比学习目标 | 包含视图内和视图间两类对比损失，提升表示的鲁棒性和视图间的一致性 |

import torch
import torch.nn as nn

# ==================== 简化的多视图序列推荐框架 ====================

class ItemEmbedding(nn.Module):
    """物品ID嵌入层"""
    def __init__(self, num_items, embed_dim):
        super().__init__()
        self.embedding = nn.Embedding(num_items, embed_dim)
    
    def forward(self, item_ids):
        return self.embedding(item_ids)


class IDViewEncoder(nn.Module):
    """ID视图：Transformer编码器捕获位置和上下文信息"""
    def __init__(self, embed_dim, num_heads, num_layers):
        super().__init__()
        self.pos_embedding = nn.Embedding(512, embed_dim)  # 位置编码
        self.transformer = nn.TransformerEncoder(
            nn.TransformerEncoderLayer(d_model=embed_dim, nhead=num_heads),
            num_layers=num_layers
        )
    
    def forward(self, seq_embeddings):
        # seq_embeddings: [batch, seq_len, embed_dim]
        positions = torch.arange(seq_embeddings.size(1)).unsqueeze(0)
        seq_embeddings = seq_embeddings + self.pos_embedding(positions)
        
        # Transformer处理：[seq_len, batch, embed_dim]
        output = self.transformer(seq_embeddings.transpose(0, 1))
        return output.transpose(0, 1)  # [batch, seq_len, embed_dim]


class GraphViewEncoder(nn.Module):
    """图视图：基于共现关系构建图，使用GNN聚合"""
    def __init__(self, embed_dim):
        super().__init__()
        self.gnn_layer = nn.Linear(embed_dim, embed_dim)
    
    def build_cooccurrence_graph(self, item_ids, window_size=3):
        """基于共现关系构建邻接矩阵（伪代码）"""
        # 构建共现图的逻辑
        pass
    
    def forward(self, seq_embeddings, adj_matrix):
        # GNN消息传递：聚合邻域信息
        aggregated = torch.matmul(adj_matrix, seq_embeddings)
        return self.gnn_layer(aggregated)


class AttentionFusion(nn.Module):
    """多视图注意力融合模块"""
    def __init__(self, embed_dim):
        super().__init__()
        # 注意力网络学习融合权重α
        self.attention_net = nn.Sequential(
            nn.Linear(embed_dim * 2, embed_dim),
            nn.Tanh(),
            nn.Linear(embed_dim, 1)
        )
    
    def forward(self, h_id, h_graph):
        """
        h_id: ID视图输出
        h_graph: 图视图输出
        α = attention(id_view || graph_view)
        """
        # 拼接两个视图表示
        combined = torch.cat([h_id, h_graph], dim=-1)
        
        # 学习融合权重
        alpha = self.attention_net(combined)  # [batch, 1]
        alpha = torch.sigmoid(alpha)  # α ∈ (0, 1)
        
        # 加权融合
        h_fusion = alpha * h_id + (1 - alpha) * h_graph
        return h_fusion, alpha


class ContrastiveLoss(nn.Module):
    """对比学习损失"""
    
    def intra_view_loss(self, z1, z2):
        """视图内对比损失：同一视图不同增强版本的一致性"""
        return -torch.log(self._similarity(z1, z2) / (z1.norm() * z2.norm() + 1e-8))
    
    def inter_view_loss(self, h_id, h_graph):
        """视图间对比损失：对齐不同视图的表示空间"""
        return torch.abs(h_id - h_graph).mean()
    
    def _similarity(self, x, y):
        return (x * y).sum(dim=-1)


class MultiViewSequentialRecommender(nn.Module):
    """完整的多视图序列推荐框架"""
    
    def __init__(self, num_items, embed_dim=64, num_heads=4):
        super().__init__()
        self.embed_dim = embed_dim
        
        # 组件初始化
        self.item_embed = ItemEmbedding(num_items, embed_dim)
        self.id_encoder = IDViewEncoder(embed_dim, num_heads, num_layers=2)
        self.graph_encoder = GraphViewEncoder(embed_dim)
        self.fusion = AttentionFusion(embed_dim)
        self.contrastive = ContrastiveLoss()
        
        # 预测层
        self.predictor = nn.Linear(embed_dim, num_items)
    
    def forward(self, item_seq, adj_matrix=None, training=True):
        """
        item_seq: 用户行为序列
        adj_matrix: 图视图的邻接矩阵
        """
        # 1. 物品嵌入
        seq_embeddings = self.item_embed(item_seq)
        
        # 2. 两种视图编码
        h_id = self.id_encoder(seq_embeddings)           # ID视图
        h_graph = self.graph_encoder(seq_embeddings, adj_matrix)  # 图视图
        
        # 3. 注意力融合
        h_fusion, alpha = self.fusion(h_id, h_graph)
        
        # 4. 预测
        # 取序列最后一个位置或池化作为最终表示
        final_repr = h_fusion[:, -1, :]
        logits = self.predictor(final_repr)
        
        if training:
            # 5. 对比学习损失（伪代码）
            loss_intra = self.contrastive.intra_view_loss(h_id, h_graph)
            loss_inter = self.contrastive.inter_view_loss(h_id, h_graph)
            return logits, alpha, loss_intra + loss_inter
        
        return logits, alpha

    # 模拟输入数据
    item_seq = torch.randint(0, NUM_ITEMS, (BATCH_SIZE, SEQ_LEN))
    adj_matrix = torch.rand(BATCH_SIZE, NUM_ITEMS, NUM_ITEMS)  # 邻接矩阵
    
    # 前向传播
    logits, alpha, contrastive_loss = model(item_seq, adj_matrix, training=True)
    
    print(f"预测logits形状: {logits.shape}")        # [batch, num_items]
    print(f"融合权重α形状: {alpha.shape}")          # [batch, 1]
    print(f"对比损失: {contrastive_loss.item():.4f}")
    
    # 观察不同序列的融合权重
    print(f"\n自适应融合权重示例:")
    print(f"  简单序列可能 α较大 → 依赖ID视图")

## ❓ 常见问题

**Q：为什么需要多视图学习？单一视图有什么不足？**

A：单一ID视图依赖大量标注数据且容易过拟合，图视图虽能提供结构化先验但可能丢失细粒度偏好。多视图学习通过融合两种视角实现信息互补，ID视图负责精确匹配，图视图负责泛化推广，两者协同提升推荐效果。


**Q：多视图注意力融合模块是如何工作的？**

A：该模块接收用户序列的全局表示作为输入，通过一个小型MLP网络输出融合权重α。α表示对ID视图的信任度，(1-α)表示对图视图的信任度。最终预测为两个视图输出的加权和。这种设计使模型能自适应地为不同序列分配置信度。


**Q：该方法在工业场景中如何落地？**

A：主要挑战在于离线图构建和在线推理延迟。建议方案：1）使用Spark离线计算物品共现图并存储；2）预计算所有物品的图嵌入；3）线上服务仅需查表获取图嵌入，结合实时计算的ID嵌入和融合权重生成推荐。延迟可通过模型蒸馏压缩至10ms级别。


**Q：对比学习在其中的作用是什么？**

A：对比学习通过构造正负样本对学习泛化表示。视图内对比损失确保同一视图不同增强版本的一致性（如对序列做不同mask），视图间对比损失拉近不同视图对同一序列的表示距离。两项损失协同提升模型在稀疏数据上的鲁棒性。
