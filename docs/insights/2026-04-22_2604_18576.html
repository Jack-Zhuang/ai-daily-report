## 📖 背景与动机

事件概率预测在 geopolitics、商业决策等领域至关重要。当前主流方法存在两个痛点：一是将所有检索证据追加到不断膨胀的上下文中，导致LLM难以聚焦关键信息；二是多轮预测缺乏系统的聚合与校准机制，预测结果常出现过度自信或极端偏差。本文提出的BLF系统针对这两个问题给出系统性解决方案。

## 💡 核心方法

### 方法概述

BLF是一个迭代式工具调用Agent，通过三阶段Pipeline实现预测：信念初始化→证据检索与摘要→贝叶斯更新→多试验聚合→分层校准输出。


### 详细设计

系统核心是「贝叶斯语言信念状态」，每次迭代中，Agent检索新证据后，由LLM生成证据摘要（自然语言形式）和似然比估计（数值概率），两者共同构成更新后的信念状态，而非简单追加到上下文。

多层试验聚合（Hierarchical Multi-trial Aggregation）：独立运行K次试验（通常K=5），每次产生独立信念轨迹。在logit空间对K个预测进行加权聚合，权重由数据依赖的先验确定，实现「收缩估计」——远离极端值同时保留信号。

分层校准（Hierarchical Calibration）：采用Platt scaling但引入分层先验，避免对稀疏或偏斜数据源的极端预测过度收缩。例如某事件历史概率约5%，模型给出1%，分层校准会适度调整而非一刀切压缩。

### 🔧 关键组件

| 组件 | 说明 |
|------|------|
| 贝叶斯语言信念状态 | 半结构化表示，包含数值概率P和自然语言证据摘要E，形式化为(P, E)元组，支持迭代更新而非追加 |
| 多层试验聚合器 | 在logit空间对K个独立试验结果进行加权平均，权重由数据依赖先验计算，实现最优收缩估计 |
| 分层校准器 | 基于Platt scaling但加入源特定的先验分布，避免对偏斜基率的极端预测过度修正 |

### 💻 代码示例

```python
import numpy as np
from typing import List, Dict

# ========== 1. 贝叶斯语言信念状态 ==========
class BayesianLanguageBelief:
    """核心信念状态：证据摘要 + 似然比"""
    
    def __init__(self, prior_prob: float):
        self.prior = prior_prob  # 先验概率
        self.belief_state = {"prob": prior_prob, "summary": "", "evidence": []}
    
    def update_with_evidence(self, evidence: str, llm_output: Dict):
        """
        LLM生成：
        - evidence_summary: 自然语言证据摘要
        - likelihood_ratio: 似然比估计 (新证据对假设的影响)
        """
        summary = llm_output["summary"]
        lr = llm_output["likelihood_ratio"]  # P(E|H)/P(E|¬H)
        
        # 贝叶斯更新：post = prior * lr / normalization
        old_prob = self.belief_state["prob"]
        new_prob = (old_prob * lr) / (old_prob * lr + (1 - old_prob))
        
        self.belief_state = {
            "prob": new_prob,
            "summary": summary,
            "likelihood_ratio": lr
        }


# ========== 2. 多层试验聚合 ==========
class HierarchicalTrialAggregator:
    """K次独立试验 + logit空间加权聚合"""
    
    def __init__(self, k_trials: int = 5):
        self.k = k_trials
    
    def run_multiple_trials(self, prompt: str, llm_call) -> List[float]:
        """运行K次独立试验"""
        predictions = []
        for _ in range(self.k):
            # 每次独立调用LLM，可能有随机性
            result = llm_call(prompt)
            predictions.append(result["probability"])
        return predictions
    
    def aggregate(self, predictions: List[float], weights: List[float] = None) -> float:
        """
        Logit空间加权聚合（收缩估计）
        logit(p) = log(p / (1-p))
        """
        if weights is None:
            weights = np.ones(self.k) / self.k
        
        # 转换到logit空间
        logits = [np.log(p / (1 - p) + 1e-10) for p in predictions]
        
        # 加权平均
        weighted_logit = np.sum([w * l for w, l in zip(weights, logits)])
        
        # 转回概率空间
        aggregated_prob = 1 / (1 + np.exp(-weighted_logit))
        return aggregated_prob


# ========== 3. 分层校准 ==========
class HierarchicalCalibration:
    """Platt scaling + 分层先验，避免极端预测过度收缩"""
    
    def __init__(self, historical_prob: float):
        self.historical_prob = historical_prob  # 历史基准概率
        self.alpha = 0.1  # 分层先验强度
    
    def calibrate(self, raw_prob: float) -> float:
        """
        分层校准：
        - 参考历史概率，避免极端收缩
        - Platt scaling: sigmoid(a*x + b)
        """
        # 计算分层影响因子（远离历史概率的预测收缩更强）
        deviation = abs(raw_prob - self.historical_prob)
        shrinkage = self.alpha / (self.alpha + deviation)
        
        # Platt参数（简化）
        a, b = 1.0, 0.0  # 实际需从数据拟合
        
        # 应用校准
        calibrated = 1 / (1 + np.exp(-(a * raw_prob + b)))
        
        # 分层调整：向历史概率收缩
        final_prob = calibrated * (1 - shrinkage) + self.historical_prob * shrinkage
        return np.clip(final_prob, 0.01, 0.99)


# ========== 整合示例 ==========
def main():
    # 模拟LLM调用
    def mock_llm(prompt):
        return {"summary": "正面证据", "likelihood_ratio": 2.5, "probability": 0.6}
    
    # 初始化组件
    belief = BayesianLanguageBelief(prior_prob=0.1)
    aggregator = HierarchicalTrialAggregator(k_trials=5)
    calibrator = HierarchicalCalibration(historical_prob=0.05)
    
    # 更新信念
    llm_out = mock_llm("分析新证据")
    belief.update_with_evidence("新数据", llm_out)
    print(f"信念更新后概率: {belief.belief_state['prob']:.3f}")
    
    # 多试验聚合
    raw_probs = aggregator.run_multiple_trials("预测结果", mock_llm)
    aggregated = aggregator.aggregate(raw_probs)
    print(f"聚合后概率: {aggregated:.3f}")
    
    # 分层校准
    calibrated = calibrator.calibrate(aggregated)
    print(f"校准后概率: {calibrated:.3f}")


if __name__ == "__main__":
    main()
```

## ❓ 常见问题

**Q：为什么语言证据摘要比直接追加检索结果更有效？**

A：因为LLM在生成摘要时完成了信息压缩和重要性排序，将长上下文转换为紧凑的信念更新信号，避免了关键信息被淹没在噪声中。


**Q：5次试验是如何选择的，是否越多越好？**

A：论文实验表明K=5是效果与成本的平衡点。K增加时边际收益递减（尤其超过10次后），同时计算成本线性增长。实际应用中可根据问题难度和数据质量动态调整。
