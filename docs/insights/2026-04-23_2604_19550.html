📖 背景与动机

点击率（CTR）预测是广告和推荐系统的核心技术，直接影响平台收入。近年来，基于Transformer的CTR模型通过堆叠更多参数获得性能提升，但工业部署面临严峻挑战：参数量增长带来内存占用爆炸、推理延迟飙升，难以满足线上服务的硬性延迟约束（通常<10ms）。传统方法要么压缩模型牺牲精度，要么保持精度但无法大规模部署。LoopCTR的核心洞察是：计算量（computation）和参数量（parameters）应该解耦——用相同参数通过循环复用（loop）实现更多计算，而非一味增加参数。这一思路打破了Scaling Law的固有认知，为工业落地提供了新路径。

💡 核心方法

方法概述

LoopCTR提出循环缩放范式，通过递归复用共享层在不增加参数的前提下扩展计算量。采用三明治架构组织Entry Block、Loop Block和Exit Block，在Loop Block中引入超连接残差（Hyper-Connected Residuals）实现信息跨层流动，配合混合专家（MoE）动态选择计算路径。训练时执行多循环深度学习，推理时零循环直接输出，实现参数效率与性能的兼得。

详细设计

LoopCTR的核心架构包含三大模块。Entry Block负责原始输入的特征编码与投影，输出作为循环入口。Loop Block是核心创新所在，包含N个可复用的共享层，每次循环都将上一循环的输出重新注入这些共享层。具体实现上，假设有L个循环深度，每层l接收来自所有循环深度的残差连接，形成超连接结构。数学表达为：第l层的输出 = SharedLayerl(聚合所有循环深度的输入) + 跨循环的跳跃连接。

超连接残差的设计是关键：不同于传统ResNet的单层残差，LoopCTR让每个子层能看到分配给它的所有循环深度的信息流。具体通过one-hot选择向量es mod n实现（n为总子层数），每个子层的聚合方式为：Output_s = Σ(α_sj · SharedLayer(X_j))，其中α为可学习的路由权重。

混合专家层则嵌入在共享层内部，每个Expert对应不同的计算模式（如短期兴趣、长期依赖等），路由器根据输入动态选择Top-K个Expert参与计算，既保证了表达能力又控制了计算量。

过程监督（Process Supervision）贯穿每个循环深度：不仅最终层有监督信号，每个循环深度都接收到对应的辅助损失监督，确保多循环学习的知识能被有效编码进共享参数。这避免了深层循环梯度消失问题，让浅层也能学到有意义的中间表示。

推理时，由于共享参数已充分训练多循环的知识，即使跳过循环直接使用Entry Block的输出，也能获得超越基线的性能，实现推理零循环（Zero-Loop Inference）。

🔧 关键组件

| 组件 | 说明 |

|------|------|

| 三明治架构（Sandwich Architecture） | 由Entry Block、Loop Block（可递归复用）、Exit Block组成。Entry处理输入编码，Loop实现循环计算扩展，Exit完成输出汇聚。这种结构简化了循环设计，让Loop Block成为纯计算模块。 |

| 超连接残差（Hyper-Connected Residuals） | 每个子层不仅接收本层上一循环的输出，还接收所有其他子层在所有循环深度的信息。通过可学习的聚合权重动态融合跨循环跨层信息，确保浅层参数也能捕获深层知识。 |

| 混合专家层（Mixture-of-Experts） | 在共享层内部设置多个Expert（如短期兴趣Expert、长期兴趣Expert、交叉特征Expert等），Router根据输入动态选择Top-K个Expert计算。这让有限参数表达多种计算模式，提升模型表达能力。 |

| 过程监督（Process Supervision） | 每个循环深度都设置监督信号，而非仅在最深层监督。这确保多循环学习的信息被编码进共享参数，也解决了深层循环梯度衰减问题，使训练更稳定。 |

❓ 常见问题

Q：为什么推理零循环还能超越基线？多循环训练学到了什么？

A：多循环训练让共享层在每轮循环中逐步精炼特征表示，相当于对输入进行多层语义加工。Process Supervision确保每轮循环都学到有意义的中间表示，这些知识被编码进共享参数。推理时即使只做一次编码，共享层已具备强大的特征变换能力，能直接输出高质量CTR预估。

Q：循环深度L如何选择？是否越深越好？

A：论文实验显示L=2-4区间性能最佳，继续增加边际收益递减且训练成本上升。实际选择需权衡：数据复杂度高时可增大L，工业场景推荐L=2或L=3。Oracle分析表明更深的循环仍有提升空间，但需配合架构优化。

Q：与知识蒸馏有何区别？LoopCTR的优势在哪？

A：知识蒸馏是将大模型知识迁移到小模型，训练和推理模型结构不同。LoopCTR训练和推理共享同一套参数，只是推理时跳过循环。优势在于：(1) 无需训练额外的蒸馏模型；(2) 推理结构极简，无任何额外开销；(3) 循环深度可按需调整，灵活平衡性能与速度。
