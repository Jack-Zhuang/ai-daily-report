## 📖 背景与动机

传统推荐系统依赖稠密向量表示用户和物品，在高维隐空间中计算相似度。这种方法虽然高效，但语义不透明，难以解释推荐理由。近年来，大语言模型（LLM）的崛起为推荐系统带来了新的可能性——通过自然语言描述用户偏好和物品特征，不仅更易理解，还能与下游推理模块无缝衔接。然而，如何构建有效的文本档案、如何设计档案格式成为新的挑战。手工设计的模板往往过于僵硬，难以适应不同任务；独立生成用户和物品档案虽各自合理，但组合后可能产生语义冲突。


**关键要点：**

- 传统稠密向量方法语义不透明，缺乏可解释性
- LLM推荐依赖自然语言档案，但档案格式设计缺乏理论指导
- 独立生成用户-物品档案可能导致语义不一致问题
- 需要一种能够联合优化档案生成和任务目标的方法

## 💡 核心方法

### 方法概述

DUET是一种交互感知的档案生成框架，通过三阶段流程联合生成用户和物品档案，并利用策略优化方法自动学习最优档案格式。


### 详细设计

DUET的核心思想是打破传统先独立建模再匹配的范式，转而将用户-物品交互信息融入档案生成过程。第一阶段为线索提取（Cue Extraction），从用户历史行为和物品元数据中抽取关键特征作为生成条件；第二阶段为档案生成（Profile Generation），基于提取的线索和精心设计的档案提示模板，由LLM联合生成用户偏好档案和物品属性档案；第三阶段为对齐优化（Alignment Optimization），通过策略梯度方法（如PPO）在线探索不同的档案格式组合，根据推荐任务奖励调整档案生成策略。关键在于，档案生成器同时接收用户历史和物品证据作为上下文条件，确保生成的档案对之间具有语义一致性。此外，框架引入互对应机制（Mutual Correspondence），使生成的档案能够相互验证和约束。

### 🔧 关键组件

| 组件 | 说明 |
|------|------|
| 线索提取器（Cue Extractor） | 从原始用户行为序列和物品属性信息中提取高价值线索，包括用户偏好模式、物品核心特征等，作为档案生成的输入条件。 |
| 联合档案生成器（Joint Profile Generator） | 基于LLM的档案生成模块，同时接收用户线索和物品线索作为上下文，生成语义对齐的用户偏好档案和物品属性档案。 |
| 策略优化器（Policy Optimizer） | 采用on-policy探索策略，通过策略梯度方法（如REINFORCE或PPO）优化档案提示模板参数，最大化推荐任务奖励。 |
| 档案对齐模块（Profile Alignment Module） | 确保生成的档案对之间语义一致，通过对比学习和互信息最大化机制增强用户-物品档案的匹配度。 |

import random

# ============================================
# DUET: Dual User-Item Profile Generation
# ============================================

class CueExtractor:
    """第一阶段：线索提取"""
    
    def extract_user_cues(self, user_history):
        """从用户历史行为中提取关键特征"""
        cues = {
            'preference_tags': self._extract_preferences(user_history),
            'interaction_patterns': self._analyze_behavior(user_history),
            'implicit_signals': self._detect_signals(user_history)
        }
        return cues
    
    def extract_item_cues(self, item_metadata):
        """从物品元数据中提取关键特征"""
        cues = {
            'item_attributes': self._extract_attributes(item_metadata),
            'category_features': self._extract_categories(item_metadata),
            'content_signatures': self._extract_content(item_metadata)
        }
        return cues


class ProfileGenerator:
    """第二阶段：档案生成"""
    
    def __init__(self, llm_model):
        self.llm = llm_model
        self.prompt_template = self._load_prompt_template()
    
    def generate_profiles(self, user_cues, item_cues):
        """
        联合生成用户偏好档案和物品属性档案
        关键：同时接收用户和物品条件，确保语义一致性
        """
        # 构建双向条件上下文
        context = {
            'user_context': self._format_user_context(user_cues),
            'item_context': self._format_item_context(item_cues)
        }
        
        # 生成用户偏好档案
        user_profile = self.llm.generate(
            prompt=self.prompt_template['user_profile'],
            context=context,
            constraints=['一致性', '可解释性']
        )
        
        # 生成物品属性档案
        item_profile = self.llm.generate(
            prompt=self.prompt_template['item_profile'],
            context=context,
            constraints=['完整性', '可匹配性']
        )
        
        # 互对应机制：相互验证和约束
        user_profile, item_profile = self._mutual_correspondence(
            user_profile, item_profile
        )
        
        return user_profile, item_profile
    
    def _mutual_correspondence(self, user_profile, item_profile):
        """互对应机制：档案间的验证与约束"""
        # 交叉验证语义一致性
        consistency_score = self._check_consistency(user_profile, item_profile)
        
        if consistency_score < 0.7:
            # 不一致则重新生成或调整
            user_profile, item_profile = self._regenerate_with_feedback(
                user_profile, item_profile
            )
        
        return user_profile, item_profile


class AlignmentOptimizer:
    """第三阶段：对齐优化"""
    
    def __init__(self):
        self.learning_rate = 0.001
        self.policy_model = None
    
    def optimize(self, user_profile, item_profile, reward_signal):
        """
        使用策略梯度方法（如PPO）优化档案生成策略
        """
        # 计算当前策略的梯度
        gradient = self._compute_policy_gradient(
            current_strategy=self.policy_model,
            reward=reward_signal
        )
        
        # 更新策略模型
        self.policy_model = self._update_policy(
            model=self.policy_model,
            gradient=gradient,
            lr=self.learning_rate
        )
        
        return self.policy_model
    
    def ppo_update(self, old_log_prob, new_log_prob, advantage):
        """PPO裁剪更新"""
        ratio = torch.exp(new_log_prob - old_log_prob)
        clipped_ratio = torch.clamp(ratio, 1 - 0.2, 1 + 0.2)
        loss = -torch.min(ratio * advantage, clipped_ratio * advantage)
        return loss.mean()


class DUETRecommender:
    """DUET 整体框架"""
    
    def __init__(self):
        self.cue_extractor = CueExtractor()
        self.profile_generator = ProfileGenerator(llm_model=None)
        self.optimizer = AlignmentOptimizer()
    
    def recommend(self, user_id, candidate_items, user_history):
        """
        推荐流程：
        1. 提取线索 -> 2. 生成档案 -> 3. 匹配对齐
        """
        recommendations = []
        
        for item in candidate_items:
            # Step 1: 线索提取
            user_cues = self.cue_extractor.extract_user_cues(user_history)
            item_cues = self.cue_extractor.extract_item_cues(item.metadata)
            
            # Step 2: 档案生成（联合生成+互对应）
            user_profile, item_profile = self.profile_generator.generate_profiles(
                user_cues, item_cues
            )
            
            # Step 3: 计算匹配分数
            match_score = self._compute_match(user_profile, item_profile)
            recommendations.append((item, match_score))
        
        # 返回排序后的推荐结果
        return sorted(recommendations, key=lambda x: x[1], reverse=True)


# ============================================
# 简化使用示例
# ============================================

def demo_duet():
    """演示DUET框架的工作流程"""
    
    # 初始化
    duet = DUETRecommender()
    
    # 模拟数据
    user_history = [
        {'action': 'click', 'item': '书籍A'},
        {'action': 'purchase', 'item': '电子产品B'},
        {'action': 'view', 'item': '书籍C'}
    ]
    
    candidate_items = [
        {'id': 1, 'metadata': '科技类书籍'},
        {'id': 2, 'metadata': '文学作品'},
        {'id': 3, 'metadata': '学习资料'}
    ]
    
    # 执行推荐
    results = duet.recommend(
        user_id='user_001',
        candidate_items=candidate_items,
        user_history=user_history
    )
    
    # 输出推荐结果
    print("=== 推荐结果 ===")
    for item, score in results:
        print(f"物品: {item['id']}, 匹配分数: {score:.3f}")


# 运行演示
if __name__ == "__main__":
    demo_duet()

## ❓ 常见问题

**Q：DUET与传统基于embedding的推荐方法有何本质区别？**

A：传统方法将用户和物品编码为稠密向量，在隐空间中进行相似度计算，语义不透明。DUET通过生成自然语言档案描述用户偏好和物品特征，不仅保留了语义可解释性，还能自然地与LLM的推理能力结合，实现更直观的推荐理由生成。


**Q：档案格式的端到端优化会不会导致过拟合到特定数据集？**

A：论文采用on-policy探索结合正则化约束来缓解过拟合风险。通过在策略优化阶段引入多样性奖励，鼓励发现泛化性更好的档案结构。此外，实验部分应该包含跨数据集泛化测试，可参考原文验证。


**Q：该方法在工业场景中的计算成本如何？**

A：主要成本来自LLM推理和策略优化两个阶段。推理阶段可通过模型量化、批处理、缓存机制优化；策略优化阶段仅需离线执行一次，优化后的档案模板可复用。综合来看，对于离线推荐场景（如每日更新推荐列表），成本可接受；实时推荐场景需进一步优化。
