📖 背景与动机

现代推荐系统面临效果与效率的两难抉择。大规模候选集召回阶段通常采用轻量级的双塔模型（Two-Tower Model），通过用户塔和物品塔分别编码特征，生成向量进行快速近邻检索。然而，传统双塔架构存在三大固有问题：一是两个塔独立训练，导致表征能力受限；二是embedding空间难以对齐，跨塔特征交互不足；三是难以适应数据分布的动态变化，在线学习场景下模型更新困难。现有方案如延迟交互（Late Interaction）和知识蒸馏虽能缓解部分问题，但会带来额外延迟或增加部署复杂度。CS3的提出正是为了在保持双塔模型高效优势的同时，系统性解决这些局限性。

关键要点：

  • 孤立的双塔架构导致用户表征与物品表征分布不一致，检索精度受限

  • 现有优化方案在效果提升和部署成本之间存在矛盾

💡 核心方法

方法概述

CS3框架引入三大协同机制形成闭环优化：Cycle-Adaptive Structure让每个塔具备自修正能力，Cross-Tower Synchronization建立双向信息通道，Cascade-Model Sharing复用精排阶段的知识。三者协同使原本孤立的两个塔形成有机整体，在不增加推理复杂度的情况下显著提升效果。

详细设计

Cycle-Adaptive Structure的核心思想是引入带权重的残差连接和自适应门控机制。设第i层的塔函数为f_θi，其中包含前向变换W_i和反向重构W'_i。模型通过对比原始特征与重构特征的差异，自动学习哪些特征是噪声需要过滤，哪些是核心信号需要强化。具体实现为：输出 = σ(g_φi(f_θi(x))) f_θi(x) + (1 - σ(...)) x，其中σ为门控函数，g_φi负责生成门控信号。

Cross-Tower Synchronization采用轻量级的交叉注意力机制，让用户塔和物品塔在训练时能感知对方的状态。每个塔维护一个轻量级的'镜像'模块，在训练阶段周期性地同步另一塔的关键参数或中间表征。同步策略包括：参数同步（每T步同步一次顶层参数）、表征同步（对齐两塔的最终输出分布）和梯度同步（双向传递梯度信息）。

Cascade-Model Sharing的核心是知识蒸馏的逆向应用。传统做法是用复杂教师模型指导学生模型，这里是用下游精排模型（通常是更复杂的交互模型）的预测结果作为监督信号，反向指导双塔模型学习更利于下游任务的表征。具体通过构造软标签，将精排模型的点击率预测概率注入双塔的对比学习目标中，使双塔学到的embedding天然更适合后续的精排任务。

class CycleAdaptiveLayer(nn.Module):

"""

自适应门控机制：根据输入动态过滤噪声/强化信号

输出 = σ(g(x)) f(x) + (1 - σ(g(x))) x

"""

def __init__(self, dim):

super().__init__()

self.forward_transform = nn.Linear(dim, dim) # W_i

self.backward_transform = nn.Linear(dim, dim) # W'_i

self.gate_network = nn.Sequential(

nn.Linear(dim, dim // 2),

nn.ReLU(),

nn.Linear(dim // 2, 1)

)

def forward(self, x):

# 原始特征经过变换

f_x = self.forward_transform(x)

# 生成门控信号

gate_signal = self.gate_network(x)

gate = torch.sigmoid(gate_signal)

# 门控后的输出：保留核心信号，过滤噪声

output = gate f_x + (1 - gate) x

# 对比重构差异，用于对比学习

reconstructed = self.backward_transform(output)

reconstruction_loss = F.mse_loss(reconstructed, x)

return output, reconstruction_loss
