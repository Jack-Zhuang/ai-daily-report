## 📖 背景与动机

大型语言模型（LLMs）在歌词到旋律生成任务中展现出潜力，但传统监督微调（SFT）方法存在明显缺陷：模型产生的旋律往往节奏混乱、音域不合适，这种现象被称为「约束违规」（Constraint Violation）。现有解决思路依赖人工标注偏好数据，成本高且难以规模化。音乐创作有其固有规律（如节拍稳定性、音程合理性、音域适配性等），如何将这些专业知识自动化地注入模型训练过程，成为本研究的切入点。


**关键要点：**

- SFT训练的歌词旋律模型存在节奏混乱、音域不当等音乐性缺陷
- 人工标注偏好数据成本高、难以大规模应用
- 音乐领域存在可形式化的规则约束，适合自动化处理

## 💡 核心方法

### 方法概述

提出基于规则约束的自动对齐框架，先通过SFT微调基座模型，再利用音乐规则自动构建偏好数据集，依次应用DPO和KTO进行对齐优化。


### 详细设计

框架包含三个核心阶段。首先是SFT阶段，使用歌词-旋律配对数据微调预训练LLM作为基座模型。其次是自动偏好数据构建阶段：定义规则化音乐约束（如节奏一致性、音程跨度限制、音域范围检查等），对SFT模型输出进行多维度评估，自动生成「优-劣」配对偏好数据（正负样本）。最后是对齐优化阶段，采用顺序策略：先用DPO在配对偏好数据上优化，使模型倾向于选择符合约束的正样本；再用KTO处理非配对负样本，进一步强化对规则违规样本的惩罚。DPO阶段损失函数关注正负样本的相对偏好概率，而KTO阶段则引入前景理论和损失厌恶机制，增强模型对不良旋律的规避能力。

### 🔧 关键组件

| 组件 | 说明 |
|------|------|
| 规则约束评估器 | 基于乐理定义的多维度评分函数，对旋律进行节奏、音程、音域等规则检查，输出违规标记和置信分数 |
| 自动偏好数据生成器 | 利用评估结果自动构建正负样本对，无需人工标注，支持配对和非配对两种格式 |
| DPO-KTO顺序优化器 | 先通过DPO学习配对偏好关系，再用KTO强化对负样本的惩罚，实现渐进式对齐 |

import torch
import torch.nn as nn
from torch.utils.data import Dataset, DataLoader

# =============================================
# 第一阶段：SFT (Supervised Fine-Tuning) 微调
# =============================================

class SFTModel(nn.Module):
    """歌词-旋律配对模型"""
    def __init__(self, base_model):
        super().__init__()
        self.model = base_model
    
    def forward(self, lyrics, melody):
        """歌词-旋律序列输入"""
        return self.model(lyrics, melody)

def sft_training_loop(train_data, base_model):
    """
    SFT阶段训练
    train_data: [(lyrics, melody, label)] 格式数据
    """
    model = SFTModel(base_model)
    optimizer = torch.optim.AdamW(model.parameters(), lr=1e-5)
    
    for epoch in range(3):
        for lyrics, melody, label in train_data:
            # 前向传播
            logits = model(lyrics, melody)
            # 交叉熵损失
            loss = nn.functional.cross_entropy(logits, label)
            
            # 反向传播
            loss.backward()
            optimizer.step()
            optimizer.zero_grad()
    
    return model  # SFT模型

# =============================================
# 第二阶段：自动偏好数据构建
# =============================================

class MusicConstraintChecker:
    """规则化音乐约束检查器"""
    
    def __init__(self):
        self.rules = {
            'rhythm_consistency': self.check_rhythm,
            'interval_span': self.check_interval_span,
            'pitch_range': self.check_pitch_range
        }
    
    def check_rhythm(self, melody):
        """节奏一致性检查"""
        return True  # 伪代码：返回True/False
    
    def check_interval_span(self, melody):
        """音程跨度限制检查"""
        return True
    
    def check_pitch_range(self, melody):
        """音域范围检查"""
        return True
    
    def evaluate(self, melody):
        """多维度评估，返回综合得分"""
        score = sum(rule(melody) for rule in self.rules.values())
        return score / len(self.rules)
    
    def is_valid(self, melody):
        """判断是否满足所有约束"""
        return all(rule(melody) for rule in self.rules.values())

def build_preference_data(sft_model, constraint_checker, prompt_data):
    """
    自动构建优-劣配对偏好数据
    
    Returns:
        preferences: [(prompt, chosen_response, rejected_response)]
    """
    preferences = []
    
    for prompt in prompt_data:
        # 生成多个候选响应
        candidates = [sft_model.generate(prompt) for _ in range(4)]
        
        # 评估每个候选
        scores = [constraint_checker.evaluate(cand) for cand in candidates]
        
        # 排序选择最优/最差
        sorted_indices = torch.argsort(torch.tensor(scores), descending=True)
        chosen = candidates[sorted_indices[0]]
        rejected = candidates[sorted_indices[-1]]
        
        preferences.append((prompt, chosen, rejected))
    
    return preferences

# =============================================
# 第三阶段：对齐优化 (DPO + KTO)
# =============================================

class DPOLoss:
    """DPO (Direct Preference Optimization) 损失"""
    
    def __init__(self, beta=0.1):
        self.beta = beta
    
    def compute(self, policy_logps, ref_logps, chosen_logps, rejected_logps):
        """
        policy_logps: 当前策略模型对样本的对数概率
        ref_logps: 参考模型对样本的对数概率
        """
        # 计算相对偏好对数概率差
        log_ratio_chosen = policy_logps['chosen'] - ref_logps['chosen']
        log_ratio_rejected = policy_logps['rejected'] - ref_logps['rejected']
        
        # DPO损失公式
        diff = log_ratio_chosen - log_ratio_rejected
        loss = -torch.log(torch.sigmoid(diff) + 1e-8).mean()
        
        return loss * self.beta

class KTOLoss:
    """KTO (Kahneman-Tark operator) 损失"""
    
    def __init__(self, beta=0.1, gamma=1.0):
        self.beta = beta
        self.gamma = gamma  # 损失厌恶系数
    
    def compute(self, policy_logps, chosen_logps, rejected_logps):
        """
        引入前景理论的效用函数
        - 对正样本：增加奖励效用
        - 对负样本：增强惩罚（损失厌恶）
        """
        # 正样本效用（收益）
        pos_util = torch.sigmoid(policy_logps['chosen'] - chosen_logps)
        
        # 负样本效用（损失）- 乘以损失厌恶系数
        neg_util = torch.sigmoid(rejected_logps - policy_logps['rejected'])
        neg_util = neg_util * self.gamma  # 放大负样本惩罚
        
        # 前景理论损失
        loss = -torch.log(pos_util + 1e-8) - torch.log(neg_util + 1e-8)
        return loss.mean() * self.beta

class AlignmentTrainer:
    """对齐优化训练器"""
    
    def __init__(self, sft_model, ref_model):
        self.policy_model = sft_model  # 当前策略模型
        self.ref_model = ref_model      # 参考模型（冻结）
        self.dpo_loss = DPOLoss(beta=0.1)
        self.kto_loss = KTOLoss(beta=0.1, gamma=2.0)  # gamma > 1 表示损失厌恶
    
    def dpo_training_step(self, batch):
        """DPO阶段：处理配对偏好数据"""
        prompt, chosen, rejected = batch
        
        # 获取对数概率
        policy_logps = self.policy_model.get_log_probs(prompt, chosen, rejected)
        ref_logps = self.ref_model.get_log_probs(prompt, chosen, rejected)
        
        # 计算DPO损失
        loss = self.dpo_loss.compute(policy_logps, ref_logps, 
                                      policy_logps['chosen'], 
                                      policy_logps['rejected'])
        return loss
    
    def kto_training_step(self, unpaired_batch):
        """KTO阶段：处理非配对负样本，强化惩罚"""
        # unpaired_batch: 包含违规样本
        policy_logps = self.policy_model.get_log_probs(unpaired_batch)
        ref_logps = self.ref_model.get_log_probs(unpaired_batch)
        
        # 计算KTO损失
        loss = self.kto_loss.compute(policy_logps, 
                                      ref_logps, 
                                      ref_logps)
        return loss
    
    def sequential_training(self, pref_data, unpaired_data, epochs=3):
        """
        顺序策略训练：
        1. 先DPO优化
        2. 再KTO处理非配对负样本
        """
        for epoch in range(epochs):
            # === DPO阶段 ===
            dpo_loader = DataLoader(pref_data, batch_size=8)
            for batch in dpo_loader:
                loss_dpo = self.dpo_training_step(batch)
                loss_dpo.backward()
                self.policy_model.optimizer.step()
                self.policy_model.optimizer.zero_grad()
            
            # === KTO阶段 ===
            kto_loader = DataLoader(unpaired_data, batch_size=8)
            for batch in kto_loader:
                loss_kto = self.kto_training_step(batch)
                loss_kto.backward()
                self.policy_model.optimizer.step()
                self.policy_model.optimizer.zero_grad()
            
            print(f"Epoch {epoch}: DPO loss={loss_dpo.item():.4f}, KTO loss={loss_kto.item():.4f}")
        
        return self.policy_model

# =============================================
# 主流程
# =============================================

def main():
    # 1. 初始化基座模型（伪代码）
    base_model = "PretrainedLLM"  # 假设的预训练模型
    
    # 2. SFT阶段

    sft_model = sft_training_loop(train_data=None, base_model=base_model)

    trainer = AlignmentTrainer(sft_model, ref_model)

        unpaired_data=None,  # 非配对负样本
        epochs=3
    )
    
    print("训练完成！")

if __name__ == "__main__":
    main()

## ❓ 常见问题

**Q：为什么DPO和KTO需要顺序使用而非同时使用？**

A：两者侧重点不同：DPO擅长处理配对偏好关系（哪个样本更好），KTO擅长处理非配对负样本的惩罚（明确知道哪个不好）。顺序使用可以先建立正向偏好基线，再强化负向规避能力，效果优于简单混合。


**Q：规则约束是否会导致生成的旋律过于标准化、缺乏创意？**

A：框架设计中规则主要用于消除明显的音乐错误（如超出人类音域的音符），而非强制统一风格。论文实验显示，对齐后的模型仍能保持旋律多样性，只是在基本音乐合理性上有所提升，创意空间得到保留。
