## 📖 背景与动机

在机器人感知、工业监控、自动驾驶等实时系统中，观测数据以流式方式持续到达。系统需要将每个新观测增量地整合到共享记忆库中，这一类记忆称为顺序关联记忆（SAM）。传统的实现方式依赖超维计算（HDC）和向量符号架构（VSA），通过绑定（binding）和打包（bundling）操作将观测信息编码并累积到记忆向量中。然而，现有方法对所有观测一视同仁，将无关或噪声数据也纳入记忆，导致记忆内容逐渐被污染，在长期运行中性能严重衰减。如何在保持计算效率的前提下，实现选择性记忆更新，成为关键挑战。


**关键要点：**

- 流式感知系统需要持续整合时间序列观测数据到共享记忆
- 传统VSA方法使用打包操作无差别地累积所有观测
- 无关信息的累积导致记忆向量逐渐偏离正确状态
- 需要在记忆保真度与计算效率之间取得平衡

## 💡 核心方法

### 方法概述

SRMU引入相关性门控机制，在每次观测到达时先评估其与当前记忆的相关性，仅当相关性超过阈值时才执行记忆更新操作。核心思想是让记忆具有“选择性遗忘”能力，拒绝接纳无关信息。


### 详细设计

SRMU的运作分为三个阶段：

1. **相关性评估**：给定当前记忆向量M和观测编码o，通过相似度度量（如余弦相似度）判断o是否与M相关。计算 sim(M, o) = cos(M, o)，若 sim > τ（门控阈值），认为观测具有相关性。

2. **选择性绑定**：对于相关观测，执行绑定操作 M' = M ⊗ o，将新信息编码进记忆。对于不相关观测，记忆保持不变 M' = M。

3. **解绑恢复**（可选）：需要检索时，执行解绑操作 o' = M ⊗⁻¹，通过逆运算恢复原始观测信息。

对于FHRR（傅里叶全息约简表示）编码，绑定操作对应元素级乘法，解绑操作对应元素级共轭乘法。SRMU的关键参数包括：维度D（通常1000-10000）、门控阈值τ、更新策略（硬门控/软门控）。

| FHRR编码器 | 将原始观测映射到高维复数向量空间，观测通过元素级随机相位编码，支持高效绑定/解绑操作 |

### 💻 代码示例

```python
import numpy as np

class SRMU:
    """选择性记忆更新单元 (Selective Memory Update Unit)"""
    
    def __init__(self, dim=1000, threshold=0.5):
        self.D = dim           # 记忆维度
        self.tau = threshold   # 门控阈值
        self.memory = None     # 记忆向量 M
    
    def cosine_sim(self, M, o):
        """相关性评估: 计算余弦相似度"""
        # 归一化后点积即为余弦相似度
        M_norm = M / np.linalg.norm(M)
        o_norm = o / np.linalg.norm(o)
        return np.real(np.dot(M_norm.conj(), o_norm))
    
    def bind(self, M, o):
        """选择性绑定: FHRR使用元素级乘法"""
        return M * o
    
    def unbind(self, M, o):
        """解绑恢复: FHRR使用共轭乘法"""
        return M * np.conj(o)
    
    def update(self, observation):
        """SRMU核心更新逻辑"""
        # Step 1: 相关性评估
        if self.memory is None:
            self.memory = observation.copy()
            return self.memory
        
        similarity = self.cosine_sim(self.memory, observation)
        
        # Step 2: 选择性绑定
        if similarity > self.tau:
            # 相关观测 → 绑定更新
            self.memory = self.bind(self.memory, observation)
        else:
            # 不相关观测 → 记忆不变
            pass
        
        return self.memory
    
    def retrieve(self, cue):
        """解绑恢复原始信息"""
        return self.unbind(self.memory, cue)


# ============== 使用示例 ==============
if __name__ == "__main__":
    # 初始化SRMU
    srmu = SRMU(dim=100, threshold=0.3)
    
    # 模拟观测序列
    observations = [
        np.random.randn(100) + 1j * np.random.randn(100),  # 观测1
        np.random.randn(100) + 1j * np.random.randn(100),  # 观测2
        np.random.randn(100) + 1j * np.random.randn(100),  # 观测3
    ]
    
    # 处理观测序列
    for i, obs in enumerate(observations):
        obs = obs / np.linalg.norm(obs)  # 归一化
        memory = srmu.update(obs)
        print(f"观测{i+1} 更新后记忆维度: {memory.shape}")
    
    # 解绑恢复
    cue = observations[0] / np.linalg.norm(observations[0])
    recovered = srmu.retrieve(cue)
    print(f"解绑恢复结果维度: {recovered.shape}")
```

## ❓ 常见问题

**Q：SRMU与LSTM等RNN的记忆机制有何区别？**

A：LSTM通过可学习的门控网络记住“模式”，而SRMU通过固定的相似度度量记住“具体内容”。SRMU更适合需要精确检索原始观测的场景，LSTM更适合模式识别任务。两者可互补使用。


**Q：如何确定合适的门控阈值τ？**

A：建议通过验证集调优：在无噪声场景下可用较高阈值（如0.7-0.8），在高噪声环境建议降低（如0.3-0.5）。也可采用软门控方式，用相似度值加权更新而非硬判决。
