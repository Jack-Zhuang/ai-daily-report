## 💡 核心方法

### 方法概述

RaTA-Tool采用检索增强的思路，首先利用MLLM理解用户的多模态查询(如截图+文字描述)，然后通过向量检索从工具库中找到候选工具，最后结合策略网络进行精细化排序选择。


### 详细设计

系统首先对HuggingFace模型卡片等工具文档进行多模态预处理，提取文本描述、代码示例、界面截图等多模态特征，并构建索引。在查询阶段，用户的任务描述(如我想识别图片中的文字)会连同可能的界面截图一起输入MLLM，MLLM生成查询向量后与工具库中的向量进行相似度检索。检索到的Top-K候选工具会进入策略网络进行二次排序，该网络通过强化学习训练，学习如何在检索结果基础上做出最优选择决策。核心训练目标是最小化策略πθ与参考策略的KL散度，通过参数β控制正则化强度，确保策略既能利用检索到的先验知识，又能灵活适应具体任务。


### 🔧 关键组件

| 组件 | 说明 |
|------|------|
| 多模态工具编码器 | 对工具的模型卡片、截图、代码等异构信息进行统一编码，输出统一的向量表示 |
| 检索-策略双阶段选择器 | 先用向量检索快速召回候选集，再用训练好的策略网络进行精细排序，兼顾效率和精度 |
| 正则化策略优化模块 | 通过KL散度正则化将检索先验与策略学习结合，β参数平衡探索与利用 |

import numpy as np

# ==================== 1. 多模态特征提取器 ====================
class MultimodalPreprocessor:
    """从工具文档中提取文本、代码、图像特征"""
    
    def __init__(self):
        self.text_encoder = None    # 文本编码器
        self.image_encoder = None   # 图像编码器
        self.code_embedder = None   # 代码嵌入器
    
    def extract_features(self, tool_doc):
        """
        提取多模态特征
        - tool_doc: 包含 description, code_snippets, screenshots
        """
        # 提取各模态特征
        text_feat = self.text_encoder.encode(tool_doc.description)
        code_feat = self.code_embedder.process(tool_doc.code_snippets)
        img_feat = self.image_encoder.encode(tool_doc.screenshots)
        
        # 加权融合 (文本0.4, 代码0.3, 图像0.3)
        fused = 0.4 * text_feat + 0.3 * code_feat + 0.3 * img_feat
        return fused

# ==================== 2. 工具索引构建 ====================
class ToolIndex:
    """构建工具向量索引"""
    
    def __init__(self):
        self.tools = []      # 工具元数据
        self.vectors = None  # 工具向量矩阵 (N, D)
    
    def build(self, tool_documents):
        preprocessor = MultimodalPreprocessor()
        vectors = []
        for doc in tool_documents:
            vec = preprocessor.extract_features(doc)
            vectors.append(vec)
            self.tools.append(doc)
        self.vectors = np.array(vectors)
        print(f"索引构建完成: {len(self.tools)} 个工具")

def retrieve_top_k(query_vec, tool_index, k=5):
    """基于余弦相似度检索Top-K工具"""
    # 计算相似度
    similarities = np.dot(tool_index.vectors, query_vec)
    
    # 取Top-K索引
    top_k_idx = np.argsort(similarities)[-k:][::-1]
    top_k_scores = similarities[top_k_idx]
    
    return top_k_idx, top_k_scores

# ==================== 5. 策略网络二次排序 ====================
class PolicyNetwork:
    """强化学习训练的策略网络"""
    
    def __init__(self, dim):
        self.theta = np.random.randn(dim)  # 网络参数
    
    def score(self, query_vec, tool_vec):
        """计算策略得分"""
        # 简单加权组合 + softmax
        combined = np.concatenate([query_vec, tool_vec])
        logits = np.dot(self.theta, combined)
        return logits
    
    def rerank(self, query_vec, candidates, candidates_vecs):
        """对候选工具二次排序"""
        scores = []
        for vec in candidates_vecs:
            s = self.score(query_vec, vec)
            scores.append(s)
        # 按得分排序
        order = np.argsort(scores)[::-1]
        return order

# ==================== 6. KL散度训练目标 ====================
def compute_kl_divergence(pi_theta, pi_ref, states, actions):
    """计算策略πθ与参考策略π_ref的KL散度"""
    log_ratio = 0
    for s, a in zip(states, actions):
        p_theta = pi_theta.prob(s, a)
        p_ref = pi_ref.prob(s, a)
        log_ratio += np.log(p_theta / p_ref)
    return log_ratio / len(actions)

def train_step(policy_net, ref_policy, query, candidates, beta=0.1):
    """
    训练步骤: 最小化KL散度
    beta: 正则化强度参数
    """
    # 获取候选工具向量
    cand_vecs = np.array([c.vector for c in candidates])
    
    # 策略网络二次排序
    ranking = policy_net.rerank(query, candidates, cand_vecs)
    selected = [candidates[i] for i in ranking]
    
    # 计算KL散度损失
    kl_loss = compute_kl_divergence(policy_net, ref_policy, 
                                     [query], [selected[0]])
    
    # 总损失 = KL散度 * beta
    loss = beta * kl_loss
    
    # 反向传播更新参数 (伪代码)
    # loss.backward()
    # optimizer.step()
    
    return loss, selected

## ❓ 常见问题

**Q：为什么需要多模态理解？纯文本匹配不够吗？**

A：不够。许多工具的核心功能通过界面截图、示例输出等视觉信息才能准确表达，纯文本匹配容易遗漏这些关键线索，尤其对视觉类工具(如图像生成、目标检测)的识别准确率较低。


**Q：策略网络和普通检索有何区别？**

A：普通检索只做相似度匹配，而策略网络会学习任务相关的选择偏好，能够根据用户意图的细微差异做出更精准的判断，比如区分「图像风格迁移」和「图像增强」这类功能相近但实际不同的工具。
