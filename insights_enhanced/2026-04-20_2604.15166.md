## 📖 背景与动机

随着深度学习模型在各行业的广泛应用，如何从已训练模型中高效移除特定知识成为重要研究课题。传统的模型重训练方法虽然效果可靠，但计算成本高昂，在大数据集场景下几乎不可行。现有的类别遗忘方法多依赖梯度优化（如SGA、EU或retraining），不仅耗时，还可能无法彻底清除遗忘类别的信息。更为关键的是，现有方法往往只是抑制了分类器头层的输出，而遗忘类别的特征表示仍然深植于网络中间层，形成"隐性记忆"。这些残留信息不仅造成选择性（selectivity）指标低下，还可能通过表示空间分析被攻击者恢复。


**关键要点：**

- 机器遗忘是隐私保护和模型更新的关键技术，但完全重训练成本过高
- 现有方法存在"虚假遗忘"问题：分类精度下降但内部表征仍保留遗忘信息
- 多层网络中遗忘方向的保留导致选择性弱化，是当前方法的核心瓶颈

## 💡 核心方法

### 方法概述

DAMP是一种无需梯度优化的单次权重手术方法，通过在网络各层计算类别原型、提取遗忘方向残差并进行投影移除，实现对遗忘类别信息的彻底清除。


### 详细设计

DAMP的核心思想是将遗忘操作建模为在表示空间中移除"遗忘方向"的过程。具体流程如下：首先，对于每个遗忘类别f，在下一层可学习算子的输入空间中计算其原型（prototype）表示。接着，将遗忘类原型相对于保留类原型进行投影，提取出"遗忘特异性方向"——即遗忘类相对于保留类的残差方向。值得注意的是，该方法会跳过接近零的残差方向（这些方向对遗忘任务无贡献），仅保留K个显著方向。然后，将这些遗忘方向投影到网络权重矩阵上，通过闭式计算直接修改权重，无需任何梯度反传。整个过程逐层进行，最终实现从浅层到深层的全方位遗忘。DAMP的理论优势在于：（1）闭合形式解保证了一次性完成；（2）架构无关性适用于任意神经网络结构；（3）严格移除表示层面的遗忘痕迹，而非仅抑制输出层。


### 🔧 关键组件

| 组件 | 说明 |
|------|------|
| 类别原型计算 | 在每层的输入空间中计算遗忘类和保留类的平均特征表示，作为方向分析的基础。 |
| 遗忘方向残差提取 | 通过遗忘类原型相对保留类原型的投影，提取仅属于遗忘类的特异性方向残差，跳过近零残差以筛选有效方向。 |
| 权重投影移除 | 将提取的遗忘方向投影到权重矩阵上，通过闭合形式直接修改权重，实现遗忘信息的彻底清除。 |

import numpy as np

# =============================================

def compute_prototype(model, dataloader, class_id, layer_idx):
    """
    计算指定类别在某一层的原型表示
    """
    # 钩子函数获取中间层激活值
    activations = []
    
    def hook(module, input, output):
        activations.append(output.detach().cpu().numpy())
    
    # 注册hook获取第layer_idx层的输出
    handle = model.layers[layer_idx].register_forward_hook(hook)
    
    # 获取该类别的样本
    for batch in dataloader:
        if batch['label'] == class_id:
            model(batch['input'])
            break
    
    handle.remove()
    
    # 计算原型（均值）
    prototype = np.mean(activations, axis=0)
    return prototype


def extract_forget_directions(prototypes_forget, prototypes_retain, k=5, threshold=1e-6):
    """
    提取遗忘特异性方向（残差方向）
    """
    forget_dirs = []
    
    for f_class in prototypes_forget:
        # 计算相对于保留类原型的残差方向
        for r_class in prototypes_retain:
            residual = f_class - r_class
            
            # 跳过接近零的方向
            if np.linalg.norm(residual) < threshold:
                continue
            
            forget_dirs.append(residual)
    
    # 选择K个最显著的方向（按范数降序）
    forget_dirs = sorted(forget_dirs, key=lambda x: np.linalg.norm(x), reverse=True)
    
    return forget_dirs[:k]


def project_to_weights(weight_matrix, forget_direction, learning_rate=1.0):
    """
    将遗忘方向投影到权重矩阵（闭式解）
    """
    # 计算投影矩阵: W @ (direction / ||direction||)
    # 核心操作：沿遗忘方向修改权重
    direction_norm = np.linalg.norm(forget_direction)
    
    if direction_norm < 1e-10:
        return weight_matrix
    
    # 闭式投影计算
    projection = np.outer(weight_matrix @ forget_direction, forget_direction) / (direction_norm ** 2)
    
    # 直接修改权重（无梯度反传）
    new_weights = weight_matrix - learning_rate * projection
    
    return new_weights


def damp_unlearning(model, forget_classes, retain_classes, dataloader, k=5):
    """
    DAMP 主流程：逐层进行机器遗忘
    """
    num_layers = len(model.layers)
    
    for layer_idx in range(num_layers):
        print(f"处理第 {layer_idx} 层...")
        
        # Step 1: 计算遗忘类和保留类的原型
        prototypes_forget = [
            compute_prototype(model, dataloader, c, layer_idx) 
            for c in forget_classes
        ]
        prototypes_retain = [
            compute_prototype(model, dataloader, c, layer_idx) 
            for c in retain_classes
        ]
        
        # Step 2: 提取遗忘特异性方向
        forget_dirs = extract_forget_directions(
            prototypes_forget, prototypes_retain, k=k
        )
        
        # Step 3: 逐方向投影到权重
        current_weights = model.layers[layer_idx].weight.data.numpy()
        
        for direction in forget_dirs:
            current_weights = project_to_weights(current_weights, direction)
        
        # Step 4: 更新权重（原地修改）
        model.layers[layer_idx].weight.data = torch.from_numpy(current_weights)
    
    return model

## ❓ 常见问题

**Q：为什么现有方法会产生"虚假遗忘"？**

A：因为它们主要修改分类器头层的偏置或权重，改变了输出分布，但遗忘类别的特征表示仍然深植于网络中间层。DAMP通过在各层移除遗忘特异性方向，从根本上解决了这一问题。


**Q：DAMP相比重训练的优势在哪里？**

A：DAMP无需访问原始训练数据，也不进行耗时的梯度反传，仅通过单次闭式计算即可完成遗忘，时间和计算成本大幅降低，同时遗忘效果更彻底。


**Q：如何确定应该移除多少个遗忘方向（K值）？**

A：K值需要在遗忘彻底性和保留精度之间权衡。论文建议通过验证集选择K：当遗忘方向残差的模长接近零时，说明该方向对遗忘贡献较小，可以跳过。建议从较大K值开始，逐步降低直到遗忘效果下降。
