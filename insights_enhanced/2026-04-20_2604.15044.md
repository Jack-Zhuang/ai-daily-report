## 💡 核心方法

### 方法概述

CoGrid提供轻量级网格世界仿真环境，支持NumPy（便于调试）和JAX（高性能）双后端无缝切换；MUG则将仿真环境编译为WebAssembly，实现浏览器端直接运行，并通过回滚网络代码处理多人交互延迟。


### 详细设计

CoGrid的核心设计遵循「环境抽象-智能体接口-奖励定义」三层架构。用户可通过继承GridAgent基类快速定义新智能体类型，利用InteractionReward基类灵活配置交互奖励。环境本身完全确定性——给定相同随机种子和动作序列，必须产生完全一致的观测和奖励变化，这对实验复现至关重要。

MUG的运行时架构采用客户端执行模式：仿真逻辑以WebAssembly形式嵌入浏览器，在本地运行以消除单玩家延迟。对于多人场景，支持两种网络模式：服务器权威模式（服务器作为唯一真值来源）和P2P模式（通过回滚机制处理网络不同步）。回滚机制允许客户端在收到对方延迟动作时暂时「猜算」后续状态，待收到确认后回滚重算。系统支持任意数量的人类和AI参与者参与同一实验，并提供实时数据采集接口。


### 🔧 关键组件

# --- 第一层：环境抽象 ---
class GridEnv:
    def __init__(self, seed=42):
        self.seed = seed  # 确定性环境
        self.grid = None
    
    def reset(self):
        self.grid = self._create_grid()
        return self._get_observation()
    
    def step(self, actions):
        """给定相同种子和动作，必须产生完全一致的输出"""
        for agent_id, action in actions.items():
            self._apply_action(agent_id, action)
        return self._get_observation(), self._get_rewards()
    
    def _get_observation(self):
        pass  # 返回当前观测
    
    def _get_rewards(self):
        pass  # 返回奖励


# --- 第二层：智能体接口 ---
class GridAgent:
    def __init__(self, agent_id):
        self.agent_id = agent_id
    
    def select_action(self, observation):
        raise NotImplementedError
    
    def update(self, obs, reward, done):
        raise NotImplementedError


# 快速定义新智能体
class RandomAgent(GridAgent):
    def select_action(self, observation):
        return {"move": random.choice(["up", "down", "left", "right"])}


# --- 第三层：奖励定义 ---
class InteractionReward:
    def calculate(self, agents, events):
        return {agent_id: 0 for agent_id in agents}


class CollisionPenalty(InteractionReward):
    def calculate(self, agents, events):
        return {agent_id: -1 if "collision" in events else 0 for agent_id in agents}


# ============================================
# MUG 运行时架构 - 伪代码示例
# ============================================

class MUGClient:
    def __init__(self, mode="server_authoritative"):
        self.mode = mode
        self.state = None
        self.pending_actions = []  # 延迟动作队列
        self.state_history = []    # 回滚用状态历史
    
    def receive_action(self, agent_id, action, timestamp):
        """接收远程动作"""
        if self.mode == "p2p":
            self.pending_actions.append((agent_id, action, timestamp))
            self._speculative_execute(agent_id, action)
        else:
            self._apply_server_state(action)
    
    def _speculative_execute(self, agent_id, action):
        """猜算后续状态（回滚机制）"""
        # 保存当前状态用于回滚
        self.state_history.append(self._clone_state())
        
        # 暂时应用动作
        self._apply_action(agent_id, action)
    
    def on_server_confirm(self, confirmed_state, action_history):
        """服务器确认后回滚重算"""
        if self.mode == "p2p":
            if self._state_mismatch(confirmed_state):
                self._rollback_to(confirmed_state)
                self._replay_actions(action_history)
    
    def _rollback_to(self, state):
        """回滚到指定状态"""
        self.state = state
    
    def _replay_actions(self, action_history):
        """重算确认的动作序列"""
        for agent_id, action in action_history:
            self._apply_action(agent_id, action)


# ============================================
# 简化使用示例
# ============================================

def demo():
    # 创建环境
    env = GridEnv(seed=123)
    obs = env.reset()
    
    # 创建智能体
    agents = {
        "player1": RandomAgent("player1"),
        "player2": RandomAgent("player2"),
        "ai_1": RandomAgent("ai_1")
    }
    
    # 配置奖励
    reward_config = CollisionPenalty()
    
    # 模拟多人交互
    for step in range(10):
        actions = {aid: agent.select_action(obs) for aid, agent in agents.items()}
        obs, rewards = env.step(actions)
        
        final_rewards = reward_config.calculate(agents.keys(), obs["events"])
        
        print(f"Step {step}: Rewards = {final_rewards}")
    
    # MUG 网络同步演示
    client = MUGClient(mode="p2p")
    client.receive_action("player1", "move_right", timestamp=100)
    client.receive_action("player2", "move_left", timestamp=99)  # 延迟到达


if __name__ == "__main__":
    demo()

## ❓ 常见问题

**Q：CoGrid与已有Minigrid多智能体扩展的主要区别是什么？**

A：CoGrid更注重实验可复现性（确定性保证）和部署便利性（双后端+Web编译），而Minigrid扩展主要聚焦仿真功能本身，缺乏端到端的实验部署能力。


**Q：如果我想在自己的网站上部署MUG实验，需要什么条件？**

A：需要一台可运行Python服务器（用于编译环境和分发WASM），以及基础的Web服务器（如Nginx）托管静态资源。MUG提供了Docker镜像简化部署流程。


**Q：回滚机制具体如何工作？会影响实验结果的正确性吗？**

A：当客户端A发送动作后等待确认时，本地继续「预测性」执行；若收到服务器权威状态与本地预测不符，则回滚至正确状态重新计算。设计上保证最终所有客户端收敛至服务器真值，不会产生永久错误状态。
